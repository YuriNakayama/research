# Markdown Pages — 要件定義

## 背景と目的

自動生成される日次リサーチレポート（`docs/daily/`）と深掘りリサーチ（`docs/research/`）のMarkdownファイルを、ディレクトリ構造をそのまま維持しながらWebブラウザ上で閲覧可能にする。Markdownファイルの追加・変更のみで自動的にWebサイトに反映され、コード変更を一切必要としない「ゼロコード変更」運用を実現する。

## ユーザーストーリー

- チームメンバーとして、ブラウザから日次レポートを素早く確認したい。GitHubを開かずに済むようにしたい。
- チームメンバーとして、蓄積されたリサーチ成果をディレクトリツリーで探索し、目的の文書にすばやくアクセスしたい。
- チームメンバーとして、Mermaid図やLaTeX数式を含むリサーチドキュメントを正しくレンダリングされた状態で閲覧したい。
- コンテンツ管理者として、Markdownファイルをgit pushするだけでWebサイトに反映させたい。コードの変更やデプロイ操作は不要としたい。

## 機能要件

1. **Markdownビューア**: `docs/research/` および `docs/daily/` 配下の全Markdownファイルをブラウザ上でレンダリング表示する
2. **ディレクトリツリーナビゲーション**: サイドバーにディレクトリ構造をツリー形式で表示し、展開/折りたたみ操作で階層を探索できる
3. **パンくずリスト**: 現在表示中のページの階層位置を示すパンくずナビゲーション
4. **catch-allルーティング**: `[[...slug]]` を使用し、任意の深さのディレクトリ構造に対応
5. **Mermaidダイアグラム対応**: `mermaid` コードブロックをブラウザ上で図として描画
6. **既存Markdown処理の維持**: LaTeX数式、GFMテーブル、コードハイライト、画像表示、目次（ToC）の既存機能を継続
7. **ディレクトリインデックス**: ディレクトリパスへのアクセス時に、配下のファイル・サブディレクトリ一覧を表示
8. **自動デプロイ**: mainブランチへのpush時にAmplifyが自動ビルド・デプロイ。ビルドスクリプトで `docs/` をfrontend内にコピー

## 非機能要件

- **認証**: 既存のCognito認証を維持（社内チームメンバーのみアクセス可能）
- **パフォーマンス**: 完全静的生成（SSG）。ランタイムコストゼロ
- **データソース**: ファイルシステムのみ（DynamoDB連携は廃止）
- **ゼロコード変更運用**: Markdownファイルの追加・変更・削除のみでコンテンツが自動反映
- **レスポンシブ**: デスクトップ（サイドバー常時表示）とモバイル（ドロワー形式）に対応

## スコープ外

- 全文検索機能（将来対応）
- 外部公開（認証なしアクセス）
- Markdownエディタ（Webからの編集機能）
- バージョン管理・差分表示
- コメント・アノテーション機能

## 用語集

| 用語 | 説明 |
|------|------|
| ゼロコード変更 | Markdownファイルのみの変更でWebサイトが更新される運用形態 |
| catch-allルート | Next.js の `[[...slug]]` による任意階層対応ルーティング |
| ディレクトリインデックス | ディレクトリパスアクセス時に表示されるファイル・フォルダ一覧ページ |
