# UrzaGPT:LoRAでチューニングしたLLMによるリアルタイムカードドラフト

## 概要

Bertram ら(arXiv 2025, 2508.08382)による UrzaGPT は、Magic: The Gathering のドラフト(パックから1枚選ぶ意思決定)を**純粋な言語タスク(次トークン予測)として定式化**し、LoRA で微調整した小型LLMでリアルタイム推薦を行うドメイン適応モデルである。狙いは「LLMの言語モデリング能力を活かして、ゲームの新エキスパンションにも素早く適応できるドラフトAI」を作ること。データは 17lands.com の公開ドラフトログ(人間の実際のピック)。

## 手法の要点

- **プロンプト形式**: 「You are an expert Magic: The Gathering drafter. My pool so far: [これまでのプール] Current pack: [現パックのカード] Which card should I pick? Respond with the card name only.」現デッキ+選択肢を提示し、カード名1つを出力させる。逐次意思決定を言語補完に変換。
- **ベースモデルとLoRA設定**: Llama-3-8B、Mistral-7B-Instruct、比較用に GPT-4o(zero-shot)。LoRA rank=8(ablationで4/8/16/32)、alpha=16、最大10,000ステップ、有効バッチ32、最大系列512トークン・出力30トークン。A100×3で1モデル約6時間。
- **データ**: 17lands の NEO エキスパンション。総ピック数 約512万、訓練に100万サンプル、テストに1万サンプルを使用。「複数の妥当なピックがある」ノイズあるグラウンドトゥルースだが、人間の選択を訓練ターゲットとする。
- **評価指標**: pick accuracy(モデルの推薦カードが人間の実選択と一致した割合)。副次的に「選んだカードがデッキの主要2色に沿うか」も測定。
- **カード表現**: カード名のみ vs カードテキスト全文。名前のみの方がプロンプトが短くスループットが高く、精度も高い(全文はプロンプト希釈で逆効果)。

## 主要な結果や知見

- **未調整の小型LLMはドラフト不能**(Llama-3-8B/Mistral-7Bはランダム約22%以下、不正な出力も)。**GPT-4o のzero-shotで43%**(名前のみ)、全文だと38%に低下。
- **LoRA微調整で激変**: わずか **1,000ステップで60%超**(大型未調整モデルを凌駕)。Llama-3-8B は10,000ステップで **66.2%** に到達、Mistral-7B-Instruct は 64.3%。ドメイン特化のSOTAは68%で、汎用LLM+軽量LoRAがそこに肉薄。
- LoRA rank の効果: rank4で約63%、rank8で66.2%、16以降は逓減。
- **限界**: 単一エキスパンション(NEO)のみ評価で、クロスエキスパンション汎化は未検証。推論レイテンシの具体値も未報告(名前のみでスループット向上とのみ言及)。

## 本コンペ(PTCG AI Battle)への応用

- **意思決定の「言語タスク化」という設計パターン**: PTCGの obs_dict は logs(行動履歴)/current(現局面)/select(選択肢)という構造で、これらを自然言語/構造化テキストにシリアライズしてLLMに「どのindexを選ぶか」を問う設計が UrzaGPT と同型に組める。特に**デッキ選択フェーズ(current=None)**や、合法手が数十程度に絞られる局面では、LLMの汎用推論+少量LoRAで強い方策を作れる可能性。
- **少データ・低コスト適応**: 「1,000ステップのLoRAで60%超」という結果は、人間の上位プレイログ(あるいは自作強エージェントのリプレイ)を教師に**安価にドメイン適応**できることを示す。PTCGスタンダードは約2000枚で頻繁に環境変化するが、新カード追加時もLoRA再学習で素早く追従できる。
- **10分制約への現実的注意**: 本論文が推論速度を詰めていない点はそのまま本コンペのリスク。「1プレイヤー最大10分・時間切れ即敗北」のため、もしLLMを推論時に使うなら (a) 量子化した小型モデル+短プロンプト(カード名のみ表現)、(b) 1手あたりの生成トークンを最小化(index/カード名のみ)、(c) 多くの局面はキャッシュ/ルールで処理しLLM呼び出しを限定、といった工夫が必須。むしろLLMは**オフラインでのデッキ構築・カード選択・戦略レポート生成(Strategy部門)**で使い、対戦中(Simulation部門)は軽量方策に蒸留する二段構えが堅実。
- **合法手index制約との整合**: LLMがカード名を出す UrzaGPT 方式は不正手を生むリスクがある(未調整モデルが不正出力したのと同様)。本コンペは `list[int]` で合法手indexを返す契約なので、LLM出力を**select内の候補に強制マッピング(制約付きデコーディング/最近傍照合)**して必ず合法手に落とすラッパが必要。
- **Strategy部門への直接活用**: 技術レポート評価のStrategy部門では、UrzaGPTのような「ピック根拠を言語で説明できる」モデルは、デッキ選択の合理性を文章化する用途にそのまま使える。
