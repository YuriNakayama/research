# M³TN: Multi-gate Mixture-of-Experts based Multi-valued Treatment Network for Uplift Modeling

- **Link**: https://arxiv.org/abs/2401.14426 (HTML: https://arxiv.org/html/2401.14426v1)
- **Authors**: Zexu Sun, Xu Chen（中国人民大学 / Gaoling School of Artificial Intelligence, Renmin University of China）
- **Year**: 2024
- **Venue**: ICASSP 2024（2024 IEEE International Conference on Acoustics, Speech and Signal Processing, Seoul, Korea, 14-19 April 2024）
- **Type**: 国際会議論文（フルペーパー）
- **M³TN の正式名称**: **M**ulti-gate **M**ixture-of-Experts based **M**ulti-valued **T**reatment **N**etwork（頭の3つの M を取って M³TN）

---

## Abstract (English)

Uplift modeling predicts how treatments (e.g., discounts, interventions) affect individual responses and is widely used in marketing and other application domains. The paper focuses on **multi-valued treatment** uplift modeling, where more than one treatment level exists. The authors identify two key limitations of existing multi-valued treatment methods: (1) uplift is computed indirectly from independently predicted responses, which does not guarantee a consistent uplift distribution between treatment and control groups and can cause **cumulative errors** as the number of treatments grows; and (2) using a large number of prediction heads (one per treatment) inflates the number of model parameters and reduces efficiency. To address these, the authors propose **M³TN**, which consists of two components: (1) a **feature representation module with Multi-gate Mixture-of-Experts (MMoE)** to improve efficiency, and (2) a **reparameterization module** that models uplift **explicitly** (response = baseline response + uplift) to improve effectiveness. Extensive experiments on a public dataset and a real-world production dataset demonstrate the effectiveness and efficiency of M³TN.

## Abstract (日本語)

アップリフトモデリングは、施策（割引・介入など）が個々のユーザーの反応にどの程度効果をもたらすかを予測する技術で、マーケティング等で広く使われる。本論文は、複数の施策水準が存在する **multi-valued treatment（多値処置）** 設定でのアップリフトモデリングを対象とする。著者らは既存手法の2つの課題を指摘する。(1) アップリフトを、独立に予測した2つの反応の差分として **間接的に** 計算するため、処置群と対照群でアップリフト分布の整合性が保証されず、処置数が増えるほど **累積誤差** を生む。(2) 処置ごとに予測ヘッドを持つとパラメータ数が膨れ上がり効率が低下する。これらに対し、著者らは **M³TN** を提案する。M³TN は (1) 効率を高める **Multi-gate Mixture-of-Experts（MMoE）による特徴表現モジュール** と、(2) 有効性を高めるためアップリフトを **明示的に**（反応 = ベースライン反応 + アップリフト）モデル化する **reparameterization module** の2要素からなる。公開データセットと実運用データセットでの実験により、有効性と効率の両立を示す。

## Overview（概要）

M³TN は、**多値処置**（例: 3段階の割引率、複数のクーポン種別、複数の介入強度）に対するアップリフト（CATE: Conditional Average Treatment Effect）推定を、単一ネットワークで効率よく・整合的に行うためのアーキテクチャである。核となるアイデアは2つ:

1. **表現の共有（MMoE）**: 全処置に共通の N 個のエキスパートネットワークを用意し、処置ごとのゲート（Multi-gate）でエキスパートを重み付け合成して処置別の表現を得る。処置ごとに独立した bottom を積む（shared-bottom / 複数タワー）方式に比べ、パラメータ数を抑えつつ処置間で知識を共有できる。
2. **アップリフトの明示的モデル化（reparameterization）**: 対照反応 μ₀(x) を共有ヘッドで予測し、各処置のヘッドは反応そのものではなく **アップリフト τ̂ᵏ(x) を直接出力** する。処置反応は μₖ(x) = μ₀(x) + τ̂ᵏ(x) と再パラメータ化して構成する。これにより、2つの独立予測の差分に伴う累積誤差を回避する。

結果として、S/T-Learner・TARNet・CFRNet・MEMENTO・Hydranet などの既存手法を上回る精度（mQini / mKendall）を、より小さいモデルサイズで達成した。

## Problem（課題）

- **間接的アップリフト計算による誤差**: 従来手法は μₖ(x)（処置反応）と μ₀(x)（対照反応）を独立に学習し、その差 τ̂ᵏ = μₖ − μ₀ をアップリフトとする。2つの予測が独立ゆえ、処置群と対照群でアップリフト分布の整合性が保証されない。
- **多値処置での累積誤差**: 処置数 K が増えるほど、各処置ヘッドの反応予測誤差が差分に累積し、アップリフトのランキング（意思決定に本質的）が悪化する。
- **パラメータの肥大化 / 効率低下**: 処置ごとに独立した予測ヘッド（あるいはタワー）を持つと、K に比例してパラメータが増え、学習・推論効率が落ちる。
- **表現の共有不足**: 処置間で共通するユーザー特性の表現学習が、独立構造では十分に活用されない。

## Proposed Method（提案手法）

### コアアイデア

「共有された表現（MMoE）」の上に「アップリフトを明示的に足し込む再パラメータ化ヘッド」を載せる。反応を直接予測するのではなく、対照反応 + アップリフトの加法分解でモデル化することで、アップリフト自体を学習ターゲットに近づける。

### 手順（numbered steps）

1. **入力特徴 x** を N 個の共有エキスパート fₙ(x)（n=1..N）に通す。
2. 各処置 k（k=0 は対照）について、ゲートネットワーク gₖ(x) = softmax(Wgₖ x) がエキスパート出力を重み付け合成し、処置別表現 φₖ(x) = Σₙ gₖ(x)fₙ(x) を得る（**Multi-gate MoE**）。
3. **対照ヘッド h₀** が φ₀ からベースライン反応 μ₀(x) = h₀(φ₀) を予測する。
4. **各処置ヘッド hₖ** は φₖ から反応ではなく **アップリフト** τ̂ᵏ(x) = hₖ(φₖ) を直接出力する。
5. 処置反応を **再パラメータ化** して構成: μₖ(x) = μ₀(x) + τ̂ᵏ(x)。
6. 観測された対照反応 y(0) と処置反応 y(k) に対して、μ₀ と μₖ を教師あり損失で同時最適化（正則化項付き）。
7. 推論時は τ̂ᵏ(x) がそのままユーザー×処置ごとのアップリフト（CATE 推定）となり、意思決定（どの処置を誰に割り当てるか）に利用する。

### Key Formulas

個体処置効果（ITE / uplift）:
$$
\tau_i^k = y_i(k) - y_i(0)
$$

条件付き平均処置効果（CATE）:
$$
\tau^k(x) = \mathbb{E}\big[y(k)\mid t=k, x\big] - \mathbb{E}\big[y(0)\mid t=0, x\big] = \mu_k(x) - \mu_0(x)
$$

Multi-gate Mixture-of-Experts による特徴表現:
$$
\phi_k(x) = \sum_{n=1}^{N} g_k(x)\, f_n(x), \quad \forall k \in \{0, 1, \dots, K\}
$$

ゲートネットワーク:
$$
g_k(x) = \mathrm{softmax}(W_{g_k}\, x)
$$

Reparameterization module（アップリフトの明示的モデル化と加法分解）:
$$
\mu_0(x) = h_0(\phi_0), \qquad
\hat{\tau}^k(x) = h_k(\phi_k), \qquad
\mu_k(x) = \mu_0(x) + \hat{\tau}^k(x), \quad \forall k \in \{1, \dots, K\}
$$

目的関数（対照反応 + 全処置反応の教師あり損失 + L2 正則化）:
$$
\min_{\Theta}\ \mathcal{L}_{\text{M}^3\text{TN}}
= \mathcal{L}\big(\mu_0(x), y(0)\big)
+ \sum_{k=1}^{K} \mathcal{L}\big(\mu_k(x), y(k)\big)
+ \lambda \lVert \Theta \rVert_2
$$

## Algorithm（擬似コード）

```
入力: 学習データ {(x_i, t_i, y_i)}、処置数 K、エキスパート数 N、正則化係数 λ
初期化: エキスパート {f_n}、ゲート {g_k}、対照ヘッド h_0、処置ヘッド {h_k}

for エポック = 1..E do
  for ミニバッチ B do
    # 1. 共有エキスパート表現
    for n = 1..N: E_n = f_n(x)                       # x はバッチ内特徴

    # 2. 処置別のゲート合成表現
    for k = 0..K:
        g_k = softmax(W_{g_k} x)
        phi_k = sum_n g_k[n] * E_n

    # 3. 対照反応
    mu_0 = h_0(phi_0)

    # 4. 各処置のアップリフトと再パラメータ化反応
    for k = 1..K:
        tau_hat_k = h_k(phi_k)                        # アップリフトを直接出力
        mu_k = mu_0 + tau_hat_k                       # 加法分解で反応を構成

    # 5. 観測処置に対応する反応で損失計算
    loss = L(mu_0, y(0)) + sum_{k=1..K} L(mu_k, y(k)) + λ * ||Θ||_2

    # 6. 勾配更新
    Θ <- Θ - η * ∇Θ loss
  end
end

推論: 各ユーザー x と処置 k に対し tau_hat_k(x) をアップリフト推定として返し、
      argmax_k または閾値ルールで処置割当を決定する
```

## Architecture / Process Flow

```
                     入力特徴 x
                         │
        ┌────────────────┼────────────────┐
        ▼                ▼                ▼
   Expert f_1        Expert f_2  ...  Expert f_N     ← 全処置で共有
        └────────────────┼────────────────┘
                         │  (各エキスパート出力)
     ┌───────────┬───────┴───────┬───────────────┐
     ▼           ▼               ▼               ▼
  gate g_0    gate g_1        gate g_2   ...   gate g_K   ← Multi-gate
  (softmax)   (softmax)       (softmax)        (softmax)
     │           │               │               │
   φ_0(x)      φ_1(x)          φ_2(x)          φ_K(x)
     │           │               │               │
   h_0 ヘッド   h_1 ヘッド      h_2 ヘッド       h_K ヘッド
     │           │(uplift)       │(uplift)        │(uplift)
   μ_0(x)     τ̂¹(x)           τ̂²(x)          τ̂ᴷ(x)
     │           │               │               │
     │    ┌──────┘        ┌──────┘        ┌──────┘
     └────┤ +             ┤ +             ┤ +          ← Reparameterization
          ▼               ▼               ▼
        μ_1(x)          μ_2(x)          μ_K(x)
     = μ_0 + τ̂¹      = μ_0 + τ̂²      = μ_0 + τ̂ᴷ
          │               │               │
          └───────── 教師あり損失 L(μ_k, y(k)) ────────┘
```

- ポイント1: エキスパートは全処置で共有、ゲートのみ処置別 → パラメータ効率と処置間の知識共有を両立。
- ポイント2: 処置ヘッドは反応でなくアップリフトを出力し、μ_k = μ_0 + τ̂ᵏ で再構成 → アップリフトを直接学習し累積誤差を抑制。

## Experiments & Evaluation

### Setup（設定）

- **データセット**:
  - **MultiTwins**: 多値処置に対応した公開データセット（既存研究由来）。
  - **Production Dataset**: 大手中国ショート動画プラットフォームの実運用データ。ユーザー 700 万人超、特徴量 108、処置は動画のシャープニング3段階（T=1,2,3）+ 対照（T=0）、反応指標は1日あたり総視聴時間。
  - 注記: プロンプトで想定された **CRITEO データセットは本論文では使用されていない**（記載なし）。
- **評価指標**:
  - **mQini**: 全処置にわたる Qini 係数の平均（↑ 高いほど良い）。
  - **sdQini**: Qini 係数の標準偏差（↓ 低いほど良い、処置間の安定性）。
  - **mKendall**: アップリフトのランク相関（Kendall）の平均（↑）。
  - **sdKendall**: その標準偏差（↓）。
- **比較手法**: S-Learner, T-Learner, TARNet, CFRNet_MMD, CFRNet_WASS, MEMENTO, Hydranet。

### Main Results（主要結果、数値）

- MultiTwins: M³TN が mQini **0.8733**（全手法中1位）、sdQini **0.1297**（最小=最も安定）。mKendall 0.8077（Hydranet 0.8277 に次ぐ2位）。
- Production: mKendall **0.6906**（1位）、mQini 0.6804（CFRNet_MMD 0.6823 に次ぐ2位）。ランキング整合性（Kendall）で優位。
- 総じて、間接差分に依存する既存手法に対し、アップリフト直接モデル化により **処置間の安定性（低 sd）とランキング精度** で優位を示した。

### Ablation（アブレーション）

- 表2の通り、**RM（reparameterization module）の寄与が最大**。これを外すと Production mQini が 0.6804 → 0.6107 に低下し、著者らの主張（明示的アップリフトモデル化の重要性）を裏付ける。
- MMoE も性能に寄与（外すと 0.6804 → 0.6408）するが、主に効率（パラメータ削減）と表現共有の面での貢献。

## 本テーマへの適用可能性

想定シナリオ: データサイエンティストが **頻度の低いマーケティング施策**（クーポン配布、メール送信など）を運用しており、施策ごとにユーザー単位のアップリフト / CATE を推定したいが、各施策のデータがスパース（サンプル数が少ない・不定期）という状況。M³TN の構造は、この「スパースな複数施策間でアップリフト推定をプール / 共有する **ベース推定器**」として直接的に有用である。

- **多値処置 = 複数施策の統一モデル化**: M³TN は K 個の処置を1つのネットワークで扱う。クーポン種別・メール文面・割引率などの各施策を「処置 k」に対応させれば、施策ごとに別モデルを組む代わりに、1つの共有モデルで全施策のアップリフトを同時推定できる。施策ごとにデータが少なくても、共通のエキスパート層で表現を学習するためデータ効率が高い。
- **MMoE による表現プーリング（共有）**: 全処置で共有される N 個のエキスパートが、施策横断で共通するユーザー特性（購買傾向・エンゲージメント等）の表現を学習する。処置別のゲートは、その共有表現を各施策に合わせて重み付けするだけなので、スパースな施策でも「他施策から借りた」表現を活用でき、少数サンプルでの過学習を抑えつつ per-user uplift を安定推定できる。これがまさに **プール / シェアリング** の要件に合致する。
- **加法分解 μ_k = μ_0 + τ̂ᵏ によるベース推定器化**: 対照反応 μ_0（施策を打たないときの基準アウトカム）を全施策で共有ヘッドとして学習し、各施策ヘッドは「そのベースからの増分（アップリフト）」だけを学ぶ。これは「共通のベースライン + 施策固有の増分」という、複数キャンペーンをプールするうえで自然な分解。新しい施策を追加する際も、ベース μ_0 を再学習せずアップリフトヘッド τ̂ᵏ を1つ足すだけで拡張でき、スパース施策の逐次追加に強い。
- **少数サンプルでの累積誤差回避**: スパースな施策では反応の差分（間接推定）ほど誤差が乗りやすいが、M³TN はアップリフトを直接学習するため、少データ環境でのランキング精度（誰に打つべきか）が安定しやすい。実際、論文でも sdQini / sdKendall（処置間の分散）が最小である点は、施策数が多く・各施策がスパースな運用での安定性に直結する。
- **実装上の留意**: 本論文の指標は Qini / Kendall であり、AUUC は 記載なし。マーケ運用で AUUC を主指標とする場合は、同じアーキテクチャで評価指標のみ差し替えて検証する必要がある。また CRITEO 等の標準マーケデータでの検証は論文にないため、自組織データでの再現・チューニング（特にエキスパート数 N と正則化 λ）が前提となる。

まとめると、M³TN の「共有エキスパート（MMoE）」+「対照反応共有 + 施策別アップリフトヘッドの加法分解」という設計は、**スパースな複数マーケティング施策にわたってアップリフト推定をプール / 共有するベース推定器** の要件を、構造レベルで直接満たす。
