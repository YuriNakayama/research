# Metamon: 人間ログからのオフラインRL + Transformer で人間級ポケモンAI

## 概要

Metamon(Grigsby et al., RLC 2025)は、Pokémon Showdown の十数年分(2014-2024)の人間対戦ログを一人称視点に再構成し、**Transformer 系列モデルを大規模オフラインRL**で学習することで、明示的探索なしに相手へ適応し人間 competitive 級(アクティブ上位10%)に達した初の手法群である。PTCG とは別タイトル(対戦ポケモン本編)だが、**不完全情報・確率的・系列適応**という性質が PTCG AI Battle と極めて近く、かつ「合法手index を返す」「観測は相手のアクティブのみ」など本コンペのインターフェース制約とも構造が一致する、最重要の直接参照点である。

## 手法の要点

### 一人称軌跡への再構成
観戦者(三人称)視点で保存されたログから、**エージェント一人称視点の軌跡**を復元するパイプラインを構築。観測は「相手のアクティブポケモンのみ」を含み、相手のチーム全体はモデルの**記憶(メモリ)で推論**する。部分観測性の強い旧世代(Gen 1-4 OverUsed)を主対象。

### 観測・行動表現
- **観測**: 87個のテキストトークン + 48個の数値特徴。利用可能な技・交代先を一定順序で符号化。
- **行動空間**: 9個の離散行動(技4 + 交代5)。PTCG の「合法手リスト index」と同型。

### オフラインRLアルゴリズムとモデル
- **Causal Transformer** が (観測, 行動, 報酬) のタプル列を処理し、探索なしで文脈から相手へ適応。
- actor-critic 系の損失で、BC項に価値ベース更新を加重(IL: w=1,λ=0 / 指数アドバンテージ重み付け / Binary+MaxQ が最良)。
- **モデル規模**: 15M / 50M / 200M パラメータ。**データ**: 再構成 ~95万軌跡(47.5万人間試合)、自己対戦で500万〜2000万超に拡張。
- **報酬**: 二値の勝敗が主、ダメージ・回復の軽い shaping。割引 \(\gamma=0.999\)(長ホライズン)。

### 学習の段階的進行
模倣学習(BC) → 人間譜面でのオフラインRL → 多様なチームで生成した自己対戦データでの微調整(SynRL V0→V2)。

## 主要な結果や知見

- **人間級到達**: 最良の SynRL-V2 は世代1-4で **GXE 64-80%**、アクティブ上位10%、Gen1OU 世界31位。人間トッププレイヤーの GXE は 74-90%。
- **探索なしで適応**: tree search を一切使わず、系列モデルの文脈学習だけで相手の戦略に適応。LLMエージェント(PokéLLMon)を上回り、強ヒューリスティック(Foul Play)を同等以上に。
- **スケーリング**: モデル/データを大きくするほど性能向上。オフラインRL+自己対戦の組合せが、人間ログだけ・自己対戦だけより強い。
- **チーム多様性が課題**: 論文の Gen 1-4 性能を多様なチーム(gen9ou)へ移すには、モデル縮小・reward shaping・長期記憶設計の反復が必要だった(VGC-Bench の知見と整合)。

## 本コンペ(PTCG AI Battle)への応用

1. **インターフェースがほぼ同型**: PTCG も「ログ+盤面+合法手リスト → index を返す」。Metamon の「テキストトークン+数値特徴の観測」「離散合法手から選択」「観測は相手アクティブのみ・残りは記憶で推論」という設計は、PTCG の観測・行動表現にほぼそのまま転用できる。
2. **探索なしで10分制約をクリア**: Metamon は明示的探索ゼロで人間級。PTCG の「1プレイヤー10分」では、毎手 MCTS は予算超過しがちだが、Transformer/系列方策の**単一 forward 推論**なら高速で、文脈(これまでのログ)から相手のデッキ・手札を推論しつつ即応できる。
3. **オフラインRLで人間ログを活用**: PTCG にトッププレイヤーの対戦ログがあれば、Metamon のパイプライン(三人称ログ→一人称軌跡→BC→オフラインRL→自己対戦微調整)を踏襲し、探索なしで強い初期方策を低コストに構築できる。自前の自己対戦が高コストでも、人間ログのオフライン学習で底上げできる。
4. **記憶で隠れ情報を補う**: PTCG は相手の手札・山札・サイドが隠れる。Metamon が相手チームを記憶で推論したのと同様、**Causal Transformer の系列記憶**で、これまでの公開情報から相手の残りデッキ内容・手札確率を暗黙にトラッキングさせるのが有効。
5. **デッキ多様性への汎化**: Metamon が単一ルールから多様チームへ拡張する際に直面した課題(縮小・shaping・記憶設計)は、PTCG の「deck.csv を変えても通用する方策」を目指す上で先行知見。多様なデッキで自己対戦データを生成し、デッキで条件付けた方策を学ぶ。
6. **報酬整形の指針**: 二値勝敗+軽いダメージ/回復 shaping、\(\gamma\) を1近傍に保つ Metamon のレシピは、PTCG の長いエピソード(サイド6枚取り切り)でも安定学習の出発点になる。
