## 一言で言うと

MIPS は「良い行動埋め込みが実務者によって定義できる」と仮定するが現実には困難、という問題意識から、**報酬モデルの中間出力をログデータから学習して行動埋め込みとして使う**手法。**本課題の技術的ギャップの中心にある論文であり、結論から言えば新規行動については何も答えを持たない**。

## 問題設定

論文の出発点は MIPS の前提への疑義である。

> "MIPS assumes that good action embeddings can be defined by the practitioner, which is difficult to do in many real-world applications. In this work, we explore learning action embeddings from logged data."

行動数が大きい、あるいは一部の行動がログ方策に十分探索されていない場合、IPS ベースの推定量は高い（あるいは無限の）分散を持つ。MIPS はこれを行動埋め込みで緩和するが、**その埋め込みをどこから調達するのかという問いに答えていない**。本論文はその欠落を埋めにいく。

## 手法

### 埋め込みの学習方法

驚くほど単純である。線形モデルを報酬予測で訓練し、その中間表現を埋め込みとして取り出す。

> "The model takes the action identity $a$, and computes a reward estimate $\hat{r}$ as a dot product between the action embedding and the context vector $x$. We train the model to minimize the MSE between $\hat{r}$ and the reward $r$."

すなわち、行動 $a$ の埋め込みを $\phi(a) \in \mathbb{R}^{d_e}$、文脈ベクトルを $x$ として

$$\hat{r}(x,a) = \phi(a)^{\top} x$$

とし、学習目的は

$$\min_{\phi} \frac{1}{n}\sum_{i=1}^{n}\left(\phi(a_i)^{\top} x_i - r_i\right)^2$$

である。埋め込みは**線形層の出力**（"the output of the linear layer"）として取り出される。著者らは任意のモデルクラスで代替可能だと述べつつ、より複雑なモデルでは有意な改善が得られなかったと報告している。

### 提案されたバリアント

| 手法 | 内容 |
|------|------|
| Learned MIPS OneHot | 行動 ID の one-hot 符号化のみを入力として埋め込みを学習 |
| Learned MIPS FineTune | 事前定義された行動埋め込みを起点に微調整する |
| Learned MIPS Combined | one-hot と事前定義埋め込みを連結して用いる |

### 行動特徴量の要件 — ここが本課題の核心

**この論文の埋め込みへの「要件」は、実質的に「行動 ID がログに存在すること」である**。$\phi$ は行動 ID から埋め込みへの**ルックアップ表**として学習される。したがって $\phi(a_{\text{new}})$ は、$a_{\text{new}}$ がログに現れない限り**定義すらされない**。これは実装上の制約ではなく定式化上の帰結である。

さらに重要な点として、論文は MIPS の **no direct effect 仮定（$a \perp r \mid x, e$）を引き継ぎながら、学習された埋め込みがそれを満たすかを検証していない**。本調査の範囲では、著者らは学習埋め込みを MIPS の枠組みにそのまま適用しており、この理論的要件への言及・検証は見当たらなかった。

なお **FineTune / Combined は事前定義埋め込みを部分的に利用する**点で、新規行動への橋渡しの萌芽を含む。事前定義埋め込みがメタデータ由来なら新規行動にも計算できるからである。**ただし論文はこれを新規行動の文脈では一切論じておらず、あくまで既存行動の埋め込み品質を上げる手段として提示している**。後述の workaround の出発点になりうるが、論文の主張ではない。

## 実験・結果

| 項目 | 合成データ | 実データ |
|------|-----------|---------|
| 行動数 | 10 〜 2,000 | $\|\mathcal{A}\| = 240$（ファッション推薦データセット） |
| 文脈次元 | $d_X = 5 \sim 100$ | 未確認 |
| 事前定義埋め込み次元 | 3 〜 20 | 4 次元（各ファッションアイテムに付与） |
| サンプル数 | 未確認 | $n = 10{,}000$ をサブサンプル |
| 報酬関数 | 線形 / ニューラルネット | 未確認 |

**主要な結果**: Learned MIPS は IPS, DM, DR といった標準的なベースラインを上回る。実データでは "Learned MIPS methods outperform IPS in about 75% of runs, more often than any other baseline"（約 75% の run で IPS を上回り、これは他のどのベースラインよりも高頻度）。

※ 各手法の MSE 絶対値、バリアント間の詳細比較は未確認。

## 本課題への適用可能性

### 効く点

- **「施策の埋め込みを手で設計するのは難しい」という問題意識が本課題そのもの**。クーポン施策の何を特徴量にすべきか（額・訴求文言・チャネル・配信時刻…）を人手で決める作業の恣意性に対する、原理的な代替案を提示している。
- **手法が単純で実装コストが低い**。線形モデルの MSE 最小化であり、公式実装も公開されている。手元のキャンペーンログで試すハードルは 4 本中最も低い。
- **既存施策の埋め込み品質を上げる用途では確実に使える**。過去に複数回実施した施策（定番のクーポン施策）については、ログから学習した埋め込みが人手設計の特徴量より報酬予測に有用な表現を与える可能性は十分ある。
- **FineTune / Combined バリアントの存在**が、メタデータ由来の事前定義埋め込みと学習埋め込みを**同じモデルの中で併用する**具体的な形を示している。後述の workaround の技術的出発点として最も近い場所にある。
- 実データが $|\mathcal{A}|=240$, $n=10{,}000$ と、本論文群の中では比較的小規模。本課題との規模ギャップが 4 本中では相対的に小さい。

### 効かない/リスク点

- **本課題の目的である新規行動には、原理的に使えない**。これが最重要の結論である。本調査の範囲で、論文は新規行動・コールドスタート・ログに存在しない行動への汎化を**一切扱っていない**。全実験は「行動が訓練データに存在し、その埋め込みが履歴から学習される」ことを前提とする。gather 段階の指摘（論点 3）は、**論文の実物を読んで確認された**。
- **これは論文の欠陥ではなく定義上の帰結である**。$\phi$ が行動 ID のルックアップ表である以上、未観測の ID に値を返せない。したがって「Learned MIPS を改良すれば新規行動も扱える」という期待は筋が悪い。必要なのは**行動 ID ではなくメタデータを入力とする $\phi$** であり、それは本論文の手法ではない。
- **規模感のギャップは依然として存在する**。行動数 10〜2,000 の実験だが、埋め込み学習の利得は行動数が多く各行動のログが薄い領域で生じる設計になっている。施策数が数十個なら、報酬モデルは施策 ID のダミー変数を入れた単純な回帰で足り、埋め込み学習という間接的な経路を取る意味が薄い。**施策 20 個・各施策に数万配信という本課題の形状では、施策ごとの効果を直接推定する方が素直**である。
- **no direct effect を学習埋め込みが満たす保証がないことを、論文自身が検証していない**。報酬予測 MSE を最小化して得た埋め込みは、報酬に効く情報を埋め込みに集める方向には働くので直感的には整合するが、**論文はこれを主張も証明もしていない**。理論的裏づけを求めるなら CAEL-MIPS（gather 一覧 06、MSE 最小化を明示的な目的にする）を見るべき。
- 実データ実験が「約 75% の run で IPS を上回る」という表現である点は、裏を返せば**約 25% の run では IPS に負けている**ということでもある。改善は確実だが劇的ではない。

## 実装ステップ

1. **本課題では、本論文を「新規施策の解」として実装してはならない**。適用範囲を既存施策に限定することを最初に明確化する。
2. **既存施策のみを対象に、Learned MIPS OneHot を公式実装で回す**。過去に複数回実施した定番施策の埋め込みが、人手設計のメタデータ特徴量より報酬を説明するかを比較する。これが本論文から得られる正味の価値。
3. **メタデータ特徴量をベースラインとして必ず併走させる**。もし人手設計のメタデータ特徴量で報酬予測性能が同等なら、**学習埋め込みを導入する理由がなくなる**（新規施策にも使えるメタデータ特徴量の方が上位互換になる）。施策数の少ない本課題では、この結果になる可能性が相応にある。**この比較こそが本論文を読む最大の実務的意義**である。
4. **Combined / FineTune バリアントをハイブリッドの出発点として検討する**。メタデータ由来の事前定義埋め込みを $\phi_{\text{meta}}(a)$ とし、既存施策にのみ学習される残差 $\Delta(a)$ を加えて $\phi(a) = \phi_{\text{meta}}(a) + \Delta(a)$ とする構成なら、新規施策では $\Delta = 0$ として $\phi_{\text{meta}}$ のみにフォールバックできる。**これは論文の主張ではなく本調査による設計提案**であり、妥当性は未検証（index.md の議論を参照）。
5. 学習埋め込みを採用する場合、no direct effect の点検を自前で行う。埋め込みに入れなかった施策属性が報酬に効いていないかを、残差分析で確認する。
