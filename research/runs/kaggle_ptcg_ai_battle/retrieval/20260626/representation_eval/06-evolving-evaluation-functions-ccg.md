# 収集型カードゲーム AI のための評価関数の進化的学習

## 概要

Miernik と Kowalski による「Evolving Evaluation Functions for Collectible Card Game AI」(arXiv:2105.01115, 2021) は、収集型カードゲーム(CCG)の**盤面評価関数(state evaluation function)を進化計算で学習** する研究である。題材は AI 研究用に設計された小規模 CCG **Legends of Code and Magic(LoCM)**(160枚, 効果決定論的, 不確実性はカード順と相手デッキの未知性のみ)。本論文は (1) 評価関数の **遺伝子表現(線形ベクトル vs 木構造)** の選択、(2) **対戦相手の選び方(自己対戦 vs 固定相手)** という2つの設計判断が、進化結果と最終的なエージェント強度に与える影響を体系的に分析する。PTCG における「盤面評価関数の設計」「評価関数の進化的最適化」のクラスタ中核トピックに対応する。

## 手法の要点

- **評価関数の3表現**:
  1. **Linear**: 20 個の重み(遺伝子)による特徴量の線形結合。最初の 12 はゲーム状態特徴(両プレイヤーの現在/最大マナ・デッキサイズ・ヘルス・次ターンドロー枚数・追加ドロー指標)、残り 8 はカード特徴(攻撃・防御・各キーワードフラグ)。`evalState`(盤面)+ Σ`evalCard`(自カード) − Σ`evalCard`(相手カード)で最終評価。
  2. **BinaryStree**: 状態評価とカード評価それぞれを二分木で表現。葉は定数または特徴量、節は加算(l+r)・乗算(l*r)・減算(l−r)・最大(max)・最小(min)。
  3. **Tree**: n 分木版。演算子はベクトルを取る加算(Σ)・乗算(∏)・max・min に加え、減算用の単項否定(−x)を持つ。木表現は線形表現を包含する(Linear→Tree への写像は可能だが逆は不可)。
- **進化アルゴリズム**: 標準的な遺伝的アルゴリズム。population=50、50世代、エリート保存=5、突然変異率=5%。評価では各個体ペアが drafts=10 ドラフト×rounds=10 ラウンド対戦。
- **対戦相手スキーム(fitness)の3種**: progressive(集団内自己対戦)、weak-op(弱い固定相手 Baseline2/WeakOp)、strong-op(強い既進化 Tree エージェント StrongOp)。
- **使用法**: 進化した評価関数を貪欲なエージェントの盤面評価ヒューリスティックとして使用(alpha-beta, RHEA, MCTS 等への組込みも想定)。

## 主要な結果や知見

- **自己改善(progressive)が固定相手より優れる**: 自己対戦で進化したエージェントの方が、固定相手に向けて進化したものより強い傾向。固定弱相手(weak-op)での進化は「忘却」が起きやすく汎化しない。
- **表現の差(weak-op トーナメント)**: Linear 平均勝率 **42.9%** vs BinaryTree **33.8%**、Tree **36.2%**。弱相手進化では Linear が最良。
- **表現の差(strong-op トーナメント)**: Linear **48.9%**、BinaryTree 45.1%、Tree 44.8% と差が縮小(約4%)。
- **自己改善の進捗(自世代対戦勝率差)**: Linear は初世代→最終世代で平均 **31% 超**の改善、Tree/BinaryTree は約 20%。Linear は早く良解に到達するが頭打ち。
- **二段階アプローチが有効**: Linear で得た良解を Tree 表現に変換して再進化する `from-Linear` 実験では、Tree-from-Linear が平均勝率 **60.2%**、Linear-from-Linear が 58.0% を達成。より表現力の高い木がブートストラップ後さらに改善。
- **結論**: 計算予算が限られるなら単純な線形表現が信頼性高く速く良解を出す。大きな予算があれば、線形解を木に変換してから進化を続ける二段階法を推奨。

## 本コンペ(PTCG AI Battle)への応用

- **盤面評価関数の設計テンプレート**: PTCG の盤面評価を「ゲーム状態特徴(自/相手のサイド残数・ベンチ数・手札数・エネルギー・HP)+カード特徴」の線形結合または木構造で表す本論文の設計は、そのまま流用できる。`evalState − Σ evalCard(相手)` の対称的構成は、PTCG の自分/相手盤面の差分評価に自然に対応する。
- **限られた計算予算下での選択**: 「予算が小さければ Linear、大きければ木への二段階」という知見は、PTCG 開発で評価関数を実装する際の現実的な指針。まず軽量な線形評価関数を進化させ、強くなったら表現力を上げる戦略が有効。
- **自己対戦による進化が頑健**: 固定相手より自己対戦(progressive)が忘却を避け汎化するという結果は、PTCG の Gaussian レーティングラダー(多様な相手と対戦)で安定した強さを得る上で重要。固定の弱い bot に過適合する評価関数を避けるべき。
- **10分・時間切れへの適合**: 進化で得た評価関数は推論時に軽量(線形なら内積1回)で、合法手 index ごとに局面を素早く評価できる。MCTS/探索の枝刈りヒューリスティックとして用いれば、10分制約内で深い読みを支えられる。
- **デッキ60枚最適化との接続**: 本論文ではドラフトフェーズのカード選択も同じ `evalCard` を流用しており、PTCG のデッキ構築フェーズとプレイフェーズで評価関数を共有する設計のヒントになる。
- **Strategy 部門への寄与**: 進化的に学習した解釈可能な評価関数(重みや木)は、技術レポートで戦略を説明しやすく、Strategy 部門の評価にも適合する。
