# Agentic Retrieval-Augmented Generation: A Survey on Agentic RAG

- **Link**: https://arxiv.org/abs/2501.09136
- **Authors**: Aditi Singh, Abul Ehtesham, Saket Kumar, Tala Talaei Khoei, Athanasios V. Vasilakos
- **Year**: 2025
- **Venue**: arXiv preprint (cs.AI, cs.CL, cs.IR)
- **Type**: Academic Paper (Survey)

## Abstract

Large Language Models have revolutionized AI, but their reliance on static training data limits their ability to provide accurate and up-to-date responses. Retrieval-Augmented Generation (RAG) has emerged as a solution, integrating real-time data retrieval to provide contextually relevant and up-to-date responses. However, traditional RAG systems lack adaptability. This paper presents Agentic RAG, which embeds autonomous AI agents using reflection, planning, tool use, and multi-agent collaboration to improve retrieval strategies. The survey provides a taxonomy of these systems, examines applications in healthcare, finance, education, and document processing, and identifies open research challenges in evaluation, coordination, memory management, efficiency, and governance.

## Abstract（日本語訳）

大規模言語モデルはAIに革命をもたらしたが、静的な訓練データへの依存が正確かつ最新の応答を提供する能力を制限している。検索拡張生成（RAG）はリアルタイムデータ検索を統合することでこの問題の解決策として登場した。しかし、従来のRAGシステムは適応性に欠ける。本論文は、省察、計画、ツール使用、マルチエージェント協調を用いる自律的AIエージェントを組み込んだAgentic RAGを提示する。本サーベイは、これらのシステムのタクソノミーを提供し、医療、金融、教育、文書処理における応用を検討し、評価、協調、メモリ管理、効率性、ガバナンスにおける未解決の研究課題を特定する。

## 概要

本論文は、検索拡張生成（RAG）の進化の最前線である「Agentic RAG」を体系的に分類・分析した初の包括的サーベイである。従来のNaive RAGからAdvanced RAG、Modular RAG、Graph RAGを経てAgentic RAGに至る5つのパラダイムの進化を整理し、自律的エージェントの組み込みによって検索・生成パイプラインがどう変革されるかを詳述している。

主要な貢献：

1. **RAG進化の5パラダイム体系化**: Naive → Advanced → Modular → Graph → Agentic RAGの段階的発展を整理
2. **7種のAgentic RAGアーキテクチャのタクソノミー**: シングルエージェントルーターから Agentic Document Workflowsまでの分類
3. **4つのエージェンティック・インテリジェンスの設計パターン**: 省察、計画、ツール使用、マルチエージェント協調
4. **5つのワークフローパターンの体系化**: プロンプトチェイニング、ルーティング、並列化、オーケストレータ・ワーカー、評価者・最適化者
5. **従来RAG vs Agentic RAG vs ADWの比較分析**: 3つのパラダイムの体系的比較
6. **7つの実践的教訓の提示**: システム設計者向けのガイダンス

## 問題と動機

- **静的訓練データの限界**: LLMは訓練データのカットオフ以降の情報を持たず、最新の事実に基づく応答ができない。RAGはこの問題を部分的に解決するが、従来のRAGには適応性の限界がある。

- **従来RAGの3つの制約**:
  1. **文脈統合の困難**: 静的パイプラインでは複数ソースからの情報を効果的に統合できない
  2. **多段階推論の欠如**: 中間的知見に基づく反復的精製ができない
  3. **スケーラビリティの問題**: 外部データ量の増加に伴うレイテンシ増大

- **エージェント的適応性の必要性**: クエリの複雑さに応じた動的な検索戦略の選択、誤りの自己検出・修正、複数専門エージェントの協調が、実世界応用の要件を満たすために不可欠である。

## 分類フレームワーク / タクソノミー

### RAG進化の5パラダイム

| パラダイム | 特徴 | 検索手法 |
|:--------:|:---:|:------:|
| Naive RAG | キーワードベース、静的 | TF-IDF, BM25 |
| Advanced RAG | 密ベクトル検索、ニューラルリランキング | Dense Retrieval, Cross-Encoder |
| Modular RAG | ハイブリッド戦略、コンポーザブル | 複合パイプライン |
| Graph RAG | グラフ構造、関係推論 | Knowledge Graph + Retrieval |
| Agentic RAG | 自律的意思決定、反復精製 | エージェント駆動動的検索 |

### 4つのエージェンティック設計パターン

1. **省察（Reflection）**: 自己フィードバックによる出力の反復的精製。批評サイクルを通じた品質向上。
2. **計画（Planning）**: タスクを管理可能なサブタスクに分解し、動的にシーケンスを構成。
3. **ツール使用（Tool Use）**: 外部API、データベース、計算リソースとの統合。
4. **マルチエージェント協調（Multi-Agent Collaboration）**: 分散タスク専門化とエージェント間通信。

### 5つのワークフローパターン

1. **プロンプトチェイニング**: 前段の出力を次段の入力とする逐次的処理
2. **ルーティング**: 入力を分類し特化プロセスに振り分け
3. **並列化**: 独立サブタスクの同時実行または投票ベースの集約
4. **オーケストレータ・ワーカー**: 中央コーディネータが専門エージェントにタスクを動的割当
5. **評価者・最適化者**: フィードバック駆動の反復的精製サイクル

### 7種のAgentic RAGアーキテクチャ

**1. シングルエージェント・ルーター**
- 1つの集中型エージェントが全検索・ルーティングを管理
- 利点：シンプル性、リソース効率、動的ソース選択
- 用途：複数知識ソースを持つカスタマーサポート

**2. マルチエージェントシステム**
- SQL、セマンティック検索、Web、推薦など専門エージェントの協調
- 利点：モジュール性、並列処理、タスク専門化
- 課題：調整の複雑性、データ統合オーバーヘッド

**3. 階層型システム**
- 多層エージェントによる戦略的監視
- 利点：戦略的優先順位付け、複雑クエリへのスケーラビリティ
- 課題：層間のオーケストレーション複雑性

**4. Corrective RAG**
- 5つのエージェント：コンテキスト検索、関連性評価、クエリ精製、外部知識、応答合成
- 特徴：文書関連性の評価→クエリ精製→必要時の外部データ取得
- 利点：反復的修正、事実性の確保、幻覚リスクの最小化

**5. Adaptive RAG**
- 分類器がクエリ複雑性を評価し適切な戦略にルーティング
- 単純クエリ：検索バイパス。中程度：単一ステップ。複雑：多段階
- 利点：リソース効率、動的戦略選択

**6. グラフベースシステム**
- Agent-G：グラフ知識ベースと非構造化検索の結合、クリティックモジュール
- GeAR：グラフ拡張による基本リトリーバの強化
- 利点：多ホップ推論、関係性の捕捉

**7. Agentic Document Workflows (ADW)**
- パーシング、検索、推論、構造化出力のエンドツーエンドオーケストレーション
- 用途：請求書処理、契約審査、保険金請求分析

## 主要な知見と分析

### 1. Agentic RAGは常に最適解ではない

サーベイの重要な教訓の一つは、「Agentic RAGは万能ではない」という点である。シンプルなRAGがタスク要件を満たす場合、不必要にエージェント的アーキテクチャを導入するとオーバーヘッドが増大するだけである。アーキテクチャの選択はタスクの複雑性に基づくべきである。

### 2. アーキテクチャが行動を規定する

システム設計は性能特性を強く規定する。Corrective RAGは事実性確保に優れるが、Adaptive RAGはリソース効率に優れる。アーキテクチャの選択はユースケースの要件に直結する。

### 3. 検索品質がボトルネック

エージェントの洗練度に関わらず、文書の関連性が主要な制約要因として残る。検索品質の向上なくしてエージェント的推論の改善は限定的である。

### 4. 自律性には制約が必要

エージェントの独立性が高まるほど、明示的な運用ガードレールが不可欠となる。安全性、信頼性、ガバナンスのメカニズムなくして自律的システムの本番投入は危険である。

### 5. プロセスベース評価の必要性

最終出力のみならず、推論ステップの評価が必要である。エージェントが正しい答えに到達しても、推論プロセスが不適切であれば信頼性は担保されない。

### 6. ドメイン知識が効果を増幅

タスク固有の専門知識はエージェント的アプローチの利点を大幅に増強する。汎用的なエージェント設計よりも、ドメイン特化の知識統合が性能向上に寄与する。

## 将来の研究方向

1. **エージェント協調と創発的行動**: マルチエージェント相互作用と予期しないシステム行動の管理。意図しない協調パターンや競合の検出と制御。

2. **出力品質を超えた評価**: 推論プロセス、解釈可能性、ロバスト性を評価する手法の開発。エージェント固有の能力を捕捉する専門ベンチマークの設計。

3. **メモリ管理**: インタラクション全体にわたる長期適応と効率的コンテキスト保持。長期記憶アーキテクチャの発展。

4. **計算効率**: 性能を維持しながらリソースオーバーヘッドを削減。エージェント呼び出し回数と精度のトレードオフの最適化。

5. **安全性とガバナンス**: 自律システムが定義された境界内で動作することの保証。責任あるデプロイメントのための安全性フレームワークの構築。

6. **クロスドメイン一般化**: 多様なタスクとドメインにわたるシステムの適応。ドメイン非依存のアプローチの開発。

## 備考

- Agentic RAGの体系的タクソノミーを初めて提示した先駆的サーベイであり、エージェントカーディナリティ、制御構造、自律性、知識表現の4次元による分類は実践的な設計指針として有用
- v4（2026年4月改訂）まで更新が続いており、急速に発展する分野の最新動向を反映している
- 7つの実践的教訓は、「Agentic RAGは常に最適解ではない」という重要な指摘を含み、過度なアーキテクチャ複雑性への警鐘を鳴らしている
- ADW（Agentic Document Workflows）は企業文書処理への特化型として独自のカテゴリを形成しており、産業応用における実用性が高い
- Corrective RAGとAdaptive RAGの対比は、事実性重視 vs リソース効率というトレードオフの明確な理解を提供する
