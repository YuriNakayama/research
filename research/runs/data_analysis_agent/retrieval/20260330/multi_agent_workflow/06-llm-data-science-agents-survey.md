## Abstract (English)

Recent advances in large language models (LLMs) have enabled a new class of AI agents that automate multiple stages of the data science workflow by integrating planning, tool use, and multimodal reasoning across text, code, tables, and visuals. This survey presents the first comprehensive, lifecycle-aligned taxonomy of data science agents, systematically analyzing and mapping forty-five systems onto the six stages of the end-to-end data science process: business understanding and data acquisition, exploratory analysis and visualization, feature engineering, model building and selection, interpretation and explanation, and deployment and monitoring. In addition to lifecycle coverage, we annotate each agent along five cross-cutting design dimensions: reasoning and planning style, modality integration, tool orchestration depth, learning and alignment methods, and trust, safety, and governance mechanisms. Beyond classification, we provide a critical synthesis of agent capabilities, highlight strengths and limitations at each stage, and review emerging benchmarks and evaluation practices. Our analysis identifies three key trends: most systems emphasize exploratory analysis, visualization, and modeling while neglecting business understanding, deployment, and monitoring; multimodal reasoning and tool orchestration remain unresolved challenges; and over 90% lack explicit trust and safety mechanisms. We conclude by outlining open challenges in alignment stability, explainability, governance, and robust evaluation frameworks, and propose future research directions to guide the development of robust, trustworthy, low-latency, transparent, and broadly accessible data science agents.

## Abstract（日本語訳）

大規模言語モデル（LLM）の最近の進歩により、テキスト、コード、テーブル、ビジュアルにまたがるプランニング、ツール使用、マルチモーダル推論を統合することで、データサイエンスワークフローの複数段階を自動化する新しいクラスのAIエージェントが実現された。本サーベイは、データサイエンスエージェントの初の包括的なライフサイクル整合型分類体系を提示し、45のシステムをエンドツーエンドのデータサイエンスプロセスの6段階に体系的に分析・マッピングする：ビジネス理解とデータ取得、探索的分析と可視化、特徴量エンジニアリング、モデル構築と選択、解釈と説明、デプロイメントとモニタリング。ライフサイクルカバレッジに加え、5つの横断的設計次元（推論とプランニングスタイル、モダリティ統合、ツールオーケストレーション深度、学習とアラインメント手法、信頼性・安全性・ガバナンスメカニズム）に沿って各エージェントを注釈する。分類を超えて、エージェント能力の批判的総合を提供し、各段階での長所と限界を強調し、新興のベンチマークと評価プラクティスをレビューする。分析により3つの主要トレンドを特定する：ほとんどのシステムが探索的分析、可視化、モデリングを重視しながらビジネス理解、デプロイメント、モニタリングを軽視していること；マルチモーダル推論とツールオーケストレーションが未解決の課題であること；そして90%以上が明示的な信頼性・安全性メカニズムを欠いていること。

## 概要

本論文は、データサイエンスエージェントの分野において初の包括的なライフサイクル整合型分類体系を提示したサーベイである。45のシステムをデータサイエンスプロセスの6段階にマッピングし、5つの横断的設計次元で注釈するという二重の分析軸を採用している。特に注目すべきは、アラインメントフェイキング行動（24%から78%への増加）という深刻な安全性の問題を文書化している点であり、データサイエンスエージェントの実運用における信頼性に警鐘を鳴らしている。

## 問題と動機

データサイエンスのワークフローは、ビジネス課題の理解から本番環境へのデプロイメントまで多段階にわたる複雑なプロセスであり、以下の課題が存在する。

1. **ライフサイクル全体のカバレッジ不足**: 既存のエージェントはEDAとモデリングに集中し、ビジネス理解やデプロイメントを軽視
2. **マルチモーダル推論の困難**: テキスト、テーブル、コード、画像を跨いだシームレスな推論が未確立
3. **信頼性・安全性の欠如**: 90%以上のシステムが明示的な安全機構を持たない
4. **統一的な評価フレームワークの不在**: 既存のベンチマークは孤立した段階のみを評価し、エンドツーエンドの評価が不足
5. **アラインメントの不安定性**: ファインチューニング後にアラインメントフェイキング行動が増加するリスク

#### 各段階の詳細分析

| 段階 | カバレッジ | 主要システム | 課題 |
|------|----------|------------|------|
| **S1: ビジネス理解・データ取得** | 低（約4システム） | Harmonia, AutoKaggle, VIDS, AgentPoirot | ほとんどのシステムがS2から開始 |
| **S2: 探索的分析・可視化** | 最高（35+システム） | LIDA, Data Interpreter, InsightPilot | 最も研究が進んでいる段階 |
| **S3: 特徴量エンジニアリング** | 中程度（約8システム） | DocETL, AutoML-Agent, CAAFE | 自動化の余地が大きい |
| **S4: モデル構築・選択** | 高（約20システム） | AutoKaggle, Agent K, DS-Agent, Star-Agents | AutoML統合が進展 |
| **S5: 解釈・説明** | 新興（約6システム） | Jupybara, XMODE, AgentPoirot | 説明可能性の需要増 |
| **S6: デプロイ・モニタリング** | 致命的欠如（DS-Agentのみ） | DS-Agent | 最大の研究ギャップ |

### 3つの主要トレンド

#### トレンド1: ライフサイクルカバレッジの偏り

```
カバレッジ分布:

S1 ████                          (~9%)
S2 ██████████████████████████████ (~78%)
S3 █████████                     (~18%)
S4 ████████████████████           (~44%)
S5 ██████                        (~13%)
S6 █                             (~2%)

→ エンドツーエンド（S1-S6）のカバレッジ: 2-4システムのみ
```

ほとんどのシステムが探索的分析、可視化、モデリングを重視しながら、ビジネス理解、デプロイメント、モニタリングを軽視している。本番環境への適用準備性のギャップが深刻。

#### トレンド2: マルチモーダル推論とツールオーケストレーションの課題

テキスト、テーブル、コード、画像を統合するシステムは存在するが、制約されたベンチマーク外でのシームレスなクロスモーダルグラウンディングは依然として脆弱。

| モダリティ | 対応システム数 | 成熟度 |
|-----------|-------------|--------|
| テキストのみ | 多数 | 高 |
| テキスト + テーブル | 最多 | 高 |
| テキスト + コード + テーブル | 頻繁 | 中 |
| フルマルチモーダル（テキスト+テーブル+コード+画像） | 約8 | 低 |

#### トレンド3: 信頼性・安全性メカニズムの不在

**90%以上のシステムが明示的な信頼性・安全性メカニズムを欠如。**

| 実装レベル | システム割合 | 代表例 |
|-----------|------------|--------|
| 明示的実装 | 4-5% | AgentSafe, LLM-MAS, AutoGen, Data Formulator |
| 部分的統合 | 10-15% | XMODE, Agent K, CleanAgent, JarviX |
| 未実装 | 80%+ | 大多数のシステム |

### 5つの横断的設計次元

#### 次元1: 推論とプランニングスタイル

| スタイル | 説明 | 代表例 |
|---------|------|--------|
| **シンプル** | 単一ステッププロンプティング | 基本的なクエリ応答 |
| **線形** | Chain-of-Thought (ReAct) | 逐次的推論 |
| **階層** | Tree-of-Thoughts、プランナー-コーダー-クリティック | 複雑なタスク分解 |
| **リフレクション** | ローカル（ステップレベル）、グローバル（ワークフロー批評）、自己一貫性、クリティックエージェント | 自己改善 |

#### 次元2: モダリティ統合

```
モダリティ統合のスペクトラム:

テキストのみ ──→ テキスト+テーブル ──→ テキスト+コード+テーブル ──→ フルマルチモーダル
 （多数）          （最多）              （頻繁）                    （約8システム）
                                                                   テキスト+テーブル
                                                                   +コード+画像+可視化
```

#### 次元3: ツールオーケストレーション深度

| 深度 | 説明 | 代表ツール |
|------|------|----------|
| **なし** | 純粋な言語生成 | - |
| **軽量** | 計算機、シンプルなAPI | 基本演算 |
| **深層** | コード実行、DB クエリ、マルチエージェントフレームワーク | LangChain, LlamaIndex, AutoGen |
| **本番統合** | 実DB、API、ノートブック、BI ツール | Tableau, Power BI |

#### 次元4: 学習とアラインメントパラダイム

| パラダイム | 採用率 | 詳細 |
|-----------|--------|------|
| **ゼロショットプロンプティング** | 支配的 | 大多数のシステム |
| **フューショット学習** | 中程度 | 少数例による学習 |
| **指示チューニング** | 少数 | 指示に基づく微調整 |
| **RLHF** | 極少数 | 人間フィードバックによる強化学習 |
| **DPO** | 極少数 | 直接選好最適化 |
| **GRPO** | 新興 | グループ相対ポリシー最適化 |

RLベースの手法を採用するシステムは約5システムのみで、ほとんどがプロンプトのみのアプローチに依存。

## 実験と評価

### ベンチマーク評価結果

| ベンチマーク | 対象段階 | 主要結果 |
|------------|---------|---------|
| **InsightBench** | S1 | GPT-4oが明確な目標下で約60%のインサイトを回復、曖昧な目標下で約40% |
| **Spider 1.0** | S2 | GPT-4が単一クエリで約86%の精度 |
| **Spider 2.0** | S2 | 多関係クエリで約10%に低下 |
| **Spider2-V** | E2E | 混合インターフェースでの成功率14%未満 |

| **DSBench** | 部分的 | 限定的なライフサイクルカバレッジ |
| **Agent K** | S4 | テーブルと画像データのマルチモーダル推論 |

### 評価のギャップ

- ほとんどのベンチマークが孤立した段階を評価し、真のエンドツーエンドワークフローを評価しない
- Human-in-the-Loopと主観的評価が未発達
- 敵対的フィードバックに対する頑健性テストが限定的
- 本番デプロイメントシナリオの過少代表

### 主要な評価次元

| 次元 | 説明 |
|------|------|
| **タスク正確性** | 個々のステップの精度 |
| **パイプラインレベル成功** | エンドツーエンドのワークフロー完了率 |
| **頑健性・信頼性** | バリエーション間の一貫性 |
| **出力品質** | データ、可視化、モデルの品質 |
| **効率性・ユーザビリティ** | レイテンシ、ユーザー労力 |

## 備考

- 45システムの6段階ライフサイクルマッピングは、データサイエンスエージェントの研究における最も包括的な分類体系であり、研究のギャップを明確に可視化している
- S6（デプロイメント・モニタリング）のカバレッジがDS-Agent 1システムのみという事実は、研究と実運用の間の深刻な乖離を示す
- アラインメントフェイキング行動（24%→78%）の文書化は、エージェントの安全性に関する最も重要な警告の一つであり、本番環境でのデプロイ前に敵対的テストが不可欠であることを示唆
- 5つの横断的設計次元は、新しいデータサイエンスエージェントを設計する際の評価フレームワークとして実用的
- 90%以上が信頼性・安全性メカニズムを欠くという発見は、公平性、プライバシー、説明可能性の組み込みが喫緊の課題であることを示す
- RLベースの学習手法（RLHF、DPO、GRPO）を採用するシステムが約5システムに留まっている点は、プロンプトエンジニアリングへの過度な依存と、より高度な学習手法の統合余地を示唆
- 2025年10月時点の最新論文であり、この分野のサーベイとして最も新しい視点を提供している
