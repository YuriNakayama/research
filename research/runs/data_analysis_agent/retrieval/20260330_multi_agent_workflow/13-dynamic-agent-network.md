## Abstract (原文)

DyLAN is a framework for LLM-based multi-agent collaboration that automatically selects a team of agents from candidates to collaborate in a dynamic communication structure toward different tasks and domains. The framework operates in two stages: Team Optimization uses an Agent Importance Score metric for agent selection, and Task Solving employs dynamic collaboration with agent team reformation and early stopping. DyLAN demonstrates improvements across code generation, decision-making, reasoning, and arithmetic tasks, with accuracy gains up to 25.0% on specific MMLU subjects.

## Abstract (日本語)

DyLANは、LLMベースのマルチエージェント協調のためのフレームワークであり、候補からエージェントチームを自動選択し、動的な通信構造で協調してタスクを解決する。フレームワークは2段階で動作する：チーム最適化段階ではAgent Importance Scoreメトリクスによるエージェント選択を行い、タスク解決段階ではエージェントチーム再編と早期停止を伴う動的協調を実行する。DyLANはコード生成、意思決定、推論、算術タスクにおいて改善を示し、特定のMMLU科目では最大25.0%の精度向上を達成した。

## 1. 概要

DyLAN（Dynamic LLM-powered Agent Network）は、マルチエージェント協調を多層フィードフォワードネットワークとして抽象化し、動的なチーム構成と通信構造を実現するフレームワークである。中核となる技術要素は以下の通り：

1. **T-FFN（Temporal Feed-Forward Network）抽象化**: エージェント間協調を時間的な多層ネットワークとして定式化
2. **Agent Importance Score**: 各エージェントの貢献度を教師なしで定量化するメトリクス
3. **早期停止メカニズム**: 合意形成に基づくコスト効率的な終了判定
4. **エージェントチーム再編**: タスク解決中の動的なエージェント削減

HumanEvalベンチマークでPass@1 = 82.9%を達成し、コード生成タスクにおける有効性を実証した。

---

## 2. 問題と動機

### 2.1 既存手法の課題

従来のLLMベースマルチエージェントシステムには以下の問題がある：

1. **固定的な組織構造**: エージェントの役割や通信パターンが事前に定義され、タスク固有の適応ができない
2. **チーム構成の非最適性**: 全エージェントを常に使用するため、不要なエージェントによるノイズや計算コストが発生
3. **計算効率の低さ**: 逐次的な推論手法（LATS、Reflexion等）はAPIコールが多く、レイテンシが大きい
4. **評価メトリクスの不在**: エージェント個別の貢献度を定量化する方法が確立されていない

### 2.2 研究動機

- タスク特性に応じた最適なエージェントチームの自動構成
- 動的なエージェント間通信構造によるロバストな協調
- 計算コストと性能のバランスを取る効率的な実行メカニズム
- エージェント貢献度の教師なし評価手法の開発

---

## 3. 提案手法

### 3.1 T-FFN（Temporal Feed-Forward Network）抽象化

マルチエージェント協調を多層ネットワークとして定式化する：

- **各層**: 1つのタイムステップを表現
- **ノード**: 個々のエージェント
- **エッジ**: エージェント間の通信チャネル

この定式化により、既存の様々なマルチエージェントアーキテクチャ（ディベート、ラウンドテーブル、階層型等）を統一的なフレームワークで表現可能になる。

### 3.2 Agent Importance Score

各エージェントの貢献度を教師なしで定量化する3段階プロセス：

1. **伝播（Propagation）**: エージェントが先行エージェントの応答を評価
2. **集約（Aggregation）**: 逆方向のメッセージパッシングにより貢献スコアを層間で集約
3. **選択（Selection）**: 重要度上位k個のエージェントで最適化チームを構成

スコア計算式：ノード v_{t-1,i} の貢献度 = 後続ノードの貢献度 × ピア評価の加重和

### 3.3 エージェントチーム再編

タスク解決中の動的なエージェント管理：

- **LLMランカー**: 各ステップでエージェントの応答品質を評価
- **動的プルーニング**: 上位ランクのエージェントのみが次ステップに参加可能
- **適応的通信構造**: パフォーマンスの低いエージェントを排除し、効率的な協調を維持

### 3.4 早期停止メカニズム

計算コスト削減のための終了条件：

- **合意基準**: 単一層内で2/3以上のエージェントが一貫した回答を持つ場合に終了
- **コスト削減効果**: APIコールを45-66%削減しつつ性能を維持
- **適応性**: タスクの難易度に応じて推論ステップ数が自動調整

## 4. アーキテクチャ

### 4.1 2段階パイプライン

```
Stage 1: チーム最適化
  候補エージェント群 → T-FFN実行 → Agent Importance Score計算 → 最適チーム選択

Stage 2: タスク解決
  最適チーム → 動的T-FFN実行 → LLMランカーによる評価 → チーム再編
      ↑                                                    ↓
      ←────────── 早期停止判定 ←── 合意チェック ←──────────┘
```

### 4.2 エージェント構成

- **候補エージェント数**: 7体（タスクに応じた役割を保持）
- **最適チームサイズ**: 3-4体（実験的に最適と判明）
- **バックボーンモデル**: GPT-3.5、GPT-4（モデル間でロバスト性を確認）

### 4.3 通信プロトコル

各層において、エージェントは前層の全エージェント出力にアクセス可能。LLMランカーが応答品質を評価し、次層への参加資格を決定する。

## 6. 実験と評価

### 6.1 評価データセット

| データセット | タスク | サイズ | 評価指標 |
|-------------|--------|------|---------|
| HumanEval | コード生成 | 164問 | Pass@1 |
| WebShop | 意思決定 | 50環境 | 報酬スコア |
| MMLU | 一般推論 | 5%サンプル（57科目） | 精度 |
| MATH | 算術推論 | 5,000問（7分野） | 精度 |

### 6.2 主要実験結果

- **HumanEval Pass@1 = 82.9%**: ベースライン比+9.7%の改善（最も顕著な結果）
- **WebShop報酬**: +17.7%の改善
- **MMLU精度**: +4.1%の改善（特定科目では最大+25.0%）
- **MATH精度**: +4.1%の改善

### 6.3 効率性分析

- **APIコール削減**: 早期停止により45-66%削減
- **逐次手法との比較**: LATS・Reflexionと比較して35-60%のAPIコール削減
- **単一実行比較**: 4-18%の性能向上（タスクに依存）

### 6.4 アブレーション分析

- **チームサイズの影響**: 3-4体が最適（7候補から選択）
- **早期停止の貢献**: 45-66%のAPIコール削減、精度低下は最小限
- **チーム再編の重要性**: オープンエンドタスクで不可欠（除去で約30%の精度低下）
- **ロバスト性**: GPT-3.5とGPT-4の両方で一貫した性能向上を確認

### 6.5 科目別分析

MMLU科目別の分析では、Agent Importance Scoreによるチーム選択が特定科目で最大25.0%の精度向上をもたらすことが確認された。タスク特性に応じた最適なエージェント構成が異なることを示唆している。

## 7. 備考

### 7.1 学術的意義

DyLANは、マルチエージェント協調を多層フィードフォワードネットワークとして統一的に抽象化した点で重要な貢献をしている。T-FFN定式化により、既存の様々なアーキテクチャを特殊ケースとして包含しつつ、動的なチーム構成と通信構造を実現した。Agent Importance Scoreは、エージェント貢献度の教師なし評価という新たな方向性を示している。

### 7.2 データ分析エージェントへの示唆

- **動的チーム構成**: データ分析タスクの種類（EDA、モデリング、可視化等）に応じたエージェントチームの自動選択に応用可能
- **早期停止**: APIコスト管理において実践的な意義。分析結果に対するエージェント間合意を終了条件とする設計が有用
- **貢献度評価**: 分析パイプラインにおける各エージェントの有効性を定量的に評価し、パイプライン最適化に活用可能

### 7.3 制限事項

- 候補エージェント数が7体と比較的小規模（より大規模な候補プールでの検証が必要）
- MMLUの5%サンプリングによる評価（全データでの検証が望ましい）
- 特定のLLMバックボーン（GPT系列）に依存した評価

### 7.4 COLM 2024での位置づけ

COLM 2024（Conference on Language Modeling）に採択された本論文は、LLMベースマルチエージェントシステムの効率化と動的構成に関する先駆的研究として位置づけられる。T-FFN抽象化とAgent Importance Scoreの概念は、後続研究に広く引用されている。
