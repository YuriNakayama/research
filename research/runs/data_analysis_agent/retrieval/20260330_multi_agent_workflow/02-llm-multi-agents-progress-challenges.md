## Abstract (English)

Large Language Model based Multi-Agents systems have achieved considerable progress in complex problem-solving and world simulation. This survey examines the development of LLM-based multi-agent systems, addressing key questions regarding which domains these systems simulate, agent profiling and communication methods, and mechanisms that enhance agent capabilities. The survey compiles commonly used datasets and benchmarks and maintains an open-source GitHub repository documenting current research.

## Abstract（日本語訳）

大規模言語モデル（LLM）に基づくマルチエージェントシステムは、複雑な問題解決と世界シミュレーションにおいて顕著な進歩を遂げている。本サーベイは、LLMベースのマルチエージェントシステムの発展を調査し、これらのシステムがどのドメインをシミュレートするか、エージェントのプロファイリングとコミュニケーション手法、およびエージェント能力を強化するメカニズムに関する主要な問いに取り組む。また、一般的に使用されるデータセットとベンチマークをまとめ、現在の研究を記録するオープンソースのGitHubリポジトリを維持している。

## 概要

本論文は、LLMベースのマルチエージェントシステム（LLM-MA）に関する包括的なサーベイであり、問題解決と世界シミュレーションの2大分野における研究動向を体系的に整理している。著者らは4つの柱からなるフレームワークを提案し、30以上の研究をこの枠組みで分類・分析している。LLM-MAシステムの急速な成長を可視化し、ソフトウェア開発、具現化エージェント、科学実験、ゲーム、社会シミュレーション、経済シミュレーションなど多岐にわたる応用領域をカバーしている。

## 問題と動機

LLMの単一エージェントとしての利用は広く研究されてきたが、複雑な現実世界の問題は単一のエージェントでは対処しきれないことが多い。マルチエージェントシステムは、複数の専門的なエージェントが協力・議論・競争することで、以下のような課題に対応する。

1. **複雑性への対処**: 現実世界の問題は多面的であり、異なる専門性を持つ複数のエージェントの協調が必要
2. **ロバスト性の向上**: 複数のエージェントによる検証・議論により、LLMのハルシネーション（幻覚）を軽減
3. **スケーラビリティ**: 新しいタスクや要件に応じてエージェントを動的に追加・調整可能
4. **世界シミュレーション**: 社会、経済、心理学的現象のシミュレーションには、多数のエージェントによるインタラクションが不可欠

## 提案手法（4柱フレームワーク）

本サーベイの中核は、LLM-MAシステムを理解するための4つの柱（ピラー）からなる体系的なフレームワークである。

### 第1の柱: エージェント-環境インターフェース（Agents-Environment Interface）

エージェントが動作する環境を3つのカテゴリに分類する。

| 環境タイプ | 説明 | 代表例 |
|-----------|------|--------|
| **サンドボックス** | 仮想・シミュレーション環境で自由な実験が可能 | ゲーム環境、コード実行環境 |
| **物理環境** | 現実世界の物理制約を伴う環境 | ロボット協調、倉庫管理 |
| **環境なし** | 外部環境とのインタラクションなし | ディベート、推論タスク |

環境の選択はシステム全体のアーキテクチャ設計に根本的な影響を与え、エージェントの相互作用パターンを決定づける。

### 第2の柱: エージェントプロファイリング（Agent Profiling）

エージェントの役割・特性・能力を定義するプロファイリング手法を3つに分類する。

- **事前定義型（Pre-defined）**: システム設計者が明示的にエージェントの役割と特性を設計
- **モデル生成型（Model-Generated）**: LLM自身がエージェントプロファイルを生成
- **データ駆動型（Data-Derived）**: 既存のデータセットからエージェントの特性を構築

これらは設計者制御から創発的なエージェント特性化に至るスペクトラムを形成し、タスクの性質に応じて最適なアプローチが異なる。

### 第3の柱: エージェント通信（Agent Communication）

通信の3つの次元を体系化する。

**通信パラダイム:**

| パラダイム | 特徴 | 適用場面 |
|-----------|------|---------|
| **協調型（Cooperative）** | 共有目標に向けた協力 | ソフトウェア開発、科学実験 |
| **ディベート型（Debate）** | 議論的インタラクション | 推論タスク、事実確認 |
| **競争型（Competitive）** | 対立する目標 | ゲーム、経済シミュレーション |

**通信構造:**

```
┌─────────────────────────────────────────────────────┐
│  階層型（Layered）         │  分散型（Decentralized）   │
│  ┌───┐                    │    ○ ─── ○               │
│  │ L │ リーダー            │   / \   / \              │
│  └─┬─┘                    │  ○   ○ ○   ○             │
│  ┌─┴──┐                   │                          │
│  │F│ │F│ フォロワー        │  P2P型の対等な通信        │
│  └──┘ └──┘                │                          │
├────────────────────────────┼──────────────────────────┤
│  集中型（Centralized）     │  共有メッセージプール      │
│    ○                      │  ┌──────────────────┐    │
│   /│\                     │  │  Message Pool     │    │
│  ○ ○ ○ ハブ型             │  │  (Pub/Sub方式)    │    │
│                           │  └──────────────────┘    │
└─────────────────────────────────────────────────────┘
```

**通信内容:** 主にテキストベースの交換であり、応用ドメインによって内容が変化する。

### 第4の柱: 能力獲得（Capability Acquisition）

エージェントが能力を発展させるメカニズムを体系化する。

**フィードバックソース（4カテゴリ）:**

1. **環境フィードバック**: 環境からの直接的な応答・結果
2. **エージェント間フィードバック**: 他のエージェントからの評価・助言
3. **人間フィードバック**: 人間の判断・修正による学習
4. **フィードバックなし**: 自律的な動作

**調整メカニズム:**

- **メモリ**: 過去の情報を検索・活用して行動を改善
- **自己進化（Self-Evolution）**: 動的なプロファイル修正による適応
- **動的生成（Dynamic Generation）**: 運用中に新しいエージェントを生成

## 実験と評価

本論文はサーベイ論文であるため、独自の実験は実施していないが、対象となった研究群から以下の知見を整理している。

### 問題解決分野の評価

- **コード生成**: HumanEvalおよびMBPPベンチマークにおいて、マルチエージェント方式が単一エージェントを上回る成果を示す研究が多数
- **具現化エージェント**: RoCoBenchにおけるマルチロボット協調の評価では、役割分担と通信構造の選択がパフォーマンスに大きく影響
- **科学ディベート**: MMLUやGSM8Kにおいて、ディベート型通信がファクトチェック精度を向上させることが複数の研究で確認

### 世界シミュレーション分野の評価

- **社会シミュレーション**: SOTOPIAベンチマークでは、エージェント数25〜17,945の規模でシミュレーションの妥当性を評価
- **ゲーム**: Werewolf、Avalon等の戦略ゲームにおいて、ゲーム理論的仮説の検証が可能であることを実証
- **経済シミュレーション**: マクロ経済モデリングにおいて、エージェントベースシミュレーションの現実との整合性を評価

### 主要な課題

1. **スケーラビリティ**: エージェント数の増加に伴う計算コストと通信オーバーヘッドの管理
2. **ハルシネーション**: LLMの出力における事実と異なる情報の生成抑制
3. **評価基準の標準化**: ドメイン横断的な統一評価フレームワークの不在
4. **長期コンテキスト**: 長期間のインタラクションにおけるコンテキスト管理の困難さ

## 備考

- 本サーベイは、LLM-MAシステムの包括的な分類体系を提供する初の試みの一つであり、4柱フレームワークは後続研究の基盤として広く参照されている
- オープンソースのGitHubリポジトリで最新の研究カタログを維持しており、分野の急速な発展に対応
- 問題解決と世界シミュレーションの二分法は、研究目的の根本的な違い（実用性 vs. 創発的行動の理解）を反映しており、フレームワーク設計に重要な示唆を与える
- 2024年初頭に公開されて以降、この分野のサーベイ論文として最も頻繁に引用される論文の一つとなっている
- 通信構造の選択（階層型 vs. 分散型 vs. 集中型 vs. 共有プール）がシステム性能に与える影響の定量的分析は、今後の研究課題として残されている
