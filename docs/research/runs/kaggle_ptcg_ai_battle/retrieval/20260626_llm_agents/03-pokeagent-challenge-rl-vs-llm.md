# PokeAgent Challenge と Metamon: 競技ポケモンでRLエージェントがLLMを圧倒した教訓

## 概要

PokeAgent Challenge(NeurIPS 2025 競技トラック, arXiv:2603.15563, Seth Karten ら)は、ポケモン対戦をAI意思決定のテストベッドとして100チーム超が参加した大規模コンペである。Battling Track(対戦)とSpeedrunning Track(RPG攻略)の2部門を持ち、対戦部門ではPokeChamp由来のLLMベースラインとMetamon(arXiv:2504.04395, Jake Grigsby ら)由来のRLベースラインが提供された。**本コンペ「PTCG AI Battle」と構造が酷似**(不完全情報・確率的・チーム構築・ラダー評価)しており、その結論「RL専門エージェントがLLMハーネスを圧倒する」は本コンペの方針決定に直結する最重要知見である。

## 手法の要点

**コンペ設計(Battling Track)**
- 二人零和・確率的・不完全情報・同時行動選択のゲーム。1ターン約9手、対戦は20〜100ターン。Gen1 OUとGen9 OUの2フォーマット。
- データセット: 観戦視点を各プレイヤー視点に再構成した人間デモ400万局+自己対戦による合成軌跡1800万+競技チーム20万超。
- 評価: Full-History Bradley-Terry(FH-BT)レーティング(全対戦履歴で推定+ブートストラップ不確実性)。補助にGlicko-1・GXE。
- **タイマー設定**: デフォルト60〜90秒ではLLM推論に不足し、LLM公平評価用に「ほぼ無制限の熟考時間」を与えるExtended Timer設定を別途用意した。

**Metamon(優勝RLベースライン)の要点**
- 観戦リプレイを「(1)現状態の再現→(2)非公開チーム推定→(3)対戦終了時の未公開情報推定→(4)情報の段階開示でtrajectory逆充填」の4段階で一人称軌跡に再構成。47.5万局→約95万系列・3800万ステップ。
- 1観測あたり87語のテキスト+48個の数値特徴。行動空間は技4 + 交代5の9離散行動。15M/50M/200Mパラメータのトランスフォーマーencoderに要約トークンを付与。
- 行動クローン(IL)+オフラインRL(指数優位重み付け/二値フィルタ/MaxQ)+自己対戦データ(SynRL)。報酬は勝敗二値中心に軽いシェーピング。
- **明示的な相手モデルや探索を持たず**、長文脈の系列モデルが相手チームと傾向を暗黙に推論する。

## 主要な結果や知見

- **RLがLLMを圧倒**: 「専門RLエージェントは探索深度を増やしたLLMハーネスを一貫して上回る」。Gen1 OU優勝はRLベースライン拡張のPA-Agent、Gen9 OU優勝は独立探索手法のFoulPlay。16枠中13チームがRLベースラインを拡張。
- **フロンティアLLMの崩壊**: 競技プレッシャー下でパニック行動を示し、戦術ミスが連鎖して敗北。失敗様式はモデル系統で異なる(「記憶破損のカスケード」「目標の振動」「過剰なプラン固執」)。
- Metamon最良モデル(SynRL-V2)はGen1OUで瞬間31位・人間上位10%相当。ヒューリスティック探索(Foul Play)と互角〜優勢、PokeLLMonを長期戦で大幅に上回る。
- **直交性の発見**: BenchPress分析で「ポケモン対戦は標準ベンチマークとほぼ直交(最大相関ρ=0.77, 平均|ρ|=0.45)」。標準ベンチで強いモデルが対戦で崩壊し、逆もある。汎用LLM性能=ゲーム性能ではない。

## 本コンペ(PTCG AI Battle)への応用

- **戦略的最重要示唆**: 同型のコンペで「学習済みRL方策 > LLMハーネス」が実証された。PTCGでも、**自己対戦データでオフラインRL/行動クローンした専門方策を主軸**にし、LLMは補助(序盤プラン・新規局面)に留める設計が有力。時間切れ即敗北の本コンペでは、ミリ秒で手を返すRL方策がGaussianラダー上で圧倒的に有利。
- **データ再構成手法の移植**: Metamonの「観戦→一人称・非公開情報の段階開示」再構成は、ポケカの自己対戦ログから学習用trajectoryを作る際にそのまま使える。隠れ手札・山札の未公開情報を期待値/分布で埋める扱いが鍵。
- **観測・行動エンコードの参考**: 「テキスト + 数値特徴 + 要約トークン」のトランスフォーマー設計は、`obs_dict`(logs/current/select)を固定長表現に落とす良い雛形。合法手index出力は分類ヘッドで実装でき`agent(obs)->list[int]`に直結。
- **時間設計**: コンペ主催はLLMの遅さのためExtended Timerを別設したが、PTCGにそんな救済は無く10分超で即敗北。**推論レイテンシをアーキ選定の第一制約**に置くべき。200Mトランスフォーマーでも1手数十ms、LLMは1手数秒〜数十秒。10分予算をどの局面に配分するかが勝敗を分ける。
- **デッキ60枚最適化**: 20万チームのメタ分析の発想で、自己対戦勝率からデッキを進化的/ベイズ最適化する別パイプラインを構築し、対戦方策と分離して最適化するのが定石。
