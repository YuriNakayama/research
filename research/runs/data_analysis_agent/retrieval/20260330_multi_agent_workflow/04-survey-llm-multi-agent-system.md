## Abstract (English)

LLM-based Multi-Agent Systems (LLM-MAS) have become a research hotspot since the rise of large language models (LLMs). However, with the continuous influx of new related works, the existing reviews struggle to capture them comprehensively. This paper presents a comprehensive survey of these studies. We first discuss the definition of LLM-MAS, a framework encompassing much of previous work. We provide an overview of the various applications of LLM-MAS in (i) solving complex tasks, (ii) simulating specific scenarios, and (iii) evaluating generative agents. Building on previous studies, we also highlight several challenges and propose future directions for research in this field.

## Abstract（日本語訳）

LLMベースのマルチエージェントシステム（LLM-MAS）は、大規模言語モデル（LLM）の台頭以来、研究のホットスポットとなっている。しかし、新しい関連研究が継続的に流入する中、既存のレビューはそれらを包括的に捉えることが困難になっている。本論文は、これらの研究の包括的なサーベイを提示する。まずLLM-MASの定義を議論し、先行研究の多くを包含するフレームワークを提供する。次に、(i) 複雑なタスクの解決、(ii) 特定シナリオのシミュレーション、(iii) 生成エージェントの評価における様々な応用の概観を提供する。先行研究に基づき、いくつかの課題を強調し、この分野における将来の研究方向を提案する。

## 概要

本論文は、2023年から2024年にかけてACL、NeurIPS、AAAI、ICLR、arXivなどの一流会場で発表された**125本の論文**を体系的に分析した大規模サーベイである。従来のアーキテクチャ中心の分類法とは異なり、目的駆動型の分類を採用し、LLM-MASの応用を3つの主要カテゴリに整理している。エージェントの構成要素（プロファイリング、メモリ、プランニング、アクション）の詳細な分析と、通信プロトコル、オープンソースフレームワークの比較評価を含む包括的な調査を提供する。

## 問題と動機

LLM-MASの研究は急速に拡大しており、以下の問題意識が本サーベイの動機となっている。

1. **既存レビューの限界**: 新しい研究の継続的な流入により、既存のサーベイが最新の研究を十分にカバーできていない
2. **統一的フレームワークの不在**: 多様なアプローチを統一的に理解・比較するための枠組みが必要
3. **応用領域の急拡大**: 問題解決、シミュレーション、評価という3つの異なる応用方向が並行して発展
4. **スケーラビリティの課題**: エージェント数の増大に伴う推論効率の低下とエラー蓄積の問題
5. **評価の断片化**: ドメインごとにベンチマークが分断され、統一的な評価基準が欠如

## 提案手法

### 3つの応用カテゴリ

本サーベイは125本の論文を以下の3つの主要カテゴリに分類する。

#### カテゴリ1: 複雑なタスクの解決（Complex Task Solving）

```
複雑なタスク解決
├── マルチステージフレームワーク
│   └── エージェントが異なる段階で逐次的に問題を解決
├── 集団的意思決定
│   └── エージェントが投票や議論を通じて統一目標に向かう
├── 自己改良メカニズム
│   └── 自己反省と反復的改善ループ
└── 通信最適化
    ├── 速度最適化（非言語通信、略式生成）
    └── 分散議論（不完全情報下での協調）
```

代表的なシステムとして、ChatDev（ソフトウェア開発）、コード生成フレームワーク、推論タスクシステムが挙げられる。

#### カテゴリ2: シナリオシミュレーション（Scenario Simulation）

```
シナリオシミュレーション
├── 社会ドメイン
│   ├── Stanford Town（25エージェントによる日常生活シミュレーション）
│   ├── 感情伝播ネットワーク
│   ├── 情報繭（フィルターバブル）
│   └── 社会運動シミュレーション
├── 物理ドメイン
│   ├── 都市システム（UGI）
│   ├── 無線ネットワーク
│   └── 交通システム
├── ゲーム環境
│   ├── Werewolf（欺瞞とリーダーシップ）
│   ├── Avalon（協力と裏切り）
│   └── Minecraft（具現化ナビゲーション）
└── 経済システム
    ├── 病院シミュレーション
    ├── マクロ経済活動（EconAgent）
    └── オークション・入札シナリオ
```

シミュレーション規模は数十から10^6エージェントまで幅広い。

#### カテゴリ3: 生成エージェントの評価（Generative Agent Evaluation）

- LLM-MASを動的評価環境として使用
- 戦略的能力、感情推論、競争行動の評価
- 教師ありファインチューニング、強化学習、合成データ生成によるエージェント訓練

### エージェント構成要素（4コンポーネント）

各エージェントは以下の4つの中核コンポーネントから構成される。

```
┌──────────────────────────────────────────────┐
│              生成エージェント                   │
│                                               │
│  ┌────────────┐      ┌────────────┐          │
│  │ Profiling   │      │ Memory     │          │
│  │ (プロファイ │      │ (メモリ)    │          │
│  │  リング)    │      │            │          │
│  │             │      │ ・長期     │          │
│  │ 自然言語に  │      │ ・短期     │          │
│  │ よる役割定義│      │ ・感覚     │          │
│  │ カスタマイズ│      │            │          │
│  │ プロンプト  │      │ 長期ホライ │          │
│  └────────────┘      │ ズンタスク │          │
│                       │ の検索     │          │
│  ┌────────────┐      └────────────┘          │
│  │ Planning   │                               │
│  │ (プランニン│      ┌────────────┐          │
│  │  グ)       │      │ Action     │          │
│  │             │      │ (アクション│          │
│  │ 長期の将来  │      │  )         │          │
│  │ 行動戦略の  │      │            │          │
│  │ 策定        │      │ 環境との   │          │
│  └────────────┘      │ インタラク │          │
│                       │ ション実行 │          │
│                       └────────────┘          │
└──────────────────────────────────────────────┘
```

| コンポーネント | 機能 | 詳細 |
|--------------|------|------|
| **プロファイリング** | 役割定義 | 自然言語による役割定義、エージェントごとのカスタマイズプロンプト |
| **メモリ** | 情報保持 | 3層システム（長期、短期、感覚）、長期ホライズンタスクのための検索機能 |
| **プランニング** | 行動計画 | 長期的な将来の行動戦略の策定 |
| **アクション** | 実行 | 環境とのインタラクション実行（制約付き選択またはオープンエンド生成） |

### 通信タイプ

- **自然言語通信**: 高い解釈可能性、最適化が困難
- **カスタムシグナル**: ポリシー勾配による最適化が可能、不透明

### 通信アーキテクチャ

| アーキテクチャ | 特徴 | 課題 |
|--------------|------|------|
| **完全結合型** | 全エージェント間で通信 | 組合せ爆発とプライバシー漏洩のリスク |
| **速度最適化型** | 非言語通信、略式生成 | 情報損失の可能性 |
| **分散議論型** | 不完全情報下での協調 | エージェント間調整の複雑さ |

### オープンソースフレームワーク

| フレームワーク | 特徴 |
|--------------|------|
| **MetaGPT** | 役割ベースの協調エンティティ設計 |
| **AgentScope** | 分散並列最適化、ローカルから分散へのシームレスな切り替え |
| **OpenAI Swarm** | 軽量マルチエージェントオーケストレーション、細粒度の開発者制御 |
| **IOA** | インターネットライクなアーキテクチャ、動的タスクへの拡張性 |

## 実験と評価

本論文はサーベイ論文であり独自の実験は実施していないが、125本の論文から得られた以下の知見を体系的に整理している。

### タスク解決の評価結果

- マルチステージフレームワークは逐次的な問題分解において効果的だが、エラーの蓄積が課題
- 集団的意思決定は単一エージェントを上回る精度を達成するが、通信コストが増大
- 自己改良メカニズムは反復回数の増加に伴い性能が向上するが、収束の保証がない

### シミュレーションの評価結果

- 社会シミュレーションでは、Stanford Townの25エージェントから10^6エージェント規模まで拡張可能
- ゲーム環境では、Werewolf（欺瞞能力）やAvalon（協力と裏切り）における戦略的行動が人間に近いパフォーマンスを示す
- 経済シミュレーションでは、EconAgentによるマクロ経済活動モデリングが現実のデータとの整合性を示す

### 主要な課題

**エージェントレベルの課題:**
- リアルなシミュレーションのためのアラインメント問題
- エージェントインタラクションにおけるハルシネーション
- 長いコンテキストの処理限界

**システムレベルの課題:**
- 自己回帰型LLMクエリによる推論効率の爆発的増大
- 逐次ラウンド間のエラー蓄積
- 大きなアクション空間によるスケーラビリティ制約

**評価の課題:**
- 客観的な集団レベルメトリクスの不在
- ドメイン横断的なベンチマークの断片化
- 統一的な評価フレームワークの欠如

## 備考

- 125本の論文を体系的に分析した点は、この分野のサーベイとして最大規模の一つであり、研究者にとって包括的なリファレンスとして機能する
- 従来のアーキテクチャ中心の分類から目的駆動型の分類への転換は、実務者がユースケースに基づいてシステム設計を検討する際に有用
- 生成エージェント評価（第3カテゴリ）の独立したカテゴリとしての位置づけは、LLM-MASが評価ツールとしても機能するという新しい視点を提供
- MetaGPT、AgentScope、OpenAI Swarmなどのフレームワーク比較は、実装者にとって具体的な選択指針を提供
- 2024年12月時点の最新情報を含む一方、この分野の急速な発展により、論文公開後に登場した重要なシステム（例：Claude MCP、OpenAI Agents SDK）はカバーされていない
