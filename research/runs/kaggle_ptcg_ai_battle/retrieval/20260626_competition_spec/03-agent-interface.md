# 03. エージェント I/F と提出形式

> ⚠️ 本章は GitHub コミュニティ実装（`wmh/ptcg-abc`）と Kaggle 検索スニペット・解説記事から再構成したもの。**観測辞書の正確なキー構造は公式スターターノートブックで確認すること**（→ [04](04-open-questions.md)）。

## 基本契約（コア I/F）

エージェントの実装は次のシンプルな関数契約に従う:

```python
def agent(obs_dict) -> list[int]:
    """
    obs_dict: ゲームログ・盤面状態・legal moves（合法手リスト）を含む観測辞書。
    戻り値:   legal moves の中から選んだアクションの index（リスト）。
    """
    ...
```

- 概念的には「**現在の盤面を見て次に何をするか決めるプログラム**」。
- 毎ターン `obs_dict` を受け取り、**合法手リストのインデックス** を返す。
- **デッキ選択フェーズ** では、アクション index の代わりに **60 枚のカード ID** を返す。

### 解説記事ベースの簡易シグネチャ
```python
def choose_action(observation):
    # observation = 現在の対戦状態（自分・相手の盤面、手札 等）
    # ここに意思決定ロジックを書く
    ...
```

## 観測（Observation）に含まれる情報

毎ターン、エージェントは以下を含む観測を受け取る:

| 要素 | 内容 |
|------|------|
| **ゲームログ** | これまでの対戦経過（双方の行動履歴） |
| **盤面状態（board state）** | 自分／相手のバトル場・ベンチ・トラッシュ・サイド枚数 等 |
| **手札（hand）** | 自分の手札 |
| **legal moves（合法手リスト）** | その時点で選択可能な行動の一覧 |

エージェントはこの観測を評価し、`legal moves` の中から最適なインデックスを選んで返す。

> 実装メモ: `wmh/ptcg-abc` では「per-`SelectContext` scoring（選択コンテキストごとのスコアリング）」という表現があり、状況（どの選択を求められているか）に応じてスコア最大の手を選ぶ設計が示唆されている。各 `SelectContext` がどの情報を提供するかは公式ドキュメント未確認。

## エージェントの必須要件（再掲）

1. 合法手がある限り、**必ず** `legal moves` から返す。
2. **範囲外（非合法）のアクションを返さない**。
3. **絶対にクラッシュしない**（例外時も合法なフォールバックを返す）。
4. **手番ごとの時間制限を厳守**（1 プレイヤー最大 10 分の持ち時間）。

## デッキ定義

- デッキは **`deck.csv`** として定義する（60 枚のカード ID のリスト）。
- カード ID は Kaggle が提供する **公式カードデータ** から取得する。
- 各エージェントリポジトリは自身の `deck.csv` を持つ。

## 提出形式

提出物は **`submission.tar.gz`** で、以下を含む:

```
submission.tar.gz
├── main.py      # agent(obs_dict) -> list[int] を実装
├── deck.csv     # 60 枚のカード ID
└── cg/          # cabt エンジンのフォルダ
```

## ローカル開発・テスト環境

| ツール | 用途 |
|--------|------|
| `kaggle-environments==1.30.1` | 公式ラダーと同一バージョンの実行環境 |
| `cabt_eval.py` | 公式 cabt 環境で meta デッキと対戦評価（「より正確」とされる） |
| `web/server.py` | ローカルブラウザ UI で人間 vs エージェント対戦 |

- 公式エンジンとカードアセットは Kaggle コンペページから別途ダウンロードが必要。

## 実装の発展段階（解説記事より）

| レベル | アプローチ |
|--------|-----------|
| **初級** | ランダム行動 → 強いカードを優先する形で段階的改善 |
| **中級** | if 文によるルールベース戦略 |
| **上級** | 機械学習／強化学習で過去の対戦から学習（PPO エージェント等の実装例あり） |

### 上位を狙うための実装上の論点
- **評価関数の精度**: 盤面の優劣を正しく定量化することが強さに直結。
- **確率的判断**: 山札の残りカードを確率的に考慮し、運負けを減らす。
- **相手モデリング**: 相手の行動から相手デッキ構成を推定する。
- **時間管理**: 1 対戦 10 分の持ち時間。計算コストの高い処理を避ける。
