## 研究の位置付け

住宅 HVAC における **RL と MPC の同一住宅・同一ベースラインによる対面比較**という、これまで稀だった精緻な実証研究。1ヶ月にわたる占有住宅での運用比較で、RL と MPC それぞれの優位点を定量化した点に最大の意義がある。

## 実験設定

- **対象建物**: 米国インディアナ州ウェストラフィエットの 208m² 1920年代築タウンハウス（占有済み）
- **HVAC システム**: 全電化、空気-空気ヒートポンプ + 段階的電気抵抗バックアップ
- **期間**: 2025年1月23日〜2月23日（冬季 30日）
- **共通モデル**: 2R1C 熱 RC ネットワーク
- **計画ホライズン**: 24時間、1時間ステップ

## 手法

| コントローラ | 手法詳細 |
|------------|---------|
| **RL (Ibex-RL)** | 模倣学習による初期化 → オンライン微調整。物理情報付き 2R1C モデルとパラメータをエンドツーエンド学習 |
| **MPC** | 手動設計のシステムダイナミクスモデル + 12時間ごとに自動チューニングされる快適性-コスト重み |
| **ベースライン** | 既存 PID コントローラ |

## 主要な結果

| 指標 | RL | MPC |
|------|-----|-----|
| ベースライン比省エネ率 | **22%**（変動 14〜30%） | **20%** |
| 共通温度域 (-7℃〜5℃) でのエネルギー消費 | 522 kWh·℃ | **536 kWh·℃** |
| 快適性正規化後の効率改善 | 7.3% | **12.7%** |
| 室内温度逸脱（不快感） | 多い | 少ない |

**重要な知見**: RL は単純な省エネ率では勝つが、**「同じ快適性レベルでどれだけ省エネできるか」では MPC が優位**。RL は予測的な setpoint 制御を見せる一方、MPC は緩やかな移行を行う。

## 限界・実装上の課題

1. **モデル精度**: RL のオンライン学習パラメータが**物理的に非現実的な値**（負の太陽透過係数）に収束。系の励起不足と imitation learning の目的との競合が原因。
2. **制御実装ミスマッチ**: コントローラは消費電力を出力するが、実装はサーモスタットの setpoint 経由。コントローラ予測の還気温度と物理サーモスタットのセンサ温度が異なる。
3. **コストパラメータ**: RL は不快感重み (wc=3) を固定、MPC は 12時間ごとに適応 → RL に不利。
4. **訓練条件**: RL と MPC が異なる期間・気象で訓練されたため、直接比較の妥当性に注意。
5. **観測期間**: 30日では長期安定性は判断不能。

## 意義

- 学術界で長く議論されてきた「RL vs MPC どちらが実用的か」の問いに、**実建物での同条件比較**で具体的回答を与えた。
- 結論は「**省エネだけ見るな、快適性正規化後を見よ**」という実用的指針。
