# Reasoning RAG via System 1 or System 2: A Survey on Reasoning Agentic RAG for Industry Challenges

- **Link**: https://arxiv.org/abs/2506.10408
- **Authors**: Jintao Liang, Gang Su, Huifeng Lin, You Wu, Rui Zhao, Ziyue Li
- **Year**: 2025
- **Venue**: arXiv preprint (cs.AI, cs.IR)
- **Type**: Academic Paper (Survey)

## Abstract

Retrieval-Augmented Generation (RAG) has emerged as a powerful framework to overcome the knowledge limitations of Large Language Models (LLMs) by integrating external retrieval with language generation. The survey examines how the field has progressed from static pipeline systems toward more sophisticated approaches. The authors categorize reasoning agentic RAG into two systems: predefined reasoning, which follows fixed modular pipelines to boost reasoning, and agentic reasoning, where the model autonomously orchestrates tool interaction during inference. The paper analyzes architectural design, reasoning strategies, and tool coordination while discussing future research directions for enhancing flexibility and robustness.

## Abstract（日本語訳）

検索拡張生成（RAG）は、外部検索と言語生成を統合することで大規模言語モデル（LLM）の知識の限界を克服する強力なフレームワークとして登場した。本サーベイは、この分野が静的パイプラインシステムからより洗練されたアプローチへとどのように進展してきたかを検討する。著者らは推論エージェンティックRAGを2つのシステムに分類する：推論を強化するための固定モジュラーパイプラインに従う事前定義推論と、推論時にモデルがツールインタラクションを自律的にオーケストレートするエージェンティック推論である。本論文はアーキテクチャ設計、推論戦略、ツール協調を分析し、柔軟性とロバスト性の向上に向けた将来の研究方向を議論する。

## 概要

本論文は、推論能力を組み込んだRAGシステムの全体像を、認知科学のデュアルプロセス理論（System 1 / System 2）の枠組みで整理した独自のサーベイである。「高速・構造化・効率的」な事前定義推論（System 1）と「低速・熟慮的・適応的」なエージェンティック推論（System 2）という二分法を導入し、各アプローチの設計パターン、実装手法、強みと制約を体系的に分析している。

主要な貢献：

1. **System 1 / System 2のフレームワーク導入**: 認知科学のデュアルプロセス理論をRAGアーキテクチャの分類に適用
2. **事前定義推論の4カテゴリ分類**: ルートベース、ループベース、ツリーベース、ハイブリッドモジュラー
3. **エージェンティック推論の2アプローチ分類**: プロンプトベース（訓練不要）と訓練ベース（RL最適化）
4. **訓練ベースアプローチの最新動向の詳細分析**: Search-R1、R1-Searcher、ReZero、DeepRetrieval、DeepResearcherなどの強化学習ベース手法
5. **産業応用と実用課題への焦点**: 企業インテリジェンス、科学報告、技術サポート、マルチモーダル処理

## 問題と動機

- **静的RAGパイプラインの限界**: 従来のRAGは「検索→生成」の固定パイプラインに依存しており、クエリの複雑さや多様性に対する適応性が欠如している。

- **推論能力の統合の必要性**: 複雑なタスク（多文書統合、多ホップ推論、法的・生物医学的領域での推論）では、単純な検索と生成の組み合わせでは不十分であり、推論メカニズムの統合が不可欠である。

- **効率性と適応性のトレードオフ**: 事前定義パイプラインは効率的だが柔軟性に欠け、完全なエージェンティックアプローチは適応的だが計算コストが高い。両者の特性を理解し適切に選択するための体系的整理が必要であった。

- **強化学習ベースのRAG最適化の台頭**: Search-R1、DeepResearcherなど、RLを用いてRAGの検索・推論行動を最適化する手法が急速に登場しており、これらの体系的分類が求められていた。

## 分類フレームワーク / タクソノミー

### 基本RAGの3段階進化

| 段階 | 特徴 | 限界 |
|:---:|:---:|:---:|
| Naive RAG | スパース検索（BM25）、単純パイプライン | 意味理解の限界 |
| Advanced RAG | 密な意味マッチング、リランキング、細粒度チャンキング | 静的パイプライン |
| Modular RAG | 離散的・構成可能コンポーネント、柔軟な組成 | 推論能力の限界 |

### System 1: 事前定義推論（4カテゴリ）

**ルートベース手法**
- コンテキスト/信頼度に基づく条件分岐
- 代表例：RAGate、Self-Route
- 特徴：事前定義されたルーティングルールによる意思決定

**ループベース手法**
- フィードバックサイクルによる反復的精製
- 代表例：Self-RAG、CRAG（Corrective RAG）
- 特徴：自己省察と批評メカニズム

**ツリーベース手法**
- 階層的情報組織化
- 代表例：RAPTOR、MCTS-RAG
- 特徴：多ホップ推論のための再帰的構造

**ハイブリッドモジュラー手法**
- 特化コンポーネントの柔軟な組成
- 代表例：Adaptive-RAG、Modular-RAG
- 特徴：LEGOのような再構成可能フレームワーク

### System 2: エージェンティック推論（2アプローチ）

**プロンプトベースアプローチ（訓練不要）**

| 手法 | 特徴 | メカニズム |
|:---:|:---:|:--------:|
| ReAct | Thought-Action-Observationの交互列 | 推論と行動の交互実行 |
| Self-Ask | 中間的質問による問題分解 | 自己質問型分解推論 |
| Function Calling | 組み込みAPIによる直接ツール呼び出し | LLM native tool use |
| Large Reasoning Models | 内在的な高度推論能力の活用 | 大規模推論モデル |
| Search-o1 | コンテキスト管理+動的検索トリガー | 推論中の動的検索 |

**訓練ベースアプローチ（RL最適化）**

*ローカル検索システム:*

| 手法 | 特徴 | 学習対象 |
|:---:|:---:|:------:|
| Search-R1 | RLによる検索タイミング・内容の学習 | いつ・何を検索するか |
| R1-Searcher | 2段階アウトカム駆動フレームワーク | 結果指向の検索最適化 |
| ReZero | リトライ報酬によるペルシステンス促進 | 粘り強い検索行動 |

*実世界検索エンジン:*

| 手法 | 特徴 | 創発的行動 |
|:---:|:---:|:--------:|
| DeepRetrieval | 検索メトリクスによるクエリ定式化の最適化 | 精密なクエリ生成 |
| DeepResearcher | ライブWeb環境でのエンドツーエンドRL | 計画・交差検証・自己省察 |

## 主要な知見と分析

### 1. デュアルプロセスフレームワークの有効性

認知科学のSystem 1/System 2理論をRAGアーキテクチャの分類に適用した本フレームワークは、効率性と適応性のトレードオフを直感的に理解するための有効な枠組みを提供する。事前定義推論は「高速だが柔軟性に欠ける」、エージェンティック推論は「低速だが高い適応性を持つ」という二項対立が明確に整理されている。

### 2. 強化学習ベースRAGの急速な進展

Search-R1、R1-Searcher、ReZero、DeepRetrieval、DeepResearcherなど、RLを用いたRAG最適化手法が2025年に集中的に登場している。特にDeepResearcherはライブWeb環境でのエンドツーエンドRL訓練を実現し、計画、交差検証、自己省察という3つの創発的行動を示した点が注目に値する。これは明示的にプログラムされたものではなく、RL訓練の結果として自然に出現した行動である。

### 3. 事前定義推論とエージェンティック推論の相補性

両アプローチは対立するものではなく、タスク要件に応じて組み合わせることが効果的である。定型的なクエリにはSystem 1の効率性が、複雑な知識集約型タスクにはSystem 2の適応性が適する。実世界アプリケーションでは両者のハイブリッドが最も実用的な選択肢となる。

### 4. 報酬設計の重要性

訓練ベースアプローチにおいて、報酬関数の設計が性能を大きく左右する。回答正確性、フォーマット正確性、検索品質を含む細粒度の報酬関数が必要であり、これらのバランスが創発的行動の質を決定する。

### 5. 過剰検索の問題

エージェンティック推論において、過剰な検索はレイテンシの増大とノイズの混入を招く。検索の必要性を正確に判断し、情報が十分であることを認識する能力の獲得が重要な課題として残されている。

## 将来の研究方向

1. **高度なツール構成**: エージェントがフィルタリング、ソート、構造化クエリなど、洗練されたAPIオプションを活用する能力の開発。単純なキーワード検索を超えた精密なツール操作。

2. **精緻な報酬関数**: 最終回答の正確性だけでなく、中間ステップの品質も評価する細粒度報酬関数の設計。推論プロセス全体を報酬対象とすることで、より質の高い推論行動の学習を促進。

3. **検索効率の最適化**: 過剰検索の防止、戦略的ソース選択、情報充足性の認識。不必要な検索を抑制しつつ、必要な情報を確実に取得するバランスの最適化。

4. **動的ロバスト性**: 未知のツール、ツール障害、変動する知識可用性への対応。適応的訓練による環境変化への耐性の構築。

5. **環境間の一般化**: 不慣れなまたは変化するコンテキストでの性能の確保。訓練環境とデプロイ環境のギャップを埋める汎化能力の獲得。

## 備考

- System 1/System 2のデュアルプロセス理論をRAGに適用した独自の分類枠組みは、実務者がアーキテクチャ選択を行う際の直感的ガイドとして高い価値がある
- DeepResearcherの創発的行動（計画、交差検証、自己省察）はRL訓練の結果として自然に出現したものであり、エージェンティック行動の設計原理に関する重要な知見を提供する
- 2025年に集中するRL+RAG手法の急増は、この分野が急速な変革期にあることを示している
- 産業応用への焦点が明確であり、企業情報システム、科学報告、技術サポート等の実用的文脈での適用指針を提供している
- RAGFlow、Haystack、LangChain-Chatchat、LightRAG、R2R、FlashRAGなどの具体的な産業システムへの言及が実務的参照価値を高めている
- 事前定義推論の4カテゴリ（ルート/ループ/ツリー/ハイブリッド）は、RAGアーキテクチャの設計空間を明確に整理している
