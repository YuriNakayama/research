# AutoGen: Enabling Next-Gen LLM Applications via Multi-Agent Conversation

- **Link**: https://arxiv.org/abs/2308.08155
- **Authors**: Qingyun Wu, Gagan Bansal, Jieyu Zhang, Yiran Wu, Beibin Li, Erkang Zhu, Li Jiang, Xiaoyun Zhang, Shaokun Zhang, Jiale Liu, Ahmed Hassan Awadallah, Ryen W. White, Doug Burger, Chi Wang
- **Year**: 2023
- **Venue**: arXiv preprint (cs.AI)
- **Type**: Academic Paper (Framework / Multi-Agent System)

## Abstract

AutoGen is an open-source framework that enables development of LLM applications using multiple agents that can converse with each other to accomplish tasks. AutoGen agents are customizable, conversable, and seamlessly allow human participation. They can operate in various modes that employ combinations of LLMs, human inputs, and tools. Using AutoGen, developers can also flexibly define agent interaction behaviors. Both natural language and computer code can be used to program flexible conversation patterns for different applications. AutoGen serves as a generic infrastructure for building diverse applications of various complexities and LLM capacities. Empirical studies demonstrate the effectiveness of the framework in many example applications, with domains ranging from mathematics, coding, question answering, operations research, online decision-making, entertainment, etc.

## Abstract（日本語訳）

AutoGenは、複数のエージェントが相互に会話してタスクを達成するLLMアプリケーションの開発を可能にするオープンソースフレームワークである。AutoGenエージェントはカスタマイズ可能で会話能力を持ち、シームレスに人間の参加を許容する。LLM、人間の入力、ツールの組み合わせを用いた様々なモードで動作可能である。開発者はエージェントの相互作用動作を柔軟に定義でき、自然言語とコードの両方を用いて異なるアプリケーション向けの柔軟な会話パターンをプログラムできる。数学、コーディング、質問応答、オペレーションズリサーチ、オンライン意思決定、エンターテインメント等の多様なドメインにおける実証研究で有効性が示されている。

## 概要

本論文は、LLMベースのマルチエージェント会話フレームワーク「AutoGen」を提案する。中核となる設計パラダイムは**Conversable Agents（会話可能エージェント）** と**Conversation Programming（会話プログラミング）** であり、複数エージェント間の柔軟な対話を通じて複雑なタスクを解決する。

主要な貢献：

1. **Conversable Agentの設計**: LLM、ツール、人間の入力を統合的に扱える汎用エージェント基盤の提案
2. **Conversation Programmingパラダイム**: 自然言語とコードの両方でエージェント間の会話パターンを柔軟に定義可能な新しいプログラミングパラダイム
3. **3種類のエージェントタイプ**: ConversableAgent、AssistantAgent、UserProxyAgentによる階層的エージェント設計
4. **6ドメインにわたる実証評価**: 数学、コーディング、質問応答、オペレーションズリサーチ、オンライン意思決定、エンターテインメントでの有効性を実証
5. **オープンソース公開**: GitHubでのフレームワーク公開と活発なコミュニティ形成

## 問題と動機

- **単一エージェントの限界**: 単一のLLMエージェントでは複雑なタスク（数学的推論、コード生成と実行の連携等）に対応しきれない場面が多い
- **既存フレームワークの柔軟性不足**: LangChain等の既存ツールはエージェント間の会話パターンを柔軟に制御する仕組みが限定的であった
- **人間の参加統合の困難さ**: LLMエージェントのワークフローに人間の入力やフィードバックを自然に組み込むことが技術的に困難
- **ツール統合の複雑性**: コード実行、Web検索、API呼び出し等の外部ツールをエージェントの会話フローにシームレスに統合する汎用的手法が不足
- **多様なアプリケーションへの対応**: 異なるドメインやタスク要件に対して、統一的なフレームワークで対応できる汎用性が求められていた

## 提案手法

### 1. Conversable Agent（会話可能エージェント）

AutoGenの基礎となるエージェント抽象で、以下の能力を統合的に持つ：

- **LLMバックエンド**: GPT-4等のLLMを用いたテキスト生成・推論
- **ツール/コード実行**: Pythonコードの動的実行、外部API呼び出し
- **人間入力受付**: ワークフロー内で人間のフィードバックを受け取る機能

### 2. 3種類のビルトインエージェント

**ConversableAgent**（基底クラス）:
- 全エージェントの基底となる抽象クラス
- メッセージ送受信、状態管理、応答生成の基本機能を提供

**AssistantAgent**:
- LLMをバックエンドとするAIアシスタント
- システムプロンプトにより専門的な役割（コーダー、数学者等）を付与
- コード生成・分析・推論タスクに特化

**UserProxyAgent**:
- 人間ユーザーの代理として機能するエージェント
- コード実行環境を内蔵し、AssistantAgentが生成したコードを実行
- 人間のフィードバックをワークフローに注入するインターフェース

### 3. Conversation Programming（会話プログラミング）

エージェント間の対話パターンを定義するパラダイム：

- **2エージェント会話**: 最もシンプルなパターン。AssistantAgentとUserProxyAgentの1対1対話
- **逐次会話チェーン**: 複数の2エージェント会話を順次接続し、前の会話結果を次に渡す
- **グループチャット**: GroupChatManagerが複数エージェントの発言順序を制御する多者間対話
- **ネスト会話**: 会話内で別の会話を再帰的に呼び出す階層的パターン

## 実験と評価

### 数学的推論（MATHデータセット）

AutoGenは2エージェント構成（AssistantAgent + UserProxyAgent）でMATHデータセットに対し**69.48%**の正答率を達成。GPT-4のバニラ使用（52.85%）やPlugin使用（55.25%）を大幅に上回った。コード実行によるフィードバックループが精度向上に寄与している。

### オンライン意思決定（ALFWorld）

ALFWorldベンチマークにおいて**98%**の成功率を達成。ReAct（78%）やReflexion（94%）を上回り、マルチエージェント会話による自己修正メカニズムの有効性を示した。

### マルチホップ質問応答

マルチホップQAタスクにおいてF1スコア**66.65%**を達成。2エージェント構成（59.17%）に対し、グループチャットパターンによる複数エージェントの協調が精度向上に貢献。

### 主要な知見

1. **コード実行フィードバック**: コード生成→実行→エラー修正のループがLLMの推論能力を大幅に強化
2. **柔軟な会話パターン**: タスクに応じた最適な会話パターンの選択が性能に直結
3. **人間参加の効果**: 適切なタイミングでの人間フィードバックが品質を保証しつつ、自動化を最大化

## 備考

- AutoGenは2023年公開以降、Microsoft Research発のオープンソースプロジェクトとして急速にコミュニティが拡大し、後続のAutoGen Studio等の派生プロジェクトを生み出している
- 論文は43ページ（本文10ページ、参考文献3ページ、付録30ページ）と大規模であり、6つの応用ドメインの詳細な実験設定と結果が付録に記載されている
- BabyAGI、CAMEL、MetaGPT等の同時期フレームワークと比較して、会話プログラミングによる柔軟性とコード実行統合が主要な差別化要因
- GroupChatManagerによるグループチャットは、発言順序の制御が性能に大きく影響するため、適切な設計が重要
- 後続研究（AutoGen Studio、AG2等）では、ノーコードツールやより高度なオーケストレーション機能が追加されている
