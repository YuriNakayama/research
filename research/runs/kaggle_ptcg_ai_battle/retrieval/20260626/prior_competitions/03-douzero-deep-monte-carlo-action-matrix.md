# DouZero — 巨大・可変行動空間カードゲームを攻略するDeep Monte Carloと行列行動エンコード

## 概要

本レポートは Zha et al. (2021) *"DouZero: Mastering DouDizhu with Self-Play Deep RL"* (arXiv:2106.06135, ICML 2021) と公式実装 (kwai/DouZero) を精読する。闘地主(DouDizhu)は **3人・協調と競争が混在・不完全情報・巨大状態空間・ターンごとに合法手が激変する最大10^4の行動空間** を持つ。CFR は3人非ゼロ和で不適、DQN は過大評価と max 演算の計算コスト、policy gradient は行動特徴を活用できず汎化に劣る。DouZero は **Deep Monte Carlo (DMC) + カード行列による行動エンコード + 並列アクター self-play** で、4GPU・数日の学習で既存 AI を全て凌駕し **Botzone 344体中1位(Elo 1625.11)** を達成した。PTCG の「巨大行動空間・合法手 index・不完全情報」に最も直接的な技術テンプレートである。

## 手法の要点

- **Deep Monte Carlo (DMC)**: 古典 MC(エピソード平均リターンで Q(s,a) を推定)を NN 関数近似に拡張。手順は (1) 方策でエピソード生成、(2) 各 (s,a) を割引累積報酬で更新、(3) π(s)=argmax_a Q(s,a)。Q テーブルを NN で置換し MSE で更新。**ブートストラップしない**ため過大評価バイアスがなく、長ホライズン・疎報酬でもエピソード長に収束が左右されない。MC の高分散は**並列化で大量サンプルを生成**して緩和。
- **カード行列による行動・状態エンコード**: 各カード組合せを **4×15 の one-hot 行列**(列=13ランク+2ジョーカー、行=その枚数)で表現。これにより `3KKK`(良いキッカー選択)で得た知識を未見の `3JJJ` にも**汎化**できる。policy gradient と異なり、**頻繁に出現しない行動も行動特徴で推論可能**。
- **合法手の扱い**: 27,472 通りの組合せのうち各局面で合法な部分集合のみを Q ネットに通して評価(全行動を走査しない)。図1の例では特定手札に **391通りの合法手**。実装上は「合法手だけイテレート」するため、巨大行動空間でも実効計算量が抑えられる。
- **ネットワーク**: 履歴手を **LSTM** で符号化、行動・状態の各カード行列と連結し **6層 MLP(隠れ512)** で state-action の Q 値を出力。
- **並列アクター**: 3ポジション(Landlord/上家/下家)ごとに global Q ネット1つ + 多数の actor プロセス(48コア・4×1080Ti で45 actor)。各 actor が局所ネットを定期同期し軌跡を生成、learner が MSE 更新。

## 主要な結果や知見

- **学習効率**: 30日訓練だが、**SL を2日で、CQN とヒューリスティクスを半日で、2ヶ月学習の DeltaDou を10日で超える**。単一サーバ(48コア+4GPU)で完結。
- **対戦成績**: DeltaDou に WP 0.586/ADP 0.258、SL に 0.659/0.700、全ベースラインを支配。Botzone で344体中1位。
- **推論速度**: DeltaDou(MCTS)・CQN(カード分解)より**桁違いに高速**(1ステップ1回の forward のみ)。実応用・大量 self-play 生成に有利。
- **DMC vs 代替**: SARSA(TD版)はわずかに劣り、Actor-Critic は失敗。「行動特徴を critic に足すだけでは巨大行動空間問題は解けない」。
- **人間超え**: 訓練5日までは人間棋譜と精度が一致、その後は精度が下がる = **人間が見つけられない新戦略を self-play で発見**。

## 本コンペ(PTCG AI Battle)への応用

1. **行動の「行列/特徴エンコード」で巨大行動空間に対処**: PTCG はカード×対象×効果で合法手が爆発し、ターンごとに激変する。DouZero の核心は**行動を index ではなく特徴行列で表現**し未見行動へ汎化する点。PTCG でも「カードID・種類・コスト・効果」を多チャンネル特徴にエンコードすれば、稀な手や新しい組合せにも頑健な方策が学べる。
2. **DMC は探索不要・高速推論で10分制約に最適**: MCTS や CFR は PTCG の巨大状態で破綻しやすい。DouZero は**推論が1回の forward**で済むため、1プレイヤー10分・1ターン数秒という制約に圧倒的に向く。学習は重いが**推論は軽い**という非対称性が本コンペの提出形式(deck.csv + agent())に合致。
3. **合法手だけを評価するパターン**: 「全行動を走査せず合法部分集合のみ Q 評価」は PTCG の合法手 index 設計に直結。合法手リストを受け取り各手を特徴化して NN に通し argmax を返す、という DouZero 型の agent() が素直な実装。
4. **並列 self-play + LSTM 履歴符号化**: 部分観測の PTCG では、これまでに公開された相手の手・場のカードを LSTM/Transformer で系列符号化することで隠れ状態の belief を暗黙的に持てる。DouZero の actor-learner 構成は手元 GPU 規模(数枚)でも回せる現実的レシピ。
5. **MC の疎報酬耐性**: PTCG も報酬は基本「勝/敗」の疎報酬。DMC はブートストラップしないため疎報酬・長ホライズンに強く、報酬整形に頼りすぎず学習できる利点がある。
