## スコープ

gather 段階の「retrieval 推奨」上位 4 本について、arXiv の abstract ページおよび全文（HTML / PMC 版）を実際に取得して精読した詳細レポートである。対象読者は、数ヶ月に一度の低頻度マーケティング施策（クーポン・販促メール）を運用し、施策ごとに対象ユーザー・訴求内容・クーポン額が異なるために 1 施策あたりのデータが薄いという課題を抱えるデータサイエンティストである。

医療統計・計量経済の語彙は `trial` / `site` / `environment` → `campaign`、`patient` → `user`、`treatment arm` → `クーポン額水準` と読み替えて評価している。

## 読む順序

**#01 → #04 → #03 → #02** を推奨する。gather の推奨順（01 → 11 → 02 → 06）から #02 を最後に下げ、#06（本 index の #04）を繰り上げている。理由は下記の通り。

1. **[#01](01-comparison-of-methods-that-combine-multiple-randomized-trials.md) を最初に読む（地図として）。** 本課題と設定が最も近く、手法空間の全体像を「単一試験手法 × 統合方式」の二軸で与える。ここで得られる 3 点が以降すべての前提になる。(a) 完全プールを棄却してよい実証的根拠、(b) 最も軽い勝ち筋（causal forest + 施策指示変数）がすでに存在すること、(c) **4 試験 $N\approx1{,}850$ を統合してなお個人 CATE の信頼区間がすべてゼロを含んだ**という不確実性の現実。(c) は本クラスタ全体に対する冷や水であり、統合の目的を個人単位から意思決定粒度へ引き下げる判断を最初に迫る。

2. **[#04](04-bayesian-hierarchical-models-with-calibrated-mixtures-of-g-priors.md) を次に読む（#01 と対で）。** 著者 Hwanhee Hong が #01 と共通で、実データも同じ大うつ病 4 RCT である。#01 が「IPD メタ分析は関数形が合えば良好、非線形では notably worse」と外側から限界を示しているため、**両者を対で読むと本手法の適用条件が正確に把握できる**。施策ごとのサンプルサイズ不揃いという本課題固有の構造に対する直接的な解答であり、CMG が優位を示した条件（高い試験間ばらつき・高い疎性・弱い修飾効果・相関した計画行列）はマーケティングデータの記述そのものである。

3. **[#03](03-bayesian-hierarchical-adjustable-random-partition.md) を三番目に読む（本課題の中核の問いへ）。** 「どの施策同士をプールしてよいか」に正面から答える。#04 の $f(n_i)$ が調整するのは**サイズによる重み**であるのに対し、#03 が調整するのは**類似性による重み**であり、両者は直交する別問題である。この対比は #04 を読んだ後でこそ明確になる。ただし読む際は二つの限界を先に押さえること。(a) タイトルの通り adaptive enrichment trial 前提であり、逐次的な標本配分という主要価値の一部は事後統合である本課題では使えない、(b) 尤度に共変量が入らないため施策別平均効果しか出ず、単独では本課題の「異質処置効果」の要求を満たさない。

4. **[#02](02-transfer-estimates-for-causal-effects-across-heterogeneous-sites.md) を最後に読む（射程は最も遠いが最も遠い）。** 実績データのない新規施策に届く唯一の設計であり、C3（実績ゼロ施策の予測）への接続経路として本クラスタから最も遠くまで届く。一方で本課題への障害も最大である。仮定 3.2（移送先が潜在アウトカムと独立に一様ランダムに選ばれる）は、「反応しそうな層を狙う」というターゲティングの本質と構造的に衝突する。また収束レートの $1/G$ 項は施策数が少ない限り消えず、**「1 施策あたりのデータを増やす」方向では解決しないこと**を明示する。実装コストも本クラスタ最高。上位 3 本で足場を固めてから、$G$ を稼ぐ設計（施策 × セグメント等）が可能かの判断とセットで読むべきである。

## gather 段階からの訂正

精読により、gather の記述に 2 点の誤りが判明した。

- **#03（gather #11 BHARP）のタイトル。** 正しくは "...Bayesian Hierarchical Adjustable Random Partition **in Adaptive Enrichment Trials**" であり、汎用 HTE 手法ではなく特定の試験デザイン向けの手法である。この差は適用可能性の評価に直接効く。
- **#02（gather #02 Menzel）の実証規模。** gather は「5 サイトという少数からの移送で利得を実証」としていたが、実際は**5 研究・640 サイト**である。$G=640$ で機能する手法が施策数一桁で機能する保証はなく、「少数からの移送の期待値の目安になる」という gather の示唆は成立しない。

## 総括: 4 本を通じて見える構図

- **統合の方式は既に整理されており、最も軽い選択肢が最良水準にある。** #01 の「causal forest + 施策指示変数つきプール」は施策 ID をカテゴリ変数として 1 本足すだけで、主要シナリオで最良。まずこれを実装し、完全プールとの MSE 差を自社データで確認するのが最短経路である。
- **本当の難所は二つに分解される。** 「サイズの不揃い」（#04 が $f(n_i)$ で解く）と「類似性による借用の可否」（#03 が分割推定で解く）は直交する別問題であり、両方に対処が要る。#04 の階層構造は全施策を交換可能と見なす点で #03 が指摘する over-shrinkage を免れない。
- **不確実性の壁は統合では消えない。** #01 の実データで $N\approx1{,}850$ を統合してなお個人 CATE の CI がすべてゼロを含み、#02 の収束レートには $1/G$ 項が残る。**施策数そのものが少ないという制約は、手法では解けない**。統合の現実的な目標は「個人別の効果を当てる」ではなく「セグメント別・クーポン額別の意思決定の精度を上げる」に置くべきである。
- **共変量を扱えるかで手法が分かれる。** #01（causal forest）と #04（線形交互作用）は共変量を扱い、#03 は扱わない。#03 を使うなら #01 系との併用が前提になる。
- **どの手法もクーポン額という連続処置強度を直接扱っていない。** 4 本すべてが二値処置（または離散アーム）を前提とする。額を共変量に入れるか、額水準をアームとみなすかは自前の設計判断であり、本クラスタの文献からは答えが出ない。ここは残された課題である。
