# Open Bandit Dataset and Pipeline: Towards Realistic and Reproducible Off-Policy Evaluation

- **Link**: https://arxiv.org/abs/2008.07146
- **Authors**: Yuta Saito, Shunsuke Aihara, Megumi Matsutani, Yusuke Narita
- **Year**: 2020 (arXiv v1) / 2021 (採択)
- **Venue**: NeurIPS 2021 Datasets and Benchmarks Track
- **Type**: データセット・ソフトウェア論文 / ベンチマーク (Off-Policy Evaluation)

---

## Abstract (English)

> Off-policy evaluation (OPE) aims to estimate the performance of hypothetical policies using data generated by a different policy. Because of its huge potential impact in practice, there has been growing research interest in this field. There is, however, no real-world public dataset that enables the evaluation of OPE, making its experimental studies unrealistic and irreproducible. With the goal of enabling realistic and reproducible OPE research, we present the Open Bandit Dataset, a public logged bandit dataset collected on a large-scale fashion e-commerce platform, ZOZOTOWN. Our dataset is unique in that it contains a set of multiple logged bandit datasets collected by running different policies on the same platform. This enables experimental comparisons of different OPE estimators for the first time. We also develop Python software called the Open Bandit Pipeline to streamline and standardize the implementation of batch bandit algorithms and OPE. Our open-source pipeline will contribute to the reproducibility and reliability of OPE research and practice and provide the fundamental infrastructure for practical applications. In addition to the software, we provide extensive benchmark experiments of existing OPE estimators using our dataset and pipeline. Our experiments open up essential challenges and new avenues for future OPE research.

## Abstract (日本語訳)

Off-Policy Evaluation (OPE) は、あるポリシーによって生成されたデータを用いて、別の（仮想的な）ポリシーの性能を推定することを目的とする。実務上のインパクトが大きいため研究が急増しているが、OPE を評価できる実世界の公開データセットが存在せず、実験研究は非現実的で再現不能であった。本論文では現実的かつ再現可能な OPE 研究を可能にするため、大規模ファッション EC プラットフォーム ZOZOTOWN 上で収集した公開ログ・バンディットデータセット「Open Bandit Dataset」を提示する。本データセットは、同一プラットフォーム上で**複数の異なるポリシーを実際に走らせて収集した複数のログ・バンディットデータ**を含む点でユニークであり、これにより異なる OPE 推定量の実験的比較が初めて可能になる。さらに、バッチ・バンディットアルゴリズムと OPE の実装を標準化する Python ソフトウェア「Open Bandit Pipeline (OBP)」を開発した。また既存 OPE 推定量の広範なベンチマーク実験を提供する。

## Overview

本研究の貢献は 3 つである。(1) 実世界の**公開データセット** Open Bandit Dataset (OBD)、(2) 標準化 Python ソフトウェア Open Bandit Pipeline (OBP)、(3) 既存 OPE 推定量の**ベンチマーク**。従来の OPE 研究は多クラス分類データをバンディットに変換した半合成実験に依存しており、真のポリシー価値・真のロギングポリシーが分からず再現性・現実性に欠けていた。OBD は同一プラットフォーム上で **Random ポリシー**と **Bernoulli Thompson Sampling (TS) ポリシー**の 2 種を実際に運用してログを収集しているため、一方を「ロギングポリシー」、他方を「評価ポリシー」とみなし、後者の**オンポリシー推定値を真値（ground truth）**として OPE 推定量の精度を測定できる。これが「OPE 推定量そのものを評価できる」初の枠組みである。

---

## Problem（課題リスト）

- OPE を評価できる**実世界公開データが存在しない** → 実験が非現実的・再現不能。
- 多クラス分類変換ベースの半合成評価は、真のロギングポリシー・真のポリシー価値が人工的で、実運用と乖離。
- OPE 推定量やバンディットアルゴリズムの実装が研究者ごとにバラバラで、**再現性・比較可能性が低い**。
- 推定量選択（どの OPE 推定量・ハイパーパラメータを使うか）に関する実務ガイドラインが不足。

---

## Proposed Method（中核アイデアと手順）

**中核アイデア**: 「複数ポリシーを実運用したログ」を公開することで、評価ポリシーのオンポリシー価値を真値として OPE 推定量を検証できるようにする。加えて標準化ソフトウェア OBP で推定量・アルゴリズムを共通 API 化する。

### 手順（ベンチマーク・プロトコル）

1. データセットからロギングポリシー（例: Bernoulli TS）で集めたログ $\mathcal{D}$ を取得。
2. 評価ポリシー（例: Random）で集めたログから、その**オンポリシー価値** $V(\pi_e)$ を経験平均で推定し、これを真値とする。
3. 各 OPE 推定量 $\hat{V}$ を $\mathcal{D}$ に適用して $\hat{V}(\pi_e;\mathcal{D})$ を算出。
4. 真値との誤差（相対推定誤差 / RMSE）をブートストラップ反復で評価。
5. 推定量間・ハイパーパラメータ間で精度を比較。

### Key Formulas

ポリシー価値（期待報酬）:

$$V(\pi) = \mathbb{E}_{p(x)\pi(a|x)p(r|x,a)}[r]$$

Direct Method (DM):

$$\hat{V}_{\mathrm{DM}}(\pi;\mathcal{D},\hat{q}) = \frac{1}{n}\sum_{i=1}^{n}\sum_{a\in\mathcal{A}}\pi(a|x_i)\,\hat{q}(x_i,a)$$

Inverse Probability Weighting (IPW/IPS):

$$\hat{V}_{\mathrm{IPW}}(\pi;\mathcal{D}) = \frac{1}{n}\sum_{i=1}^{n} w(x_i,a_i)\, r_i,\quad w(x,a)=\frac{\pi(a|x)}{\pi_0(a|x)}$$

Self-Normalized IPW (SNIPW):

$$\hat{V}_{\mathrm{SNIPW}}(\pi;\mathcal{D}) = \frac{\sum_{i=1}^{n} w(x_i,a_i) r_i}{\sum_{i=1}^{n} w(x_i,a_i)}$$

Doubly Robust (DR):

$$\hat{V}_{\mathrm{DR}}(\pi;\mathcal{D},\hat{q}) = \frac{1}{n}\sum_{i=1}^{n}\Big[ \sum_{a}\pi(a|x_i)\hat{q}(x_i,a) + w(x_i,a_i)\big(r_i-\hat{q}(x_i,a_i)\big) \Big]$$

## Algorithm（擬似コード）

```
Input: logged data D (behavior policy π0), evaluation policy πe,
       list of OPE estimators {V̂_k}, ground-truth value V(πe)
Output: accuracy (RMSE) of each estimator

# --- OBP workflow ---
1. dataset = OpenBanditDataset(behavior_policy="bts", campaign="all")
2. bandit_feedback = dataset.obtain_batch_bandit_feedback()
3. (optional) train q̂ via regression on {(x,a,r)}
4. for b in 1..B (bootstrap):
5.     D_b = resample(bandit_feedback)
6.     for each estimator V̂_k:
7.         est_k[b] = V̂_k(πe; D_b, q̂)
8. for each estimator V̂_k:
9.     RMSE_k = sqrt( mean_b (est_k[b] - V(πe))^2 )
10. return {RMSE_k}
```

## Architecture / Process Flow

```
                Open Bandit Dataset (ZOZOTOWN)
        ┌──────────────┬──────────────┬───────────────┐
        │  Random log  │ Bernoulli TS │  (per campaign)│
        └──────┬───────┴──────┬───────┴───────────────┘
               │              │
   behavior policy π0    evaluation policy πe (on-policy value = ground truth)
               │
     ┌─────────▼──────────────────────────────┐
     │  Open Bandit Pipeline (OBP)             │
     │  ┌──────────┐ ┌─────────┐ ┌───────────┐ │
     │  │ dataset  │→│ policy  │→│    ope    │ │
     │  │ module   │ │ module  │ │  module   │ │
     │  └──────────┘ └─────────┘ └─────┬─────┘ │
     └───────────────────────────────── │──────┘
                                         ▼
                DM / IPW / SNIPW / DR / SNDR / Switch-DR / DRos
                                         ▼
                     RMSE vs ground-truth → estimator ranking
```

## Experiments & Evaluation

### Setup
- データ: Open Bandit Dataset（ALL / Men's / Women's の 3 キャンペーン）。
- ロギングポリシーと評価ポリシーの組（例: Bernoulli TS を behavior、Random を evaluation、あるいはその逆）。
- 真値: 評価ポリシーのオンポリシー価値（該当ポリシーで実際に集めたログの経験平均）。
- 評価指標: 相対推定誤差 / RMSE。B=200（ないし多数）ブートストラップ反復。

### Main Results
- **DRos（Optimistic Shrinkage）**がハイパーパラメータ自動調整下で概ね最小誤差を達成し、IPW/DM/DR より 30〜60% 低い誤差（ar5iv 抽出値）。
- DM は回帰モデルの誤指定に強く依存し、キャンペーンによっては誤差が大きい。
- IPW はロギングと評価ポリシーの乖離が大きいと分散が増大。
- **推定量とそのハイパーパラメータ選択が実務上のキモ**であることをベンチマークが示す。

### Ablation
- サンプル数（例: n=300,000 など）を変えると推定量の相対順位が変化し得ることを示す。厳密なアブレーション表の数値は原論文 Figure を参照（本文抽出では一部のみ確認、確認できない値は「記載なし」）。

## 本テーマへの適用可能性

本テーマ（クーポン/メール配信の低頻度キャンペーン最適化、A/B テストなしでログから新方針をオフライン評価）に対して、OBD/OBP は**評価基盤そのもの**として直接的に有用である。

- **OPE でのクーポン方針評価**: 過去のクーポン配信ログ（誰にどのクーポンを、どの確率で配ったか＝ロギングポリシー $\pi_0$、およびコンバージョン/購買を報酬 $r$）に対し、OBP の `ope` モジュールで IPW/DR/DRos を適用すれば、A/B テストを走らせずに新しいターゲティング方針 $\pi_e$（例: 高反応セグメントにクーポンを集中）の期待成果を推定できる。ロギング確率 $\pi_0(a|x)$ さえ記録しておけば、OBP の共通 API（`BanditFeedback` 形式）にログを流し込むだけで複数推定量を横並び比較できる。
- **推定量選択の指針**: OBD ベンチマークの示唆（DR / DRos の安定性、IPW の分散問題）は、配信頻度が低くログが少ないマーケ現場でそのまま効く。サンプルが少ないほど IPW の分散が致命的になるため、報酬回帰 $\hat q$ を併用する DR/DRos を第一候補にできる。
- **複数キャンペーンのプーリング**: OBD が「同一プラットフォーム上の複数ポリシーのログ」を扱う設計は、複数のクーポン/メールキャンペーンのログを 1 つの `BanditFeedback` に統合し、文脈 $x$ にキャンペーン属性を含めてプーリング評価する発想に直結する。キャンペーンごとにデータが薄くても、共通の文脈・行動埋め込みで束ねれば OPE の分散を抑えられる。
- **実装コストの低さ**: OBP は推定量・データ前処理・合成データ生成・回帰モデル学習を標準 API で提供するため、社内マーケログを OBP フォーマットに整形するだけで、DM/IPW/SNIPW/DR/Switch-DR/DRos を即座に試せる。オフライン評価パイプラインの雛形として最短距離。
