# 深層強化学習によるアリーナ・ドラフト方策の学習(Vieira et al.)

## 概要

Vieira, Tavares & Chaimowicz(SBGames 2020 / IEEE)は、CCGのアリーナ(ドラフト)モードを**ゲーム非依存のMDPとして定式化し、深層強化学習(PPO)でドラフト方策を学習**する手法を提案。テストベッドは LOCM(30ターン、各ターン3枚から1枚選択)。3つの状態表現/ネットワーク変種を比較し、学習方策が既存最良エージェントを上回ること、さらに「あるコンペ参加エージェントが本手法のドラフトを使うと10位→4位に上昇」することを示した。本研究の独自貢献は、カードIDではなく**カード特徴量(feature)で方策を学習**し「良いカードとはどういうものか」を学ぶ点と、過去ピックとのシナジーを明示的に扱った点。

## 手法の要点

- **MDP定式化**: 状態には2種。**S1(マルコフ的)** = これまでの全ピック + 現在の3枚選択肢(シナジー考慮可だが状態空間巨大)。**S2(非マルコフ的)** = 現在の3枚のみ(状態空間 |C|/k choose で小さいが過去ピック無視)。行動 A={1,...,k}(提示k枚から1枚)。報酬は終端のみ(勝/分/負=+1/0/−1、companion battle agent の対戦結果)、非終端は0。割引 γ=1。
- **3つのドラフトエージェント**: 
  - **History**: S1 + MLP(全過去ピックを列挙、シナジー考慮)。
  - **LSTM**: S2 + LSTM層(過去ピックを列挙せず内部状態で保持)。
  - **Immediate**: S2 + MLP(過去無視、最も単純)。
- **カード特徴量**: LOCMの15属性のうちカード種別・コスト・攻撃・防御・能力(6種のbool)・ヘルス/ドロー修飾を採用(IDと名前は除外)。数値は正規化、種別はone-hot。これにより未知カードへも汎化。
- **学習**: 自作のOpenAI Gym互換エンジン上でPPO(stable-baselines)。先攻有利を踏まえ先攻/後攻で別々に2インスタンスを訓練。self-play(過去版の相手と対戦)。1セッション=3万自己対戦エピソード、12チェックポイントで各1000試合評価。ハイパラはTPE(hyperopt)で6構成×50反復チューニング。
- **推論速度**: 訓練済みドラフタは**1ピックあたり1ミリ秒未満**(訓練は max-attack battler で約30分、greedy で約3時間)。

## 主要な結果や知見

- **意外にも Immediate(最も単純・過去無視)が全シナリオで最良**、次いで LSTM、History の順。3万エピソードの予算では「小さな状態空間の単純さ」が「過去ピック考慮能力」を上回った。LSTM は学習終盤も上昇傾向で、より長く訓練すれば伸びる余地。
- LOCMのカード/ルールが単純すぎて意味あるシナジー/アーキタイプが創発しにくい可能性、または companion battle agent がシナジーを活かせない可能性を著者は指摘。
- **対既存ドラフタ**: 提案手法は random / max-attack / コンペ上位の coac・closet-ai・icebox を全て上回り、max-attack battler 使用時は元の max-attack ドラフタ相手に**最大83.5%勝率**。差はほぼ全てp<0.005で有意。
- **実戦価値**: 本手法のドラフトを組み込んだ参加エージェントが CoG 2019 LOCM 大会で10位→4位に上昇。

## 本コンペ(PTCG AI Battle)への応用

- **「IDではなく特徴量で学ぶ」が約2000枚プールの鍵**: PTCGは巨大カードプール+頻繁な環境更新。カードを生のindexでなく**属性ベクトル(タイプ、HP、技ダメージ、エネルギーコスト、進化段階、特性フラグ等)で表現**して方策を学べば、未知/新カードへも汎化でき、obs_dict の select 候補をその場でベクトル化して評価する設計に直結する。
- **状態表現の単純さが効くという教訓**: History/LSTM より Immediate が勝った結果は、本コンペでも「全履歴(logs)を重く取り込むより、current の局面と select 候補に集中した軽量方策」がサンプル効率・推論速度の両面で有利になり得ることを示唆。10分制約下では特に重要で、まず軽量モデルから始めるべき。
- **先攻/後攻の分離学習**: LOCMで先攻有利に応じて方策を分けたのと同様、PTCGも先攻/後攻で最適戦略が変わる。obs_dict から手番情報を読み、先攻/後攻別の方策(または条件付き入力)を用意すると性能が上がる。
- **報酬の与え方**: 終端勝敗のみのスパース報酬+γ=1(全ピックが等価)という設計は、本コンペでデッキ全体の良し悪しを学ぶ際の素直な出発点。ただしPTCGは1試合が長く高分散なので、複数試合平均や(Q-DeckRec/DSA-MEのような)中間勝率近似でシグナルを密にする工夫が併用価値あり。
- **1ms未満の推論**: 訓練後の方策が極めて軽い点は、対戦中(Simulation部門)に毎手呼び出しても10分制約を圧迫しないことを意味し、本コンペの実装方針(重い学習はオフライン、推論は軽量NN)を裏づける。
