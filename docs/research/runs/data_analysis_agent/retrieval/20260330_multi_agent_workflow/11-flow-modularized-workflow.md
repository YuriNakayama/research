# Flow: Modularized Agentic Workflow Automation

- **Link**: https://arxiv.org/abs/2501.07834
- **Authors**: Boye Niu, Yiliao Song, Kai Lian, Yifan Shen, Yu Yao, Kun Zhang, Tongliang Liu
- **Year**: 2025
- **Venue**: ICLR 2025
- **Type**: Academic Paper (Framework / Workflow Automation)

## Abstract

With the rapid development of large language models, agents based on LLMs have gradually been applied to real-world tasks. Increasing task complexity has driven the evolution of multi-agent frameworks, which focus on workflow adjustment among agents. In this paper, we define workflows using activity-on-vertex graphs, enabling continuous refinement through dynamic subtask allocation. We emphasize modularity in workflow design, evaluating parallelism and dependency complexity to ensure efficient concurrent execution of subtasks, effective goal achievement, and enhanced error tolerance. Our framework supports dynamic workflow updates based on historical performance, improving task execution over iterative cycles. Empirical evaluations demonstrate the effectiveness and efficiency of our approach across various practical tasks.

## Abstract（日本語訳）

大規模言語モデルの急速な発展に伴い、LLMベースのエージェントは徐々に実世界のタスクに適用されるようになった。タスクの複雑化がマルチエージェントフレームワークの進化を促し、エージェント間のワークフロー調整に焦点が当てられている。本論文では、Activity-on-Vertex（AOV）グラフを用いてワークフローを定義し、動的なサブタスク割当による継続的な改善を可能にする。ワークフロー設計におけるモジュール性を重視し、並列性と依存関係の複雑度を評価することで、サブタスクの効率的な並行実行、効果的な目標達成、エラー耐性の向上を保証する。フレームワークは過去の実行履歴に基づくワークフローの動的更新をサポートし、反復サイクルを通じたタスク実行の改善を実現する。実証評価により、多様な実用タスクにおけるアプローチの有効性と効率性を実証する。

## 概要

本論文は、マルチエージェントシステムのワークフローをActivity-on-Vertex（AOV）グラフとして形式化し、モジュール性・並列性・依存関係の複雑度を数学的に定義する新しいフレームワーク「Flow」を提案する。ICLR 2025に採択された研究であり、動的ワークフロー更新と反復的改善を通じて、既存フレームワーク（AutoGen、MetaGPT等）を大幅に上回る性能を達成する。

主要な貢献：

1. **AOVグラフによるワークフロー形式化**: ワークフローをActivity-on-Vertex有向非巡回グラフとして数学的に定義し、厳密な分析基盤を提供
2. **並列性と依存関係複雑度の指標**: ワークフローのモジュール性を定量的に評価する2つの新指標を提案
3. **Theorem 3.1**: ワークフローのモジュール性と実行効率の関係を理論的に証明
4. **動的ワークフロー更新**: 過去の実行履歴に基づくワークフローの自動改善メカニズム
5. **大幅な性能改善**: 平均成功率93%を達成し、AutoGen（66.7%）、MetaGPT（71%）を大幅に上回る

## 問題と動機

- **ワークフロー設計の理論的基盤の欠如**: 既存のマルチエージェントフレームワーク（AutoGen、MetaGPT、CrewAI等）はワークフローを経験的に設計しており、最適性の数学的保証がない
- **静的ワークフローの限界**: 事前定義された固定ワークフローは、タスクの変動や実行時エラーへの適応性が低い
- **並列性の未活用**: 多くのフレームワークがサブタスクを逐次実行しており、並列実行可能なサブタスクの同時処理が十分に活用されていない
- **モジュール性の欠如**: ワークフローのコンポーネント再利用性や独立した改善が困難
- **エラー伝播の問題**: 一つのサブタスクの失敗がワークフロー全体に波及するカスケード障害への対処が不十分

## 提案手法

### 1. AOVグラフによるワークフロー形式化

ワークフローを有向非巡回グラフ（DAG）として定義：

$$G = (V, E)$$

ここで：
- $V = \{v_1, v_2, ..., v_n\}$: 各頂点がサブタスクを表す
- $E \subseteq V \times V$: 辺がサブタスク間の依存関係を表す
- 辺 $(v_i, v_j) \in E$ は $v_i$ の完了が $v_j$ の開始の前提条件であることを示す

### 2. 並列性（Parallelism）指標

ワークフローの並列実行度を定量化する指標：

$$P(G) = \frac{\sum_{i=1}^{L} |S_i|}{L}$$

ここで $S_i$ はレベル $i$ で同時実行可能なサブタスクの集合、$L$ はクリティカルパスの長さ。$P(G) > 1$ であれば並列実行による効率化が可能。

### 3. 依存関係複雑度（Dependency Complexity）指標

ワークフロー内のサブタスク間依存の複雑さを評価：

$$D(G) = \frac{|E|}{|V| \cdot (|V| - 1) / 2}$$

$D(G)$ が低いほどモジュール性が高く、サブタスクの独立した改善・置換が容易。

### 4. Theorem 3.1（モジュール性と効率の関係）

**定理**: 並列性 $P(G)$ が高く依存関係複雑度 $D(G)$ が低いワークフローは、実行効率とエラー耐性の両面で優れる。

具体的には：
- 並列性の向上は実行時間の短縮に直結
- 依存関係の削減はエラー伝播範囲を限定
- 両指標の最適化はワークフロー全体の成功率を向上

### 5. 動的ワークフロー更新

過去の実行履歴に基づくワークフローの自動改善：

1. **実行ログ分析**: 各サブタスクの成功/失敗率、実行時間を記録
2. **ボトルネック特定**: 低成功率・長時間のサブタスクを自動検出
3. **ワークフロー再構成**: サブタスクの分割、並列化、依存関係の再編成を動的に実行
4. **反復改善**: 更新後のワークフローを再実行し、継続的に最適化

## 実験と評価

### 実験設定

- 複数のタスクカテゴリ（コード生成、データ分析、文書作成、複合タスク）で評価
- AutoGen、MetaGPTをベースライン比較対象として使用
- 自動評価（成功率、実行時間）と人間評価（4段階スコア）の両方を実施

### 定量的結果

- **平均成功率93%**: AutoGen（66.7%）に対して+26.3ポイント、MetaGPT（71%）に対して+22ポイントの大幅改善
- **複合タスクでの優位性**: 最も複雑な複合タスクにおいて30%以上の改善を達成。AOVグラフによるサブタスク分解と並列実行の効果が顕著
- **実行時間の削減**: 並列実行により、逐次実行比で最大70%の時間短縮

### 人間評価結果

- **総合スコア3.54/4.0**: 人間評価者によるタスク完了度、出力品質、エラー耐性、効率性の4項目で一貫して高評価
- 特にエラー耐性（3.45/4.0）で他手法との差が大きく、動的ワークフロー更新による自己修復能力が評価された

### 動的更新の効果

- 反復サイクルを重ねるごとに成功率が着実に向上
- 初期サイクルでの大幅改善と、後期サイクルでの漸進的改善パターンが観察
- ボトルネックサブタスクの自動検出と再構成が改善の主要因

### Theorem 3.1の実証

- 並列性指標 $P(G)$ が高いワークフロー構成ほど実行時間が短い（相関係数 -0.87）
- 依存関係複雑度 $D(G)$ が低いワークフロー構成ほど成功率が高い（相関係数 -0.82）
- 理論的予測と実験結果が高い整合性を示し、定理の実用的妥当性を確認

### 主要な知見

1. **形式化の価値**: AOVグラフによる数学的形式化により、ワークフローの最適化が体系的に可能
2. **動的更新の有効性**: 静的ワークフローに対して動的更新は一貫した性能改善をもたらす
3. **モジュール性と成功率の相関**: 並列性が高く依存関係が低いモジュール的ワークフローが最も高い成功率を達成
4. **スケーラビリティ**: タスクの複雑度が増すほど、Flowの他手法に対する優位性が拡大

## 備考

- ICLR 2025に採択された研究であり、マルチエージェントワークフローの形式化と最適化に関する理論的な基盤を提供する重要な貢献
- GitHubリポジトリ（tmllab/2025_ICLR_FLOW）でコードが公開されており、再現性が高い
- AOVグラフはプロジェクト管理（PERT/CPM）で広く使われる概念であり、これをマルチエージェントシステムに適用するアイデアは直感的かつ強力
- Theorem 3.1は並列性と依存関係複雑度の2指標でワークフロー品質を評価可能にし、設計時の定量的ガイドラインとして実用的
- AutoGen（66.7%）やMetaGPT（71%）に対する93%の成功率は大幅な改善であるが、タスクセットや評価条件の違いに注意が必要
- 動的ワークフロー更新はLLMの自己反省（self-reflection）能力に依存しており、基盤モデルの性能が結果に影響する
- 人間評価スコア3.54/4.0は高水準であるが、評価者数やタスク分布の詳細は論文の付録を参照する必要がある
