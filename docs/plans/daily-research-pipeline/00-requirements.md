# Daily Research Pipeline — 要件定義

## 背景と目的

現在のAuto Research Pipelineは、単一のリサーチプロンプトを毎日実行してレポートを生成する仕組みになっている。これを拡張し、複数のリサーチ領域を管理でき、CSVリストから毎日1件ずつ順番にリサーチを実行する「Daily方式」に移行する。

これにより：
- 複数のリサーチ領域（legal_tech等）を同時に管理できる
- 各領域ごとにリサーチ対象リストを事前に準備し、計画的にリサーチを進められる
- 全領域のレポートを1通のメールにまとめて通知できる

## ユーザーストーリー

- リサーチャーとして、リサーチ領域ごとにリサーチ対象リストをCSVで管理し、毎日1件ずつ自動的にリサーチを実行してほしい
- リサーチャーとして、全領域の当日レポートを1通のメールでまとめて受け取りたい
- リサーチャーとして、新しいリサーチ領域を簡単に追加・切り替えできるようにしたい

## 機能要件

### 1. ディレクトリ構造

```
docs/daily/
  <domain>/           # リサーチ領域（例: legal_tech）
    list/
      resources.csv   # リサーチ対象リスト（research-gatherスキルの出力を変換）
    report/
      20260329.md     # 日次レポート（YYYYMMDD形式）
      20260330.md
```

### 2. CSVリスト管理

- research-gatherスキルの出力（Markdownテーブル）を事前にCSV変換して配置
- CSVカラム構成:
  - `title`: リサーチ対象タイトル
  - `url`: リソースURL
  - `authors`: 著者
  - `year`: 発表年
  - `venue`: 掲載先
  - `summary`: 概要
  - `status`: 実行状態（`pending` / `done`）
- パイプライン実行時、`status=pending`の先頭行を1件選択してリサーチを実行
- 実行完了後、該当行の`status`を`done`に更新

### 3. リサーチ実行

- 各領域のCSVから未実行の先頭1件を取得
- 共通のプロンプトテンプレートを使用し、CSVの情報（タイトル、URL等）を埋め込んでClaude CLIで実行
- 出力を `docs/daily/<domain>/report/YYYYMMDD.md` として保存
- 全領域を順次実行（1回のパイプライン実行で全領域分）

### 4. メール通知

- 全領域のレポートを1通のメールにまとめて送信
- 各領域のレポートをMarkdown + PDF添付
- 既存のPR通知メールは廃止し、daily方式のレポート通知のみとする

### 5. 設定管理

- `research-config.yaml`を拡張し、複数のdaily領域を定義可能にする
- 領域の追加はYAMLに1エントリ追加するだけで完了する簡易さ

### 6. 既存機能の移行

- 既存の単一プロンプト実行方式を完全に廃止
- `research-config.yaml`の`research`セクション（`prompt_path`、`output_dir`等）を新方式に置き換え
- 不要になったファイル・コードは互換性を考慮せず削除

## 非機能要件

- **パフォーマンス**: 各領域のリサーチはClaude CLIのタイムアウト（30分）以内に完了すること
- **信頼性**: 1領域が失敗しても他の領域のリサーチは続行すること
- **拡張性**: 新しい領域の追加がYAML設定の変更のみで完了すること

## スコープ外

- CSVの自動生成（research-gatherスキルの実行はパイプライン外で事前に行う）
- research-gatherスキル出力からCSVへの自動変換ツール（初期段階では手動変換）
- リサーチ結果の品質評価・フィルタリング
- 複数レポートの統合・サマリ生成

## 用語集

| 用語 | 説明 |
|------|------|
| Daily方式 | CSVリストから毎日1件ずつリサーチを実行する新方式 |
| リサーチ領域（domain） | リサーチのテーマ分類（例: legal_tech） |
| リソースCSV | research-gatherスキル出力をCSV変換したリサーチ対象リスト |
| プロンプトテンプレート | CSVの情報を埋め込んでClaude CLIに渡す共通テンプレート |
