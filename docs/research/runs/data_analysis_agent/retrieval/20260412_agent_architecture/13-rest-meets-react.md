# ReST meets ReAct: Self-Improvement for Multi-Step Reasoning LLM Agent

- **Link**: https://arxiv.org/abs/2312.10003
- **Authors**: Renat Aksitov, Sobhan Miryoosefi, Zonglin Li, Daliang Li, Sheila Babayan, Kavya Kopparapu, Zachary Fisher, Ruiqi Guo, Sushant Prakash, Pranesh Srinivasan, Manzil Zaheer, Felix Yu, Sanjiv Kumar
- **Year**: 2023
- **Venue**: arXiv preprint (cs.CL)
- **Type**: Academic Paper

## Abstract

The paper addresses challenges in answering complex questions requiring multi-step reasoning and integrating external information. The researchers developed a ReAct-style agent enhanced through a ReST-like (Reinforced Self-Training) iterative training approach. Their method enables a fine-tuned small model that achieves comparable performance on challenging compositional question-answering benchmarks with two orders of magnitude fewer parameters after just two iterations. The approach combines knowledge retrieval with LLM reasoning using growing-batch reinforcement learning with AI feedback, implementing self-distillation for model refinement and enabling continuous self-improvement despite non-differentiable external interactions.

## Abstract（日本語訳）

本論文は、マルチステップ推論と外部情報の統合を必要とする複雑な質問への回答における課題に取り組む。研究者らはReST（Reinforced Self-Training）様の反復的訓練アプローチにより強化されたReActスタイルのエージェントを開発した。提案手法は、わずか2回の反復で、挑戦的な構成的質問応答ベンチマークにおいて、2桁少ないパラメータ数で同等の性能を達成するファインチューニングされた小規模モデルを実現する。本アプローチはAIフィードバックによる成長バッチ強化学習を用いて知識検索とLLM推論を組み合わせ、自己蒸留によるモデル改良を実装し、微分不可能な外部インタラクションにもかかわらず継続的な自己改善を可能にする。

## 概要

本研究は、ReActフレームワークに基づくLLMエージェントの自己改善手法を提案するものである。従来のReActエージェントは大規模なプロンプテッドモデルに依存していたが、本手法ではReST（Reinforced Self-Training）の反復的訓練アプローチを適用することで、大幅に小さなモデルへの知識蒸留を実現する。

主要な貢献は以下の通り：

1. **ReAct + ReST統合フレームワーク**: 外部ツール呼び出しを含むマルチステップ推論エージェントに対して、反復的な自己改善訓練を初めて適用
2. **効率的な知識蒸留**: PaLM 2-Lの性能を、2桁少ないパラメータ数のPaLM 2-XSモデルに蒸留することに成功
3. **AI フィードバックによる品質管理**: 人間のラベルなしに、LLMベースの報酬モデルを用いた軌跡の品質フィルタリングを実現
4. **データ品質の重要性の実証**: データ量よりもデータ品質が自己改善において重要であることを実験的に示した

データ分析エージェント研究の文脈において、本論文はエージェントの自己改善メカニズムの実践的な設計指針を提供する点で重要である。

## 問題と動機

- **大規模モデルへの依存**: 既存のReActエージェントはGPT-4やPaLM 2-Lなどの大規模モデルに依存しており、推論コストが高い
- **人間ラベルの取得困難**: マルチステップ推論の中間ステップに対する人間のアノテーションは高コストかつ困難
- **非微分可能な外部インタラクション**: 検索エンジンやAPIへのツール呼び出しを含むエージェントの軌跡は、標準的な勾配ベースの最適化が直接適用できない
- **自己改善の限界**: 単純なプロンプティングベースの自己修正では、性能改善が限定的であることが先行研究で示されている

## 提案手法

**ReST + ReAct 統合アプローチ（Iterative Self-Improvement for Tool-Using Agents）**

### ReActエージェントの設計

5段階の推論ステップで構成される検索エージェント：

1. **判断ステップ（Decision）**: 追加情報が必要かどうかを判断
2. **Web検索（Search）**: 外部検索APIへのクエリ実行
3. **要約（Summarization）**: 検索スニペットの処理・統合
4. **回答生成（Answer Generation）**: 初期回答のドラフト作成
5. **自己修正（Self-Revision）**: 関連性と根拠の2段階の検証チェック

「コードとしてのプロンプト」アプローチを採用し、指示を有効なPythonコードとしてフォーマットすることで、モデルに構造化された出力を促す。

### ReST様の反復訓練プロセス

外側-内側ループ構造のアルゴリズム：

**Growフェーズ**: 現在のポリシーを用いて多様な質問（HotpotQA、ELI5等から2,000問をサンプリング）に対する軌跡を生成

**Improveフェーズ**: LLMベースの報酬モデルを用いて軌跡をフィルタリング・ランク付けし、選択された例でファインチューニング

このプロセスを繰り返し、後の反復ではより高品質な合成データを使用して小規模モデルの訓練を行う。

### サンプリング戦略

- 温度: T=0.5で軌跡を生成
- ステップごとに複数サンプル（L/Sモデルで4、XSモデルで16）
- 最小パープレキシティによる選択を報酬モデルランキングで精緻化

### Figure 1: 自動評価と人間評価の相関

LLMベースの自動評価と人間評価のピアソン相関係数が0.98（p = 6.6 x 10^-8）を達成。自動評価の信頼性を強く支持する結果であり、大規模な自己改善ループの実用性を裏付けている。

### Figure 2: データ品質 vs データ量のトレードオフ

第1世代から第2世代への移行（データサイズは同程度だが品質が向上）で9%の精度向上が得られた一方、データ量のみの増加では同等の効果が得られなかった。データ品質がエージェント訓練において決定的に重要であることを示す。

### Figure 3: 軌跡数と性能の関係

質問あたりの軌跡数を増やすことによる限界効用の逓減を示す。2軌跡で2.2%の改善が得られるが、追加の軌跡からの改善は漸減する。

## 実験と評価

### 実験設定

- **教師モデル**: PaLM 2-L（プロンプテッドベースライン）
- **生徒モデル**: PaLM 2-XS、S、Lバリアント
- **報酬モデル**: 指示チューニング済みPaLM 2-L（出力ランキング用）
- **ファインチューニング**: 全パラメータ更新（LoRAではない）
- **検索制限**: 軌跡あたり最大10回の検索

### 評価ベンチマーク

- **Bamboogle**（125問）: 直接検索では解決困難だがWikipedia証拠で回答可能な半敵対的2ホップ質問
- **BamTwoogle**（100問）: 2〜4ホップの推論を要する補完的テストセット

### 主要結果

**蒸留の有効性**: 2回のReST反復後、XSモデルが65.9%の精度を達成（ベースラインLモデルの70.3%に対して）。パラメータ数を2桁削減しながら同等の性能を実現。

**データ品質の優位性**: データ量の増加よりもデータ品質の向上が大きな性能改善をもたらす。第1世代から第2世代への品質向上で9%の精度向上。

**自己修正の限定的効果**: 自己修正ステップは0.5〜1.0%の性能向上に寄与するが、その効果はモデルサイズが大きいほど顕著。

**人間フィルタリングの逆効果**: 予想に反して、人間によるフィルタリングされていないデータの方がフィルタリング済みデータより2.5%高い精度を示した。訓練セットサイズの縮小と部分軌跡の汚染が原因と推測される。

### LLMベースランキング

人間の嗜好に基づく別個の報酬モデルを訓練する代わりに、PaLM 2-Lにサンプリング出力のランク付けを指示する方式を採用。パープレキシティベースのオンポリシーロールアウトを維持しつつ、オフポリシーの軌跡フィルタリングを実現。

## 備考

### データ分析エージェントへの示唆

本研究は、データ分析エージェントの設計において以下の示唆を提供する：

1. **小規模モデルの実用性**: 大規模教師モデルからの知識蒸留により、推論コストを大幅に削減しながら実用的な性能を維持できる
2. **自己改善ループの設計**: Grow-Improveサイクルは、データ分析タスクにおけるエージェントの反復的改善に直接応用可能
3. **品質重視のデータ戦略**: エージェント訓練データの品質が量よりも重要であり、AI フィードバックによる自動品質管理が有効

### 限界と今後の課題

- PaLMファミリーに限定された評価であり、他のモデルファミリーへの汎用性は未検証
- 評価ベンチマークが比較的小規模（125問、100問）であり、大規模評価が必要
- 自己修正ステップの効果が限定的であり、より高度な修正メカニズムの探索が望まれる
- 検索エンジンへの依存により、検索品質がエージェント性能のボトルネックとなりうる

### 関連研究との位置づけ

- **ReAct（Yao et al., 2023）**: 推論と行動の交互実行フレームワーク。本研究の基盤
- **ReST（Gulcehre et al., 2023）**: 強化自己訓練。本研究ではエージェント設定に拡張
- **STaR（Zelikman et al., 2022）**: 自己教師型推論改善。本研究とは訓練信号の取得方法が異なる
- **Toolformer（Schick et al., 2023）**: ツール使用の自己教師学習。本研究はマルチステップ推論に特化
