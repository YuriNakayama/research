## Abstract (原文)

The paper proposes a "puppeteer" paradigm where a centralized orchestrator dynamically directs multiple LLM-based agents ("puppets") through reinforcement learning. The orchestrator is trained to adaptively sequence and prioritize agents, enabling flexible and evolvable collective reasoning. Testing on various scenarios demonstrates improved performance with lower computational requirements, with benefits traced to more compact, cyclic reasoning structures.

## Abstract (日本語)

本論文は、中央集権的なオーケストレーター（「パペティア」）が強化学習を通じて複数のLLMベースエージェント（「パペット」）を動的に指揮する「パペティア」パラダイムを提案する。オーケストレーターはエージェントの順序付けと優先順位付けを適応的に学習し、柔軟かつ進化可能な集団推論を実現する。各種シナリオでの実験により、より低い計算コストで性能が向上することが示され、その利点はよりコンパクトで循環的な推論構造に起因することが明らかになった。

## 1. 概要

本研究は、マルチエージェント協調における組織構造の設計を、強化学習による動的なオーケストレーションへと転換するフレームワークを提案する。従来の静的なエージェント配置（直列、並列、階層型）とは異なり、「パペティア」と呼ばれる中央ポリシーがタスクの状態に応じてエージェントを逐次的に選択・割り当てする。このアプローチにより、エージェント間の協調パターンが学習を通じて自然に進化し、コンパクトかつ効率的な推論グラフが創発される。

主要な技術要素は以下の3点である：
- **Puppeteerポリシー**: マルコフ決定過程としてエージェント選択を定式化
- **REINFORCE最適化**: ポリシー勾配法によるオーケストレーターの学習
- **進化する有向グラフ**: 学習を通じてコンパクト化・循環化するエージェント間トポロジー

## 2. 問題と動機

### 2.1 既存手法の限界

マルチエージェントシステムの既存研究は、以下の問題を抱えている：

1. **静的な組織構造**: エージェントの配置や通信パターンが事前に固定されており、タスクの複雑さや特性に適応できない
2. **スケーラビリティの欠如**: エージェント数やインタラクションの増加に伴い、計算コストが線形または二次的に増大する
3. **手動設計への依存**: 効果的な協調パターンの設計に人間の専門知識が必要であり、汎化性に限界がある
4. **精度と効率のトレードオフ**: 多くの手法で性能向上は計算コスト増大を伴う

### 2.2 研究課題

- エージェントの選択と順序付けをタスクの状態に応じて動的に決定する方法
- 性能と計算効率を同時に最適化可能なフレームワーク
- エージェント間の協調パターンを自動的に発見・進化させるメカニズム

## 3. 提案手法

### 3.1 Puppeteerパラダイム

システム全体を人形劇のメタファーで捉え、以下の構成要素を定義する：

- **Puppeteer（パペティア）**: 中央のオーケストレーションポリシー π(S_t, τ)
- **Puppets（パペット）**: 個々のLLMエージェント群
- **Stage（ステージ）**: タスク実行環境

各タイムステップtにおいて、パペティアはシステム状態 S_t とタスク仕様 τ に基づいてエージェント a_t を選択し、そのエージェントが出力 o_t を生成してシステム状態を更新する。

### 3.2 マルコフ決定過程としての定式化

エージェント選択プロセスをMDPとして定式化：
- **状態**: S_t = 全エージェント状態 + コンテキスト情報の結合
- **行動**: エージェント選択 a_t ∈ A（利用可能なエージェント集合）
- **遷移**: Φ(S_t, a_t, o_t) → S_{t+1}
- **報酬**: r - λ·C_T（正確性 - コストペナルティ）

### 3.3 REINFORCE最適化

ポリシー勾配法（REINFORCE）を用いてパペティアポリシーを最適化する：

- **報酬設計**: 複合報酬関数 = ソリューション品質 - λ × 計算コスト
- **非終端状態**: 割引率 γ とステップ毎コスト C_t = F·log(1+t/φ) を適用
- **ハイパーパラメータ λ**: 精度と効率のトレードオフを制御
- **F**: FLOPs（浮動小数点演算回数）
- **φ**: 最大ステップ予算

### 3.4 進化する有向グラフ

学習の進行に伴い、エージェント間のインタラクショントポロジーが自然に進化し、2つの特徴的な構造が創発する：

1. **コンパクション（圧縮）**: グラフ密度が学習を通じて増加し、エージェントが密な通信サブネットワークにクラスタリングされる。ハブエージェントが反復的な洗練のために優先的に選択される
2. **サイクリシティ（循環性）**: エージェントが以前の協力者と繰り返し関与する循環パターンが出現し、再帰的な批評と情報再利用を可能にする

## 4. アーキテクチャ

### 4.1 システム全体構成

```
タスク仕様 τ
    ↓
Puppeteerポリシー π(S_t, τ)
    ↓ エージェント選択
Agent Pool [Agent_1, Agent_2, ..., Agent_n]
    ↓ 出力生成 o_t
状態更新 Φ(S_t, a_t, o_t) → S_{t+1}
    ↓
終了条件判定 → 最終出力 or ループ継続
```

### 4.2 エージェントプール構成

**Mimas設定（小規模モデル）**:
- Qwen-2.5-7B
- LLaMA-3.1-8B
- Mistral系バリアント

**Titan設定（大規模モデル）**:
- GPT-4バリアント
- Gemini
- Claude
- LLaMA-3.1-405B

### 4.3 ハイパーパラメータ

- **探索幅 W**: エージェント選択の候補数（デフォルト: 4）
- **チェイン深度 D**: 最大推論ステップ数（デフォルト: 2）
- **デフォルト設定**: W44D22（幅4、深度2）が精度と効率の最適バランス

## 6. 実験と評価

### 6.1 評価データセット

| データセット | ドメイン | タスク種別 |
|-------------|---------|-----------|
| GSM-Hard | 閉域 | 算術推論 |
| MMLU-Pro | 閉域 | 知識評価 |
| SRDD | 開域 | ソフトウェア開発 |
| CommonGen-Hard | 開域 | 創造的生成 |

### 6.2 主要実験結果

- **Titan設定**: 平均性能が0.6893→0.7731に向上（+12.2%）
- **トークン消費**: 学習を通じてほぼ全タスクで削減
- **Puppeteer-Mono（単一ベースモデル）**: 大半の単一エージェント手法を上回る
- **Puppeteer（異種モデル混合）**: 多様なモデル間の相補的インタラクションにより最高性能を達成

### 6.3 創発構造の分析

進化したトポロジーの分析により、以下が確認された：
- コンパクション：グラフ密度の増加、ハブエージェントの出現
- サイクリシティ：循環パターンによる再帰的批評構造の形成
- 線形チェーンではなく、木構造・循環・ブランチ間接続を含む有向グラフの形成

### 6.4 効率性の知見

- Titan設定：短い推論チェーンによるトークン削減
- Mimas設定：低コストエージェントの優先選択によるトークン削減
- 性能と効率の同時最適化が可能であることを実証

### 6.5 拡張実験

- **ALFWorld**: 具現化環境（embodied environment）への拡張実験により、テキスト推論を超えたフレームワークの汎用性を実証
- **最適化サンプル数**: 200サンプルでの最適化は複雑タスクには不十分な場合がある

## 7. 備考

### 7.1 学術的意義

本論文は、マルチエージェント協調を強化学習による動的オーケストレーションとして再定式化した点で重要な貢献をしている。従来の静的設計に依存するアプローチから脱却し、タスク特性に応じた協調パターンの自動発見を可能にした。特に、学習を通じてコンパクトかつ循環的な推論構造が創発するという知見は、マルチエージェントシステム設計の新たな方向性を示している。

### 7.2 データ分析エージェントへの示唆

- 動的オーケストレーションパラダイムは、データ分析パイプラインにおけるエージェント間協調にも適用可能
- 分析タスクの複雑さに応じてエージェントの選択と順序を適応的に決定するメカニズムとして有用
- コスト効率の最適化は、LLM APIコスト管理において実践的な意義を持つ

### 7.3 制限事項

- 200サンプルでの最適化は複雑なタスクには不十分な可能性
- 現在のハイパーパラメータは弱いモデル構成に適さない場合がある
- 進化に要する計算コスト自体の評価が限定的

### 7.4 関連研究との位置づけ

NeurIPS 2025採択論文として、マルチエージェントシステムの自動設計における最先端を代表する。AFlow（ワークフロー探索）やEvoAgent（エージェント進化）と比較して、より原理的なRLベースのアプローチを採用している点が差別化要因である。
