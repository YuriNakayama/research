# Markdownウェブビューア — 要件定義

## 背景と目的

日次リサーチパイプラインが生成するMarkdownレポートを、スマホやPCから快適に閲覧できるウェブサービスを構築する。現在はメール通知とGitHub PRでのみ結果を確認しているが、ナレッジベースとしての蓄積・閲覧体験の向上を目指す。

## ユーザーストーリー

- **US-1**: リサーチャーとして、ドメイン別のレポート一覧を表示し、タップで詳細を閲覧したい（数式・テーブル・図表を正しくレンダリング）
- **US-2**: スマホユーザーとして、通勤中にレポートを読みやすいレイアウトで閲覧したい（日本語タイポグラフィ最適化、テーブル横スクロール、ダークモード）
- **US-3**: チームメンバーとして、ログインして認証済みユーザーのみがレポートにアクセスできるようにしたい
- **US-4**: リサーチャーとして、レポート内の見出しから目次で素早くジャンプしたい

## 機能要件

1. **レポート一覧表示**: ドメイン別にレポートを一覧表示。CSVメタデータ（title, authors, year, venue）を表示
2. **レポート詳細表示**: Markdownを完全にレンダリング（数式: KaTeX、テーブル、コードブロック、Mermaid図、画像）
3. **目次（TOC）自動生成**: レポートの見出しから目次を生成。スマホではドロワー、PCではサイドバーに表示
4. **ダークモード**: ライト/ダークテーマの切り替え。システム設定に連動 + 手動切替
5. **モバイル最適化**: レスポンシブデザイン、日本語フォント（Noto Sans JP）、適切な行間・文字サイズ
6. **認証**: Amplify Auth (Cognito) によるログイン制御。招待制（管理者がユーザー追加）

## 非機能要件

- **パフォーマンス**: SSG + CDN配信。FCP 1.5秒以内、LCP 2.5秒以内（モバイル4G環境）
- **アクセシビリティ**: WCAG 2.1 AA準拠。スクリーンリーダー対応、キーボードナビゲーション、十分なコントラスト比
- **セキュリティ**: Cognito認証必須、HTTPS強制、XSS対策（rehype-sanitize）、CSPヘッダー設定
- **コスト**: Amplify無料枠内での運用を目標（月額$5以内）
- **可用性**: Amplify + CloudFront のマネージドサービスに依存（SLA 99.9%）

## スコープ外

- コメント・アノテーション機能
- マークダウンの Web 上での編集機能
- 多言語対応（i18n）— 日本語UIのみ
- ブックマーク・タグ付け機能（将来イテレーション）
- 全文検索（将来イテレーション）

## 用語集

| 用語 | 説明 |
|------|------|
| ドメイン | リサーチの分野分類（例: legal_tech）。research-config.yaml の domains で定義 |
| デイリーレポート | 日次パイプラインで自動生成されるMarkdownレポート（docs/daily/<domain>/report/） |
| リサーチレポート | 手動実行で生成されるMarkdownレポート（docs/research/<domain>/report/） |
| CSVメタデータ | レポートの元論文情報（title, url, authors, year, venue, summary, status） |
