## 読む順序

**[03] One-hot news** → **[01] CISI-Net** → **[06] ACFR** → **[02] CPA** → **[05] Overlap Violations** → **[04] CaML**

理由は明快である。**03 を最初に読まなければ、01・02 を読んだ後に「実装しよう」という誤った結論に至る**。03 は検証設計を先に固めさせるための文献であり、実装の前提条件を規定する。

- **03（反証）を最初に**: 何を検証すべきかを先に確定させる。ここで規定される one-hot ベースラインと leave-one-campaign-out が、以降すべての手法の評価軸になる。
- **01（中核手法）**: ユーザーのアイデアの既存実装を把握する。マーケティング実データでの検証事例。
- **06（連続処置）**: 01 の balancing とクーポン額の連続性の緊張を理解する。01 とセットで読むべき。
- **02（雛形）**: 加法分解と用量スケーリングという、移植価値の高い設計を得る。
- **05（テキスト）**: 訴求文面を扱う際の識別上の地雷を知る。
- **04（ゼロショット）**: 未実施施策への接続。C3 への橋渡し。

## 施策特徴量化は成立するか — 賛否の証拠

### 賛成側の証拠

**[01] CISI-Net** は、task embedding による施策パターン間のパラメータ共有が機能することを、シミュレーション（$K=3$、$N=50{,}000$）と**日本のモバイル決済プラットフォームの販促データ 2 種**（71 共変量）で示した。TARNet・CFR-WASS・TECE-VAE・NCoRE を上回り、同一加盟店内の販促は相乗効果、加盟店間はカニバリゼーションという、ドメイン知識と整合する解釈可能な知見を得た。**マーケティングという同一ドメインでの検証**である点が最大の強みである。

**[02] CPA** は、「基底状態 + 摂動埋め込み + 共変量埋め込み」の加法分解により、未観測の薬剤組み合わせを **$R^2 = 0.81$** で予測した（訓練事例ゼロ）。用量を埋め込みの非線形スケーリング（$s(0)=0$ 制約付き）として扱うことで、未観測用量への内挿を構造的に実現した。chemCPA 拡張では、事前学習済み分子エンコーダを固定し小さな写像のみを学習することで、**完全に未観測の薬剤**への予測を可能にした。

**[04] CaML** は、**745 単剤 / 10,325 perturbagen** という規模で、ゼロショット予測が**テスト介入のデータで直接訓練したベースラインを上回る**（RATE@0.999 で 0.48 vs 0.40）ことを示した。単剤のみの訓練から未観測の薬剤ペアへ汎化した。理論的な汎化バウンドも与えている。

### 反証側の証拠

**[03] One-hot news** は、5 つの主要な薬剤相乗効果予測モデル（DeepSynergy・DeepDDS・MatchMaker・MARSY・JointSyn）において、**分子特徴量を one-hot 符号化（＝単なる ID）へ差し替えても性能がほぼ変わらない、むしろ向上する**ことを示した。

| 検証 | 結果 |
|------|------|
| MatchMaker (DrugComb, LPO) | 特徴量 MSE 97.23 vs one-hot 100.25 — 差はわずか **3.11%** |
| DeepSynergy (O'Neil, LTO) | one-hot が **0.91% 良い** |
| MARSY (DrugComb, LTO) | one-hot が **6.4% 良い** |
| MatchMaker LDO（両方新規薬剤） | 特徴量 PCC **0.13** vs one-hot 0.10 — **双方とも破綻** |

さらに決定的なのが**合成データによる機序の特定**である。

| データ構造 | 真の特徴量の回復（Jaccard） |
|-----------|--------------------------|
| **反復なし**（エンティティの再利用なし） | **1.00 — 完全回復** |
| **反復あり**（50 薬剤 × 50 細胞株の完全交差）、LDO | **0.38 / 0.43 — 回復不良** |

**モデルは特徴量を学習する能力を持っている。しかし同じエンティティが繰り返し登場する構造になると、ラベルの共変動を利用する方が容易になり、特徴ベースの学習が阻害される。** 事前学習済み表現（MoLFormer）でも、残差モデリングでも改善しなかった。

### 賛否をどう秤にかけるか — 決定的な観察

賛成側と反証側は、**矛盾しているのではなく、異なるデータ構造を報告している**。両者を分ける変数は **エンティティ（介入）の数** である。

| 文献 | 介入数 | 結果 |
|------|-------|------|
| [04] CaML (LINCS) | **10,325** | 汎化成功 |
| [04] CaML (Claims) | **745** | 汎化成功 |
| [02] CPA (sci-Plex3) | **188 化合物** | 未観測用量・組み合わせへ汎化 |
| [03] One-hot news (DrugComb) | 3,057 薬剤 | LDO で PCC 0.13 に**崩壊** |
| [03] One-hot news (O'Neil) | **38 薬剤** | one-hot が**上回る** |
| [01] CISI-Net (実データ) | **3 施策 / 7 パターン** | one-hot 比較は**未実施** |
| **本課題** | **数十本（推定）** | **—** |

**ユーザーの施策数は、ショートカットが最も起きやすい領域（O'Neil の 38 薬剤と同等かそれ以下）に位置する。** CaML が成功した介入数より 1〜2 桁少ない。

さらに悪いことに、**本課題のデータ構造は One-hot news の「反復データセット」（Jaccard 0.38、回復不良）そのものである**。1 つの施策が数千〜数万のユーザーへ配信される＝同一エンティティが大量に反復する構造であり、「非反復データセット」（Jaccard 1.00、完全回復）とは正反対である。

**MatchMaker が 426,239 組み合わせ・3,057 薬剤という大規模データを持ちながら LDO で PCC 0.13 に崩壊した**事実は、決定的な含意を持つ。**サンプル数（行数）を増やしても、エンティティの多様性が足りなければ汎化しない。** これは「施策を特徴ベクトル化すればデータが実質的にプールされて増える」という期待への直接の反証である。増えるのは行数であって、**施策の多様性ではない**。

加えて、賛成側の 2 本には検証上の穴がある。

- **[01] CISI-Net は one-hot ベースラインとの比較を報告していない**（少なくとも本調査の取得範囲では未確認）。実データは 3 施策・7 パターンであり、task embedding は実質的に 7 個のルックアップテーブルと区別がつかない。
- **[02] CPA の摂動埋め込みは、明示的に「学習可能なベクトル辞書」＝ ID ルックアップそのものである**。しかも CPA は One-hot news と**同じ創薬分野**の手法であり、批判の射程内にある。CPA 自身も「性能は訓練条件からの非類似度と逆相関する」「chemCPA は最低限の用量事例を要し、単剤観測を完全に除外すると性能が大幅に低下する」と認めている。**CPA が示したのは訓練分布の近傍への汎化であって、真のゼロショットではない。**

### 判定

**ユーザーのアイデアは論理的には正しいが、本課題のデータ構造は、それが機能するための条件を満たしていない可能性が高い。**

より正確には以下である。

1. **施策の特徴ベクトル化は必要条件であって十分条件ではない。** 特徴量を与えてもモデルがそれを ID として使うだけなら、パラメータ共有もゼロショット汎化も起きない。
2. **機能するか否かはデータ構造（施策の本数と反復構造）に依存し、本課題は最も不利な側にある。** 施策数が数十本で、各施策が大量に反復するという構造は、One-hot news がショートカットを特定した条件そのものである。
3. **しかしこれは検証可能な問いであり、安価に答えが出る。** one-hot ベースラインと leave-one-campaign-out の比較は数日で実装でき、数ヶ月の実装投資の可否を判定できる。
4. **失敗しても撤退先がある。** One-hot news の著者が推奨する用途限定——「テスト済みパネル内での優先順位付け」＝「実施済み施策内でのターゲティング最適化」——は、それ自体が十分に価値のある成果である。**未実施施策の予測だけが目的でないなら、プロジェクトは死なない。**

**したがって、実装に着手する前に検証を行うべきである。** 以下がその設計である。

## 実装前に実施すべき検証設計

**これは実装の付随作業ではない。実装に着手すべきかを決める意思決定そのものである。** 順序と撤退基準を含めて、着手前に確定させる。

### フェーズ 0: データ構造の実地確認（数時間）

コードを書く前に、以下を数える。

| 確認項目 | 危険信号 | 根拠 |
|---------|---------|------|
| **異なる施策の本数**（行数ではない） | 50 本未満 | [03] O'Neil の 38 薬剤でショートカットが発生 |
| 施策あたりの平均サンプル数 | — | [01] は交互作用推定に $N \geq 10{,}000$ を要求 |
| **施策特徴量で各施策が一意に識別できるか** | できてしまう | 識別できるなら特徴量は ID と情報量的に等価 |
| クーポン額の実施水準数 | 5 未満 | 連続処置手法（[06]）の前提が崩れる |
| **訴求文面のユニーク数** | 施策数と同じ | [05] の DistilBERT ファインチューニングが成立しない |
| 施策と時期の交絡 | 各施策が特定月のみ | 施策効果と季節効果が識別不能 |
| 施策ごとの非配信群（コントロール）の有無 | 欠けている | [04] の pseudo-outcome が構成できない |

**施策本数が 50 未満、かつ施策特徴量で各施策が一意に識別できる場合、フェーズ 1 の結果は否定的になる公算が高い。** それでもフェーズ 1 は実施する価値がある（安価であり、判定が確定する）。

### フェーズ 1: leave-one-campaign-out + ablation（数日）

**評価プロトコル**。[03] の 5 分割戦略を本課題へ読み替える。

| 略称 | 本課題での定義 | 用途 |
|------|--------------|------|
| **ランダム分割**（LTO 相当） | 行をランダムに分割 | 報告はするが**意思決定には使わない** |
| **未実施の組み合わせ**（LPO 相当） | 「額 × 文面」の未実施の組み合わせを除外 | 中間的な難易度 |
| **未配信セグメント**（LCO 相当） | 特定のユーザーセグメントを除外 | ターゲティング汎化 |
| **leave-one-campaign-out**（LDO 相当） | **1 施策を丸ごと訓練から除外し、その施策で評価。全施策で繰り返す** | **主判定軸** |

**比較すべきモデル群**。これらを並べて leave-one-campaign-out で評価する。

| モデル | 役割 |
|--------|------|
| **A. one-hot ベースライン** | 施策 ID の one-hot のみ。**主要な比較対象** |
| **B. 平均予測器** | 全体平均、施策平均、セグメント平均 | 
| **C. 施策特徴量モデル** | 額 + 文面埋め込み + チャネル |
| **D. shuffled features** | 特徴量を施策間でシャッフル（分布保持、対応破壊）。**C と差がなければ特徴量の中身は使われていない** |
| **E. ablation 群** | 額のみ / 文面のみ / チャネルのみ / 各 1 つを除外 |

**注**: leave-one-campaign-out では、除外された施策の one-hot ベクトルは訓練時に存在しない。モデル A は未知施策に対して構造的に予測できないため、A の LDO 性能は「施策特徴量が何も寄与しない場合の下限」を与える。**モデル C が A を明確に上回らなければ、施策特徴量は汎化に寄与していない。** 同時に、ランダム分割では A が C に匹敵しうる（[03] の結果）ため、両分割の対比自体が診断になる。

### フェーズ 2: 合成データによる機序の切り分け（数日、フェーズ 1 が否定的な場合）

[03] の合成データ実験を再現する。**既知の「施策特徴量 → 効果」の関係を人工的に構築**し、**実データと同じ施策数・同じ反復構造**でモデルが特徴量を回復できるかを見る。

- **回復できる** → 問題はモデル・特徴量設計にある。改良の余地がある。
- **回復できない** → **問題はデータ構造にある。モデルを改良しても解決しない。** [03] の機序（反復構造が特徴学習を阻害する）が本課題で再現されたことになり、撤退または用途限定の判断根拠になる。

この切り分けは決定的である。「モデルが悪いのか、データが悪いのか」を答えの出ない議論にせず、実験で決着させる。

### 撤退基準（着手前に宣言する）

**事後に基準を動かせば、この検証全体が無意味になる。** 以下を実験前に文書化する。

1. モデル C（施策特徴量）が leave-one-campaign-out でモデル A・B を **明確に上回らない**場合 → 施策特徴量化は機能していないと結論する。
2. モデル D（shuffled）がモデル C と**同等**の場合 → 特徴量の中身は使われていない。同上。
3. フェーズ 2 で合成データでも特徴量が回復できない場合 → データ構造の問題であり、モデル改良では解決しない。

**撤退先**: [03] の著者が推奨する用途限定に従い、「**実施済み施策内でのターゲティング最適化**」へ目標を切り替える。これは達成可能性が高く、実務価値も十分にある。未実施施策のゼロショット予測を諦めることは、プロジェクトの失敗ではない。

### フェーズ 1 が肯定的だった場合の実装順序

1. **[04] の pseudo-outcome 設計を採用する**。成果を直接予測せず、RA-learner 等で CATE 推定値を作り回帰ターゲットにする。**施策数に依存せず効く設計**であり、最優先で移植する。
2. **[02] の加法分解と用量スケーリング**（$s(0)=0$ 制約）を骨格にする。深層生成モデル全体ではなく、成果回帰として組む。
3. **[01] の balancing penalty** を追加する。[02] には交絡調整がなく、非無作為ターゲティングに耐えないため必須。
4. **[06] の警告に従い $\gamma$（balancing 強度）を掃く**。強くかけすぎるとクーポン額と反応の関係が表現から消える。dose-response 曲線が平坦化しないかを可視化する。
5. **[05] に従い訴求文面の扱いを決める**。文面全体を処置にせず、二値属性へ分解する。overlap の回復（傾向スコアが 0/1 に張り付いていないか）を必ず確認する。
6. **訓練条件からの距離を不確実性指標として出力する**（[02] の知見）。新規施策が施策特徴空間のどこに落ちるかを可視化し、遠いものは予測を信用しない運用ルールを設ける。
7. **最終的には A/B テストによる前向き検証**を行う。[04] が限界として挙げる通り、後ろ向きデータでの検証は前向きの効果を保証しない。

## 未解決の論点（文献に処方箋が存在しない）

1. **balancing の要件がモダリティごとに矛盾する**。[01] は施策パターン間で Wasserstein により分布を揃えよと言い、[06] は連続処置で独立性を強制すると情報を失うとして KL + attention による経路分離を提案し、[05] はテキスト処置では overlap が原理的に破綻すると言う。**「クーポン額（連続）× チャネル（離散）× 文面（テキスト）」を単一の表現へ統合するときの統一的な処方箋は、6 本のいずれにも存在しない。** ここが本課題の実質的な研究課題である。
2. **[01] の Wasserstein と [06] の KL のどちらを採るか**。[06] は高次元で IPM の近似が信頼できないと批判する。文面埋め込みを含む高次元の施策表現なら、この批判が当たる。
3. **施策数が数十本という制約下で、事前知識の注入が不可避である可能性**。GEARS（知識グラフ）や LLM の事前学習済み埋め込みのように、**学習すべき自由度を減らす**方向が、[03] の反証を踏まえるとむしろ本命になりうる。ただし [03] は MoLFormer（事前学習済み分子表現）でもショートカットを防げなかったと報告しており、**この路線にも保証はない**。
