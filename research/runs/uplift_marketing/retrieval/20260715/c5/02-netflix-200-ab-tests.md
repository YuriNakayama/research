## 一言で言うと

Netflix の 200 件の A/B テスト・1098 のテストアームを使い、**14 日分のデータから構成した surrogate index による意思決定が、63 日目の処置効果を直接測定した意思決定と約 95% 一致する**ことを実証した。しかも有効だったのは線形の "auto-surrogate" という最も単純な構成である。

## 問題設定

surrogate index approach は短期成果から長期的インパクトを推定する手法として近年広く用いられるようになった（"have recently become a popular method"）。しかし **それが実際の意思決定ツールとしてどの程度機能するか** の大規模な実証評価は乏しい。

本論文の問いは理論的な識別可能性ではなく、きわめて実務的である。

> 14 日のデータから作った surrogate index で下す判断は、63 日待って直接測定して下す判断と、どの程度一致するのか。

この問いの立て方が本課題の関心と完全に一致する。本課題が求めているのは正確な効果量ではなく「次の施策をどう設計するか」の判断材料であり、評価軸は **推定精度ではなく意思決定の一致率** であるべきだからである。

## 手法

### 記法と設定

- $Y_{63}$: 63 日目に直接測定される長期成果
- $S_{14}$: 14 日目までに観測される短期成果ベクトル
- $\tau_{63} = \mathbb{E}[Y_{63}(1) - Y_{63}(0)]$: 直接測定される長期処置効果
- $\hat\tau_{\text{SI}}$: surrogate index に基づく処置効果推定量

### auto-surrogate

本論文が焦点を当てるのは **線形の "auto-surrogate" モデル** である。abstract の定義は "linear 'auto-surrogate' models that utilize the shorter-term observations of the long-term outcome of interest"、すなわち **関心のある長期成果そのものの、より短期の観測値を surrogate として使う** 構成である。

一般の surrogate index が「質的に異なる複数の proxy」（→ [01](01-the-surrogate-index.md) の $S$ = 開封・クリック・初回購入）を合成するのに対し、auto-surrogate は **同じ指標の時間的に早い断面** を使う。

$$
h(S_{14}) \;=\; \beta_0 + \sum_{t=1}^{14} \beta_t \, Y_t
$$

ここで $Y_t$ は $t$ 日目に観測された長期成果と同種の指標である。長期成果が「63 日間の視聴時間」なら、auto-surrogate は「1〜14 日目の各日の視聴時間」の線形結合になる。

本課題に翻訳すると、$Y$ = 6 ヶ月 LTV に対する auto-surrogate は **「初月の購買額」**（同じ「購買額」という指標の早い断面）であり、開封・クリックのような質的に異なる proxy ではない。

この単純さが実務上の含意を持つ。auto-surrogate は「別種の proxy が長期成果を媒介するか」という難しい問いを回避し、「同じ指標の早期値が後期値を予測するか」という遥かに素直な問いに置き換えている。surrogacy assumption は形式的には同じ

$$
Y_{63} \;\perp\!\!\!\perp\; W \;\bigm|\; S_{14}
$$

であるが、$S_{14}$ が $Y_{63}$ と同種の指標であることで、媒介の完全性が構造的に成り立ちやすい（処置の効果が視聴時間に現れるなら、それは 14 日目までの視聴時間にも現れているはず）。

### 評価の設計

理論的な推定誤差ではなく **意思決定の一致** で評価する。

1. 各テストアームについて $\hat\tau_{\text{SI}}$（14 日 surrogate index）と $\hat\tau_{63}$（63 日直接測定）を計算する
2. それぞれから導かれる統計的推論（有意性判定・符号）を比較する
3. 一致率を測る
4. さらに「launch する」判断（正かつ統計的に有意）に限定して precision / recall を評価する

## 本課題への適用可能性

### 効く点

**1. 評価軸が本課題の関心と完全に一致する**

「意思決定の一致率」「launch 判断の precision / recall」という評価設計は、本課題にそのまま流用できる。効果量の小数点以下の精度ではなく **次の施策をどう設計するかの判断が変わらないか** が問題だからである。RMSE で評価すると実務価値を見誤る。**この評価設計の流用が、本論文から得られる最大の実務的資産**かもしれない（数値そのものより設計思想の方が転用可能性が高い）。

**2. 線形 auto-surrogate という最小構成の正当化**

95% の一致を達成したのが線形モデルであった事実は、**いきなり複雑な proxy 合成や表現学習に走る必要がない**ことを示す。本課題での第一歩は明確である。

> $Y$ = 6 ヶ月 LTV に対して、$S$ = **初月の購買額**（と、その週次分解）を線形で回す。

開封・クリックのような質的に異なる proxy の合成は、auto-surrogate がどこまで行けるかを測ってからでよい。この順序は実装コストの面でも仮定の強さの面でも合理的である。

**3. auto-surrogate は surrogacy assumption が成り立ちやすい**

前述の通り、$S$ と $Y$ が同種の指標であることは媒介の完全性を構造的に支える。「クーポンが LTV に効くなら、その効果は初月の購買額にも現れているはず」という主張は、「クーポンの効果はすべて開封とクリックを経由する」という主張より遥かに信じやすい。**本課題で最初に試すべきは、gather の議論が想定していた質的に異なる proxy 群ではなく、auto-surrogate である**というのが本論文からの実務的な示唆である。

**4. 高頻度環境での「答え合わせ」の存在自体が資産**

Netflix は 200 テスト分の答え合わせができた。本課題ではできない。しかし **他社が答え合わせをして「線形 auto-surrogate は概ね機能する」という事前分布を提供してくれた**こと自体に価値がある。本課題は自前で 200 テストを積む代わりに、この事前分布を借りて少数の自前検証で確認する、という戦略を取れる。

### 効かない/リスク点

**1. これは「200 の実験があるからできた研究」である —— 本課題の核心的困難がここに凝縮している**

本論文は本課題のパラドックスの最も鮮明な例示になっている。

> Netflix は **surrogate index が使えるかどうかを検証するために、200 件の A/B テストで 63 日待った**。

つまり本論文の結論を自前で得るには、待ち時間を短縮したい当の対象について、待ち時間を短縮せずに大量に実験する必要がある。本課題（年 2〜4 回の施策）では、同等の検証に **50〜100 年** かかる計算になる。これは誇張ではなく単純な割り算である。

したがって本課題での本論文の使い方は 1 つしかない。**自前で再現するのではなく、結論を事前分布として借りる**。ただし借用の妥当性は次項の理由で限定的である。

**2. Netflix の成果指標と LTV では auto-surrogate の成立しやすさが違う**

Netflix の長期成果（視聴時間・継続利用と推測されるが **具体的な指標は未確認**）は **習慣的・連続的** な行動である。14 日目までの視聴が 63 日目の視聴を予測するのは、視聴が習慣として持続するからである。

対して本課題の LTV に対する「初月購買額」は事情が異なる。**クーポンによる初回購入は、習慣の形成ではなく一時的な価格反応かもしれない**。この場合、初月購買額が高いこと自体が LTV の高さを意味しない —— むしろ「値引き時のみ買う層」を拾っている可能性がある。auto-surrogate の成立は指標の性質に依存し、**購買（離散的・価格弾力的）は視聴（連続的・習慣的）より成立しにくい**と考えるべき合理的な理由がある。

**3. 95% という数値は Netflix の実験ポートフォリオの性質に依存する**

一致率は「テスト群の効果量の分布」に強く依存する。効果が大きいテストが多ければ一致率は上がり、ゼロ近傍のテストが多ければ下がる。Netflix の UI 変更等の施策群と、クーポン施策では効果量の分布が違う。**95% をそのまま期待してはならない**。

**4. 施策の異質性 —— Netflix の 200 テストは「同じプロダクト上の変更」**

Netflix のテストは同一プロダクト・同一母集団上の変更であり、比較的均質である。対して本課題の施策は **対象ユーザー・訴求・クーポン額が毎回異なる**。前回施策で構成した index が今回も有効という保証がない（surrogate の妥当性は intervention-specific）。この点で本論文の設定は本課題より遥かに恵まれている。

**5. 母集団の時間的安定性 —— 63 日 vs 数ヶ月**

Netflix の 63 日というスケールでは母集団はほぼ安定している。本課題の施策間隔（数ヶ月）では、その間に顧客層・商品構成・季節性が変化する。前回検証した index が今回も有効とは限らない。

**6. surrogate paradox に対する保証にはならない**

95% 一致は「概ね合う」ことを示すが、**残り 5% に何が含まれるか**が問題である。もしその 5% が「短期は正、長期は負」という符号反転（= 本課題で最も恐れるべき安売り常連化のパターン）に偏っているなら、平均的な一致率の高さは慰めにならない。gather 段階が引用していた「誤 launch ゼロ」という記述は本文未確認であり、**この非対称な安全性は現時点では根拠を確認できていない**。→ [03](03-imperfect-surrogates-many-weak-experiments.md) が扱う問題であり、本論文だけで安心してはならない。

**7. 79% / 65% の解釈が確定していない**

前述の通り、abstract の文面からこの 2 数値の指標名を確定できない。仮に recall が 65% なら、**直接測定なら launch したはずの施策の 3 分の 1 を取りこぼす**ことを意味する。年 2〜4 回しか施策を打てない本課題で 3 分の 1 の取りこぼしは、高頻度環境より遥かに重い（次の機会が数ヶ月後だから）。この数値の解釈確定は実務判断に直結する。

## 実装ステップ

1. **本文 PDF の入手**: abstract のみで本レポートを書いており、79%/65% の指標名、誤 launch の有無、限界の記述が未確認。arXiv の HTML 版（ar5iv 等）を試すか、別経路で PDF を取得する。

2. **auto-surrogate を第一候補として設計する**: $Y$ = 6 ヶ月累積購買額に対し、$S$ = 初月の週次購買額ベクトル `[week1_amount, week2_amount, week3_amount, week4_amount]`。gather が想定していた開封・クリックからではなく、**ここから始める**。

3. **観察データで線形 $h$ を学習する**: $Y$ が確定済みの過去顧客で $\hat{Y} = \beta_0 + \sum_t \beta_t Y_t$ を回帰。線形から始め、複雑化は性能が足りない場合のみ。

4. **意思決定の一致率で評価する（RMSE ではなく）**: 長期成果が確定済みの過去施策について、以下の 2×2 分割表を作る。

   | | 直接測定: launch | 直接測定: 見送り |
   |---|---|---|
   | **surrogate: launch** | 一致 | 誤 launch（**最重要**） |
   | **surrogate: 見送り** | 取りこぼし | 一致 |

   本課題では **誤 launch セル（特に直接測定が有意に負のケース）をゼロに保つ**ことを最優先の受け入れ基準とする。

5. **習慣性の検証 —— auto-surrogate が成り立つ前提を測る**: クーポン経由の初回購入者について、初月購買額と 6 ヶ月 LTV の相関を、自然発生の購買者と比較する。**両者で相関構造が違えば auto-surrogate は使えない**（安売り常連化の兆候そのもの）。これが本課題における本論文の適用可否を決める単一の最重要チェックである。

6. **95% を目標値にしない**: 自前の一致率を測り、それが 70% でも 80% でも、その数値を前提に意思決定の設計をする。Netflix の数値は借り物の事前分布であって目標ではない。
